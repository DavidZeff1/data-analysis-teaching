Reports & Publishing

Report Structure

Report (.pbix file)
  └── Page 1
  │     └── Visuals, slicers, text
  └── Page 2
  │     └── Visuals, slicers, text
  └── Page 3
        └── ...

Each page is like a slide in a presentation

Adding Pages

// New page
Click + icon next to page tabs
OR
Right-click tab → New page

// Rename page
Double-click tab → Type new name

// Duplicate page
Right-click tab → Duplicate page

// Page order
Drag tabs to reorder

Design Best Practices

Layout
- Most important visual → top left
- Related visuals grouped together
- Consistent alignment (use guides)
- White space is good!

Colors
- Use a consistent color palette
- View → Themes to apply preset themes
- Or create custom theme (JSON)

Navigation
- Clear page names
- Use buttons for navigation
- Consider bookmarks for different views

Adding Text & Images

// Text box
Insert → Text box
- Add titles, descriptions
- Format with rich text options

// Image
Insert → Image
- Add logos, icons
- Can link to URLs

// Shapes
Insert → Shapes
- Rectangles for grouping
- Lines for separation

Bookmarks & Buttons

// Bookmarks - save a view state
View → Bookmarks pane → Add
- Captures: filters, slicer selections, visible visuals

// Buttons - for navigation
Insert → Buttons → choose type
- Assign action: Bookmark, Page navigation, URL

Example: Toggle between views
1. Create "Chart View" bookmark
2. Create "Table View" bookmark  
3. Add button → Action → Bookmark → select

Publishing

// Publish to Power BI Service
Home → Publish
- Sign in with work/school account
- Select workspace
- Report appears in app.powerbi.com

// Sharing options (in Power BI Service)
- Share with specific users
- Publish to web (public link)
- Embed in Teams/SharePoint
- Create an App (bundle of reports)

// Scheduled refresh
Dataset settings → Scheduled refresh
- Set times to auto-update data

Mobile Layout

// Create mobile-optimized view
View → Mobile layout
- Drag visuals to phone canvas
- Resize for mobile screen
- Different layout, same data

Users see mobile layout in Power BI app

Key intuitions

  • Build for your audience - executives want summaries, analysts want detail
  • Use page hierarchy: Overview → Detail pages
  • Bookmarks are powerful for toggling views without new pages
  • Test cross-filtering - click around to verify interactions
  • Save frequently (.pbix file)
  • Performance: fewer visuals per page = faster loading

Practice

  1. Create a 3-page report: Overview, Sales Detail, Profit Analysis
  2. Add a title text box to each page
  3. Apply a theme (View → Themes)
  4. Create two bookmarks: one with filters applied, one without
  5. Add buttons to toggle between the bookmarks
