Statistics for Data Analysis

Why Statistics Matters

Statistics helps you make confident decisions from data. As a data analyst, you'll use statistics daily to summarize data, compare groups, detect patterns, and determine if your findings are real or just random noise.

This section covers: The practical statistics you'll actually use on the job - not academic theory, but tools for real analysis. Every concept includes real examples and exercises using sample datasets.

Sample Datasets Used in This Section:

  • ‚Ä¢ E-commerce Orders: 10,000 orders with revenue, quantity, customer info
  • ‚Ä¢ Website Analytics: Daily visitors, conversions, bounce rates
  • ‚Ä¢ Employee Data: Salaries, departments, tenure, performance
  • ‚Ä¢ A/B Test Results: Control vs treatment conversion data

üöÄ The Analyst's Real-World Workflow

In the real world, nobody hands you a dataset and says "calculate the standard deviation." They ask vague questions. Here is how you actually use these tools:

1

"Why are sales down this week?" (The boss never asks about p-values)

2

Use Mean/Median to see the trend. Use Standard Deviation to see if it's just normal noise.

"Average daily sales are $9k, down from $10k. But standard deviation is $2k, so a $1k drop might be random."

3

If you make a change to fix it, use A/B Testing to prove it worked.

Summarize and describe your data. This is where every analysis starts. Before building models or running tests, you need to understand what your data looks like.

‚úì Best for: Categorical data

A Z-score tells you how many standard deviations a value is from the mean.

Z = (Value - Mean) / Std Dev

Example: Is a $150 order unusual?

  • |Z| 1-2: Somewhat unusual
  • |Z| 2-3: Unusual (~5% of data)
  • |Z| > 3: Very rare, possible outlier

  • ‚Ä¢ Time ordering + theory - Cause must precede effect, plus logical explanation

4. A/B Testing & Statistical Significance

The most important practical skill for data analysts. Determine if observed differences are real or just random chance.

What is A/B Testing?

Compare two versions of something to see which performs better, using randomization to establish causation.

The Question: Is the 0.3 percentage point difference REAL, or just random chance?

The Logic of Hypothesis Testing

We can't prove B is better. Instead, we try to rule out that the difference is due to chance.

Step 1: State the Hypotheses

H‚ÇÄ (Null): "No real difference"

The default assumption we try to disprove

H‚ÇÅ (Alternative): "There IS a difference"

What we're trying to prove

Step 2: Calculate the p-value

"If there were truly no difference, what's the probability of seeing results this extreme by random chance?"

Small p-value = unlikely to be chance = evidence against H‚ÇÄ

üí° Understanding p-values (This is crucial!)

What p-value IS:

"Assuming the null hypothesis is true (no real difference), the probability of observing a result at least as extreme as what we got."

Example: p = 0.03

"If A and B were truly identical, there's only a 3% chance we'd see a difference this large by random luck."

Since 3% is pretty unlikely, we conclude the difference is probably real.

What p-value is NOT:

  • ‚ùå NOT "the probability B is better than A"
  • ‚ùå NOT "the probability the null hypothesis is true"
  • ‚ùå NOT a measure of how BIG the effect is

üßÆ How to Calculate Significance (Z-Test) from Scratch

Going beyond the napkin math. This is how we get the exact p-value.

1. The Inputs

Control (A)

Users ($n_1$): 10,000

Conv ($x_1$): 320

Rate ($p_1$): 0.032

Treatment (B)

Users ($n_2$): 10,000

Conv ($x_2$): 350

Rate ($p_2$): 0.035

2. Pooled Probability ($\hat{p}$)

If H‚ÇÄ were true, both groups are the same. What's the overall rate?

$\hat{p} = \frac{x_1 + x_2}{n_1 + n_2} = \frac{320 + 350}{20,000} = \frac{670}{20,000} = 0.0335$

3. Standard Error (SE)

How much do we expect the difference to vary by chance?

$SE = \sqrt{\hat{p}(1-\hat{p})(\frac{1}{n_1} + \frac{1}{n_2})}$

$SE = \sqrt{0.0335(0.9665)(\frac{2}{10,000})} = \sqrt{0.0323(0.0002)} \approx 0.00254$

4. Significance Calculation

How many standard errors apart are they?

$Z = \frac{p_2 - p_1}{SE} = \frac{0.035 - 0.032}{0.00254} = \frac{0.003}{0.00254} = 1.18$

Lookup Z=1.18: p-value ‚âà 0.238 (or 23.8%)

Not Significant (p > 0.05)

‚ö†Ô∏è Common A/B Testing Mistakes

1. Peeking at Results

Checking p-values repeatedly and stopping when you see significance inflates false positive rate.

‚úì Fix: Decide sample size upfront, don't stop early

2. Underpowered Tests

Not enough users to detect the effect size you care about.

‚úì Fix: Calculate required sample size before starting

3. Multiple Testing Without Correction

Testing 20 metrics? One will be "significant" by chance alone.

‚úì Fix: Pre-specify primary metric, use Bonferroni correction for multiple comparisons

4. Ignoring Practical Significance

A 0.01% lift can be "statistically significant" with enough data, but meaningless for business.

‚úì Fix: Ask "Is this effect size worth acting on?"

5. Confidence Intervals

Instead of a single estimate, give a range that likely contains the true value. Essential for communicating uncertainty.

What is a Confidence Interval?

Average Order Value Estimate

$82
$94

$88 (95% CI: $82 - $94)

Correct interpretation:

"We are 95% confident that the true population average order value is between $82 and $94."

What this means technically:

If we repeated this sampling process 100 times, about 95 of those intervals would contain the true population value. We don't know if THIS particular interval contains the truth, but the method works 95% of the time.

üßÆ How to Calculate a Confidence Interval from Scratch

Want to put a range on your estimate? Here is the recipe.

1. Get Sample Stats

  • Sample Mean ($\bar{x}$) = $100
  • Std Dev ($s$) = $20
  • Sample Size ($n$) = 64

2. Calculate Standard Error (SE)

The uncertainty of the mean decreases as square root of n.

$SE = \frac{s}{\sqrt{n}} = \frac{20}{\sqrt{64}} = \frac{20}{8} = 2.5$

3. Choose Confidence Level (Z)

How sure do you want to be?

  • 90% Confidence ‚Üí Z = 1.645
  • 95% Confidence ‚Üí Z = 1.96 (Standard)
  • 99% Confidence ‚Üí Z = 2.576

4. Margin of Error ($ME$)

$ME = Z \times SE = 1.96 \times 2.5 = 4.9$

5. The Result

Interval = [Mean - ME, Mean + ME]

Interval = [100 - 4.9, 100 + 4.9]

[ $95.10, $104.90 ]

Methods for Detecting Outliers

Z-Score Method

How many std devs from the mean?

Z = (value - mean) / std_dev

Common threshold: |Z| > 3

‚úì Good for: Normal distributions

‚úó Affected by outliers themselves

IQR Method (Tukey's Fences)

Based on quartiles - robust to skew

Lower fence: Q1 - 1.5 √ó IQR

Upper fence: Q3 + 1.5 √ó IQR

Outside fences = outlier

‚úì Good for: Skewed data, robust to extremes

‚úì Used in box plots

What To Do With Outliers

Keep them if:

  • ‚Ä¢ It's a valid data point (big customer, holiday spike)
  • ‚Ä¢ It represents important edge cases
  • ‚Ä¢ You're analyzing the full distribution
  • ‚Ä¢ The outlier IS the insight (fraud, errors to fix)

Remove/investigate if:

  • ‚Ä¢ It's likely a data entry error
  • ‚Ä¢ It's a test transaction
  • ‚Ä¢ It would distort your analysis
  • ‚Ä¢ You can verify it's invalid

üí° Best practice: Always document what you did with outliers and why. "Removed 12 orders over $10,000 as they were identified as wholesale bulk orders, not individual customer purchases."

Translating Statistics for Business Audiences
Technical Language Business Translation
"p-value is 0.02" "We're confident this improvement is real, not random chance"
"Correlation is 0.7" "These two things are strongly related - when one goes up, the other usually does too"
