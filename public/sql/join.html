JOIN

What it does

JOIN combines rows from two or more tables based on a related column between them.

Types

INNER JOIN  - Only matching rows from both tables
LEFT JOIN   - All rows from left table + matches from right
RIGHT JOIN  - All rows from right table + matches from left (not in SQLite)
FULL JOIN   - All rows from both tables (not in SQLite)

Syntax

SELECT columns
FROM table1
JOIN table2 ON table1.column = table2.column;

Examples

-- Get album titles with artist names
SELECT Album.Title, Artist.Name
FROM Album
JOIN Artist ON Album.ArtistId = Artist.ArtistId;

-- Get track names with their genre
SELECT Track.Name, Genre.Name AS GenreName
FROM Track
JOIN Genre ON Track.GenreId = Genre.GenreId;

-- Multiple joins: track with album and artist
SELECT Track.Name AS Track, Album.Title AS Album, Artist.Name AS Artist
FROM Track
JOIN Album ON Track.AlbumId = Album.AlbumId
JOIN Artist ON Album.ArtistId = Artist.ArtistId;

-- LEFT JOIN: all customers, even without invoices
SELECT Customer.FirstName, Invoice.Total
FROM Customer
LEFT JOIN Invoice ON Customer.CustomerId = Invoice.CustomerId;

Key intuitions

  • Think of JOIN as matching IDs between tables
  • INNER JOIN is the default - just writing JOIN means INNER JOIN
  • LEFT JOIN keeps all left rows, fills NULLs where no match
  • Use table aliases to shorten: FROM Album a JOIN Artist ar ON a.ArtistId = ar.ArtistId
  • Always specify which table a column comes from when names could be ambiguous
