🏁 Dashboard Project

📋 The brief

Your task

The head of sales has ten minutes before a board meeting. Build them one screen that answers: how are we doing, where is the growth, and what needs attention? No scrolling, no explanation from you — the dashboard has to speak for itself.

That constraint is the whole exercise. A dashboard isn't "all the charts I can make from this data" — it's a deliberate selection, and everything you leave off is as much a decision as what you keep.

The four questions it must answer

1. How are we doing overall?

Headline numbers: revenue, profit, margin, order count — each with a comparison so the number means something.

2. Which parts of the business perform?

Breakdown by region and category. Where does the money come from, and where does it actually stay?

3. Which way are we heading?

The trend over time, at a granularity that shows the shape without noise.

4. What needs attention?

The exceptions — loss-making sub-categories, the discount/profit relationship. This is where the value is.

The rule that makes dashboards good

Every element must earn its place by changing a decision. Before adding a chart, ask "what would someone do differently after seeing this?" If the honest answer is "nothing", leave it out. Most weak dashboards fail by addition, not omission.

Why bother with the separation?

When someone asks "can we see this by quarter instead of month?", you change one summary block on Calc and the chart follows. In a workbook where charts read directly from raw data, that same request means rebuilding the chart. The layers are what make the thing changeable — and dashboards always get change requests.

1️⃣ Prepare the data

  1. Import and duplicate

    Load the Superstore CSV into Raw (see Setup). Duplicate the sheet and rename the copy Data. You now have a safety net.

  2. Run the quality checks

    Apply the data quality report: row count, blanks, untrimmed text, non-numeric values, dates that parse. Fix anything it flags before going further — a dashboard built on unclean data is worse than no dashboard, because it looks authoritative.

  3. Convert to a Table

    Ctrl T, then name it Orders in Table Design. Everything downstream will reference it by name, so it survives new rows being added.

  4. Add the helper columns

    Six columns that every later formula will lean on. Type each once in the first data row — a Table fills the rest automatically.

    // Order Year
=YEAR([@[Order Date]])

// Month Start — a real date, so it sorts and charts correctly
=DATE(YEAR([@[Order Date]]), MONTH([@[Order Date]]), 1)

// Profit Margin, guarded against a zero sale
=IFERROR([@Profit] / [@Sales], 0)

// Days to Ship
=[@[Ship Date]] - [@[Order Date]]

// Discounted? — for the exception analysis
=IF([@Discount] > 0, "Discounted", "Full Price")

// Loss flag
=IF([@Profit] < 0, 1, 0)
  5. Format the Month Start column

    Ctrl 1 → Custom → mmm yyyy. It stays a real date but reads as "Nov 2016".

Block A — the KPI formulas

// Current-year totals (put the year in a cell, say Calc!$B$1)
=SUMIFS(Orders[Sales], Orders[Order Year], $B$1)
=SUMIFS(Orders[Profit], Orders[Order Year], $B$1)

// Prior year — same formula, one year back
=SUMIFS(Orders[Sales], Orders[Order Year], $B$1 - 1)

// Growth vs last year
=IFERROR(B3/B4 - 1, "—")

// Overall margin
=IFERROR(B5/B3, "—")

// Distinct orders — COUNTA counts line items, not orders
=COUNTA(UNIQUE(FILTER(Orders[Order ID], Orders[Order Year]=$B$1)))
The order-count trap

Each Superstore row is one line item, so counting rows overstates orders by roughly 5×. Count distinct Order IDs instead. If you're on an older Excel without UNIQUE, a pivot table's "Distinct Count" option does the same job. Getting this wrong is the most common error in this project.

Title bar → KPI row → performance charts → exception charts. Four zones, one screen.

  1. Turn off the gridlines

    View → uncheck Gridlines. Nothing makes a worksheet look more like a finished product in less time.

  2. Build a grid with row heights and column widths

    Set every column on the Dashboard sheet to the same narrow width (say 3), then use merged blocks of those columns as your layout grid. It gives you far finer alignment control than working with default column widths.

  3. Set the background

    Fill the whole sheet with a very light grey (#F6F8FB) and make each card white. The subtle contrast is what separates the cards without needing borders everywhere.

  4. Check it fits

    Zoom to 100% and confirm the entire dashboard is visible on a typical laptop screen — roughly 1440×900. If it doesn't fit, cut something. Don't shrink everything.

4️⃣ The KPI row

Four cards across the top. Each needs three things: the number, what it is, and how it compares. A number with no comparison is trivia.

The anatomy of a good KPI card

  • Value — large, bold, rounded sensibly ($733K, not $733,215.42)
  • Label — small, grey, above or below
  • Comparison — vs last year, with direction and colour
  • Optional sparkline — the 12-month shape in one cell

Formatting the comparison

// Custom number format for the change cell:
▲ 0.0%;▼ 0.0%;—

// Positive shows ▲, negative ▼, zero shows a dash.
// Add conditional formatting for green/red.
Margin changes in points, not percent

Margin going from 13.4% to 12.7% is a fall of 0.7 percentage points, not 0.7%. Writing "▼ 5.1%" (the relative change) is also defensible but must be labelled. Mixing the two is a classic way to mislead a board without meaning to.

Sparklines belong here

Select the cell under a KPI → Insert → Sparklines → Line → point it at the 12 monthly values on Calc. In Google Sheets: =SPARKLINE(Calc!B2:B13). It adds trend context in the space of one cell, which on a dashboard is the best trade there is.

5️⃣ The four charts

Each one is a deliberate choice — see the Chart Decision Tree for the reasoning.

📈 Monthly trend — combo chart

Why: the goal is change over time, so time goes on the x-axis. Sales as columns, Profit as a line on a secondary axis — two metrics on very different scales.

Watch for: label both axes with units, or nobody can tell which scale belongs to which series.

📊 Region & Category — stacked bar

Why: shows regional totals and their category mix at once. Horizontal, sorted by total, largest at the top.

Watch for: only the bottom segment has a common baseline, so keep it to three categories.

🔻 Loss-makers — sorted bar

Why: comparison across categories, and the negative values are the point. Sort ascending so the biggest losses appear first.

Watch for: colour the negative bars red and everything else a single neutral colour. This is the chart that drives action.

⚬ Discount vs Margin — scatter

Why: the question is whether two numeric variables move together. Discount on x, profit margin on y, a trendline through it.

Watch for: plot the aggregated discount bands rather than 10,000 raw points — the pattern is clearer and the file stays fast.

Write titles that state the finding

"Sales by Region" describes the axes. "West leads on revenue but Central converts it best" tells the reader what you concluded. The second one does work; the first one just labels. If you can't write a finding-title, you may not have a finding — which is itself worth knowing.

Chart formatting checklist

  • Delete the chart border and any background fill
  • Fade gridlines to a light grey, or remove them entirely
  • Remove the legend when there's only one series; use direct labels where you can
  • Use one colour for context and one accent colour for the point you're making
  • Bar and column charts start at zero — always
  • Format axis numbers as $0K or $0.0M, not raw digits
  • Match the fonts and colours across all four charts

6️⃣ One control, not five

Interactivity is tempting and easy to overdo. Add one filter — a Region selector — and make the entire dashboard respond to it.

Option A — a dropdown cell

Data validation list on a cell, containing "All" plus each region. Then make the summary formulas respect it:

// "All" means no region filter at all
=IF($B$1="All",
   SUMIFS(Orders[Sales], Orders[Order Year], $B$2),
   SUMIFS(Orders[Sales], Orders[Order Year], $B$2,
                            Orders[Region], $B$1))

Works everywhere, including Google Sheets. Requires the wrapper on each formula.

Option B — a slicer

If your Calc blocks are pivot tables: click one → Insert → Slicer → Region. Then use Report Connections to attach that one slicer to every pivot on the sheet.

Far nicer to use, and no formula changes — but Excel only, and every block must be a pivot on the same source.

Then test the edges

Select each region in turn and confirm every number and chart moves. Check that the four regional totals still add up to the "All" figure. And check a selection with sparse data — does anything show #DIV/0! or an empty chart? Wrap the ratios in IFERROR so it degrades to "—" instead of an error.

7️⃣ Polish and ship

  1. Hide the working sheets

    Right-click Raw and Calc → Hide. The person opening this should see a dashboard, not your workings. (Keep them hidden, not deleted — you'll need them next month.)

  2. Protect the layout

    Review → Protect Sheet on the Dashboard, leaving only the control cell unlocked. It stops a stray click from deleting a chart.

  3. Add a data-as-of stamp

    A small line in the corner: "Data through 30 Dec 2017 · 9,994 line items · refreshed [date]". It takes one cell and prevents the most common dashboard question there is.

  4. Set the opening view

    Put the cursor in A1 on the Dashboard sheet and save. The workbook opens exactly where you left it — a small detail that makes it feel finished.

  5. Do the five-second test

    Show it to someone who hasn't seen the data. Ask them what they take away. If it isn't roughly what you intended, the dashboard is the thing that needs fixing.

✅ Self-review

Go through these honestly before calling it done.

Correctness

  • Regional totals sum to the grand total
  • Orders are counted distinctly, not as line items
  • Every ratio is wrapped in IFERROR
  • Percentage changes use the right base year
  • No hard-typed numbers on the Dashboard sheet
  • Grand total reconciles to =SUM(Orders[Sales])

Communication

  • It fits on one screen with no scrolling
  • Every chart title states a finding
  • Colour is used for meaning, not decoration
  • Bar charts start at zero
  • Axis units are labelled
  • Someone unfamiliar with it gets the point in five seconds
The question that separates a report from an analysis

When you're finished, write three sentences underneath: what you found, why it's happening, and what you'd do about it. If the dashboard doesn't support all three, it's a data display rather than an analysis. Doing this well is most of what the job actually is.

🚀 Going further

Make it refreshable

Rebuild the Data layer in Power Query instead of by hand. Next quarter's file drops in and one Refresh updates the whole dashboard.

Add a cohort view

Group customers by the year of their first order and track their spend since. It answers "are we growing by acquiring, or by keeping?" — a much better question than total revenue.

Quantify the discount finding

You'll see margin collapse above about 20% discount. Work out what profit would have been if those orders had been capped at 20% — that number is a recommendation with a price tag attached.

Rebuild it elsewhere

Make the same dashboard in Tableau, Power BI or Streamlit. Building one thing four ways teaches you more about each tool than four separate tutorials.
