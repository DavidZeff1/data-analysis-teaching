✂️ Text Functions

Real-world data is messy. Text functions let you extract, clean, combine, and reformat strings so your data is ready for analysis.

⏱️ 25 min 🧹 Data cleaning 🔗 Needs: Formulas
On this page Why they matter Extracting Finding Dynamic extraction Combining Cleaning Replacing Formatting Practice

💡 Why they matter

  • Extract information — state codes from order IDs, first names from full names
  • Clean data — remove extra spaces, fix capitalization
  • Combine fields — build full addresses and labels
  • Transform data — split, join, replace, and reformat text

📐 Extracting parts of text

LEFT — characters from the beginning
=LEFT(B2, 2)   // "CA-2016-152156" → "CA"
=LEFT(B2, 7)   // → "CA-2016"
RIGHT — characters from the end
=RIGHT(B2, 6)  // "CA-2016-152156" → "152156"
=RIGHT(G2, 4)  // "Claire Gute" → "Gute"
MID — characters from the middle
=MID(B2, 4, 4)  // → "2016" (start at 4, take 4)
=MID(B2, 1, 2)  // same as LEFT(B2, 2) → "CA"

🎯 Finding text position

=FIND("-", B2)     // position of first dash → 3
=FIND(" ", G2)     // position of space → 7
=SEARCH("ca", B2)  // case-insensitive → 1

// FIND is case-sensitive, SEARCH is not
=FIND("ca", "CA-2016")    // #VALUE! error
=SEARCH("ca", "CA-2016")  // 1

⚙️ Dynamic extraction (FIND + LEFT/RIGHT/MID)

Combine FIND with the extractors to pull variable-length text:

// First name (everything before the space)
=LEFT(G2, FIND(" ", G2) - 1)
// "Claire Gute" → FIND=7 → LEFT(G2, 6) → "Claire"

// Last name (everything after the space)
=RIGHT(G2, LEN(G2) - FIND(" ", G2))
// LEN=11, FIND=7 → RIGHT(G2, 4) → "Gute"

// Year from Order ID "CA-2016-152156"
=MID(B2, FIND("-", B2) + 1, 4)  // "2016"

🔗 Combining text

// CONCAT function
=CONCAT(J2, ", ", K2)   // "Henderson, Kentucky"

// Ampersand (&) — most common
=J2 & ", " & K2           // "Henderson, Kentucky"
=G2 & " (" & H2 & ")"    // "Claire Gute (Consumer)"

// TEXTJOIN — join with a delimiter; TRUE skips blanks
=TEXTJOIN(", ", TRUE, J2, K2, M2)
// "Henderson, Kentucky, South"

// Numbers auto-convert when joined to text
="Order #" & B2 & " - $" & R2

🔠 Changing case

=UPPER(G2)   // "Claire Gute" → "CLAIRE GUTE"
=LOWER(G2)   // → "claire gute"
=PROPER(G2)  // "CLAIRE GUTE" → "Claire Gute"

🧹 Cleaning data

// Remove leading/trailing/extra spaces
=TRIM("  Claire   Gute  ")  // → "Claire Gute"

// Remove non-printable characters
=CLEAN(A2)

// Length of text
=LEN(G2)         // "Claire Gute" → 11
=LEN(TRIM(G2))   // length after cleaning

Invisible spaces are the #1 cause of lookups failing to match. TRIM before comparing.

🔁 Replacing & substituting

// SUBSTITUTE — replace text (all occurrences)
=SUBSTITUTE(I2, "United States", "USA")
=SUBSTITUTE(B2, "-", "/")    // → "CA/2016/152156"

// Replace only the first occurrence
=SUBSTITUTE(B2, "-", "/", 1)  // → "CA/2016-152156"

// REPLACE — replace by position
=REPLACE(B2, 1, 2, "NY")   // → "NY-2016-152156"

💲 Formatting numbers as text

=TEXT(R2, "$#,##0.00")    // 1234.5 → "$1,234.50"
=TEXT(R2, "0%")           // 0.15 → "15%"
=TEXT(C2, "MMMM D, YYYY") // → "January 15, 2016"
=TEXT(R2, "000000")       // 42 → "000042"

// Combine a formatted number with text
="Total: " & TEXT(R2, "$#,##0.00")

⭐ Repeating text

=REPT("*", 5)        // "*****"
=REPT("★", U2/100)    // visual rating from profit
=REPT("-", 20)       // a separator line

🧠 Key intuitions

  • Position starts at 1, not 0 (unlike most programming)
  • FIND is case-sensitive, SEARCH is not
  • & is the fastest concatenation
  • TRIM is essential — invisible spaces break matching
  • LEN verifies data — check codes are the right length
  • TEXT is for display — numbers → formatted strings
  • SUBSTITUTE replaces text; REPLACE replaces by position
