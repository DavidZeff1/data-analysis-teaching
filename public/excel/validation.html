🧼 Cleaning & Validation

Dirty data produces confident, wrong answers. Analysts spend most of their time here — not because cleaning is glamorous, but because every number downstream depends on it. This is the systematic version of that work.

⏱️ 40 min ⭐ Core skill 🔗 Needs: Logical, Text, Tables
On this page Why it matters The workflow 1. Profile 2. Duplicates 3. Missing values 4. Text cleanup 5. Types & ranges 6. Errors 7. Prevention Quality report Practice

💡 Why this comes before analysis

A single trailing space in a Region value doesn't announce itself. It just quietly excludes rows from your SUMIFS, and your regional totals come out slightly wrong — plausible enough that nobody notices.

The six problems you'll actually meet

  • Duplicates — the same record entered twice
  • Inconsistent values — "USA", "U.S.A.", "United States"
  • Missing values — blanks that should hold data
  • Wrong types — numbers and dates stored as text
  • Invalid entries — negative quantities, future order dates
  • Whitespace — invisible, and it breaks every lookup

⚠️ How bad data hides

Bad data rarely throws an error. It returns a number that looks reasonable. That's exactly why you check before you analyse rather than trusting the output to complain.

The habit: every total you produce should be reconciled against a number you already trust.

🛑 Before you change anything: make a backup

Right-click the sheet tab → Duplicate, and rename it Raw_Original. Cleaning is destructive and Remove Duplicates cannot be undone once the file is saved and closed. Keeping the untouched original also lets you prove what you changed.

Clean in helper columns, not in place

Put =TRIM(G2) in a new column rather than overwriting column G. You can compare old against new, and only when you're satisfied do you paste-special-as-values over the original. Overwriting first and checking later is how good data gets destroyed.

Five formulas that tell you, in seconds, whether a column needs work.

// Count rows (COUNTA counts anything non-empty)
=COUNTA(G:G) - 1                    // minus the header

// Blanks
=COUNTBLANK(G2:G9995)

// Distinct values — how many customers, regions, products?
=COUNTA(UNIQUE(G2:G9995))          // modern Excel / Sheets
=SUMPRODUCT(1/COUNTIF(G2:G9995,G2:G9995))  // older Excel

// Range of a numeric column — do the extremes look sane?
=MIN(R2:R9995)  &  " to "  &  MAX(R2:R9995)

// Earliest and latest dates
=MIN(C2:C9995)  // format as a date, or you'll see 41,000-ish
The two-minute eyeball test

Open the filter dropdown on every text column and read the distinct values. Sort each numeric column both ways and look at the extremes. You will find most problems this way before writing a single formula.

👯 Step 2 — Duplicates

First, decide what "duplicate" means here

In the Superstore data, the same Order ID appears on many rows — one per product in that order. Those are not duplicates; that's the grain of the data. A true duplicate is a fully repeated row. Getting this wrong and deleting "duplicates" destroys real records.

Four ways to find them

1. See them — conditional formatting

Excel: Home → Conditional Formatting → Highlight Cell Rules → Duplicate Values.

Sheets: Format → Conditional formatting → Custom formula → =COUNTIF(A:A, A1)>1

2. Count them — COUNTIF
=COUNTIF($A$2:$A$9995, A2)
// how many times this value appears

=IF(COUNTIF($A$2:$A$9995,A2)>1,
   "Duplicate", "Unique")
3. Multi-column — a composite key
// "Duplicate" = same order AND same product
=COUNTIFS($B$2:$B$9995, B2,
          $N$2:$N$9995, N2)

// Or build one key and count that
=B2 & "|" & N2 & "|" & R2
4. Remove them — the built-in tool

Excel: Data → Remove Duplicates, then tick the columns that define a duplicate.

Sheets: Data → Data cleanup → Remove duplicates.

Note the row count before and after — that difference is your audit trail.

Remove Duplicates keeps the first occurrence

It does not ask which copy you want, and it does not tell you what it deleted. If the records differ in some column you didn't tick — a later timestamp, a corrected value — you may silently keep the wrong one. Flag with COUNTIFS and inspect before you delete.

🕳️ Step 3 — Missing values

Finding blanks is easy. Deciding what they mean is the actual work.

Finding them

=COUNTBLANK(G2:G9995)     // how many
=ISBLANK(G2)                // is this one empty?

// Which rows are incomplete?
=IF(COUNTBLANK(A2:U2)>0, "Incomplete", "OK")

// Blanks per column, for a quality report
=COUNTBLANK(G2:G9995) / ROWS(G2:G9995)

Select the range and press Ctrl G → Special → Blanks to jump straight to every empty cell.

Filling them

=IF(ISBLANK(G2), "Unknown", G2)   // a label
=IF(ISBLANK(R2), 0, R2)           // a zero
=IF(ISBLANK(G2), G1, G2)          // carry the value down
=IF(ISBLANK(R2), MEDIAN($R$2:$R$9995), R2)
⚠️ Blank is not zero

An empty Sales cell means "we don't know". Filling it with 0 means "this sold nothing" — a different claim entirely, and one that drags your averages down. AVERAGE skips blanks but counts zeros. Only substitute 0 when zero is genuinely the right answer.

Choosing what to do

OptionUse whenRisk
InvestigateAlways try this firstNone — blanks often point at a broken export

✂️ Step 4 — Text cleanup

The most common and most invisible category of problem. Full detail lives in Text Functions; this is the cleaning-specific subset.

// Detect the problem before you fix it
=SUMPRODUCT(--(LEN(M2:M9995) <> LEN(TRIM(M2:M9995))))

// The standard cleanup for anything imported
=TRIM(CLEAN(M2))     // spaces + non-printable characters

// Non-breaking spaces (CHAR 160) survive TRIM — very common
// in data copied from web pages. Kill them explicitly:
=TRIM(SUBSTITUTE(M2, CHAR(160), " "))

Inconsistent values

The filter dropdown is your detector: if a Region column shows five entries and you expected four, one of them is a variant.

// Standardise case so grouping works
=PROPER(TRIM(K2))          // "new york" → "New York"
=UPPER(TRIM(M2))           // codes and abbreviations

// Fix known variants
=SUBSTITUTE(I2, "U.S.A.", "USA")

// Chain them when there are several
=SUBSTITUTE(SUBSTITUTE(J2, "St.", "Street"), "Ave.", "Avenue")
For more than three variants, use a mapping table

Chained SUBSTITUTEs become unreadable fast. Instead, build a two-column lookup — messy value on the left, clean value on the right — and use =IFERROR(XLOOKUP(M2, Map[Raw], Map[Clean]), M2). It's readable, auditable, and anyone can add a row to it. Power Query's Replace Values step does the same job with no formula at all.

🔢 Step 5 — Types and ranges

Numbers stored as text

These look identical on screen but sum to zero. The telltale signs: left alignment, and a little green triangle in the corner.

=ISNUMBER(R2)                // FALSE means it's text
=SUMPRODUCT(--NOT(ISNUMBER(R2:R9995)))   // how many are broken

// Convert
=VALUE(R2)                   // "123" → 123
=R2 * 1                       // the quick version

// When there's a stray currency symbol or thousands separator
=VALUE(SUBSTITUTE(SUBSTITUTE(R2, "$", ""), ",", ""))
The one-click fix

Select the column → Data → Text to Columns → Finish. Excel re-parses every cell and converts the whole column in place. It's the fastest fix for both text-numbers and text-dates.

Range and plausibility checks

Type checks tell you the value is a number. Range checks tell you it's a believable number.

// Discount should be between 0 and 1
=IF(AND(T2>=0, T2<=1), "OK", "Out of range")

// Quantity should never be negative or zero
=IF(S2<=0, "Invalid quantity", "OK")

// Ship Date must not precede Order Date
=IF(D2 < C2, "Ships before ordered!", "OK")

// No date should be in the future
=IF(C2 > TODAY(), "Future date", "OK")

// Statistical outliers — more than 3 standard deviations out
=IF(ABS(R2 - AVERAGE($R$2:$R$9995)) > 3*STDEV($R$2:$R$9995),
   "Outlier?", "Normal")
An outlier is not automatically an error

The largest sale in the Superstore data is a genuine $22,638 order. Flagging outliers means "look at this", not "delete this". Deleting inconvenient extremes is how you end up with a tidy dataset that describes a business you don't have.

🚨 Step 6 — Errors: reading and handling

What each one means

  • #N/A — a lookup found nothing
  • #VALUE! — wrong type, or mismatched range sizes
  • #REF! — a referenced cell was deleted
  • #DIV/0! — divided by zero or a blank
  • #NAME? — misspelled function, or your version lacks it
  • #NUM! — impossible number, e.g. the root of a negative
  • #SPILL! — a dynamic array is blocked

Finding and handling them

=ISERROR(A2)        // any error
=ISNA(A2)           // specifically #N/A

// Count errors in a range
=SUMPRODUCT(--ISERROR(V2:V9995))

// Replace with something readable
=IFERROR(R2/S2, "—")
=IFNA(VLOOKUP(...), "Not found")
IFERROR hides the symptom, not the cause

Wrapping everything in IFERROR makes a sheet look clean while burying real problems — a #REF! from a deleted column becomes an innocent-looking blank. Diagnose first, suppress second. Prefer IFNA where you only expect lookup misses, so genuine errors still surface.

🛡️ Step 7 — Prevention: data validation rules

Cleaning fixes the past. Validation rules stop the same mess from arriving again — essential for any sheet other people type into.

Excel: Data → Data Validation · Google Sheets: Data → Data validation

The rule types

  • List a dropdown of allowed values
  • Number between, greater than, whole number only
  • Date restricted to a valid window
  • Text length exact character counts, e.g. ZIP codes
  • Custom any formula returning TRUE/FALSE

Custom rule examples

// Positive numbers only
=A2 > 0

// No duplicates allowed in this column
=COUNTIF($A:$A, A2) = 1

// Must look like an email
=ISNUMBER(FIND("@", A2))

// No leading or trailing spaces
=EXACT(A2, TRIM(A2))

// Ship date can't precede order date
=D2 >= C2
  1. Put your allowed values somewhere real

    List the four Ship Modes in cells on a Lists sheet rather than typing them into the rule box. Now they're visible, editable, and reusable.

  2. Point the rule at that range

    Choose List and set the source to =Lists!$A$2:$A$5. A dropdown appears in every cell you applied it to.

  3. Write a helpful error message

    On the Error Alert tab, replace the default with something actionable: "Ship Mode must be one of the four options in the dropdown." A rejection with no explanation just gets worked around.

  4. Know the limit

    Validation only checks values typed into cells. Pasting bypasses it entirely, and it never touches data that's already there. Use Data → Data Validation → Circle Invalid Data to find existing violations.

Document what you changed

Keep a Changelog sheet: date, column, what you did, how many rows it affected, and why. When someone asks in three months why the totals differ from the source system, that sheet is the answer. It's also what separates an analysis someone can trust from one they simply have to believe.

🧠 Key intuitions

  • Back up before you touch anything — Remove Duplicates is not undoable
  • Clean in helper columns, then paste values over the original once verified
  • Profile before you fix — you can't clean what you haven't looked at
  • TRIM everything imported — whitespace is the most common invisible bug
  • Blank ≠ zero — one is unknown, the other is a measurement
  • An outlier is a question, not a defect
  • Reconcile every total against something you already trust
  • Prevention beats cleaning — validation rules stop the next round
  • Write down what you changed — undocumented cleaning is unrepeatable
