🎚️ SUMIFS & COUNTIFS

Conditional aggregation is the single most-used analyst skill in spreadsheets: total the Sales, but only for the West region, only in 2017, only where the discount was zero. Master this and you can answer most business questions without a pivot table.

💡 The idea: filter, then aggregate

SUM adds everything. SUMIF adds only the rows that pass a test. Everything on this page is a variation of that one move:

All rows
9,994 line items

2

The mental model

Read =SUMIFS(R:R, M:M, "West") as: "Add up column R, for every row where column M says West." Every function here reads the same way — only the verb changes.

➕ SUMIF — one condition

SUMIF Add values that meet a single condition
=SUMIF( range, criteria, [sum_range] )
  • range The column to test — where the condition is checked
  • criteria What to look for: "West", ">500", a cell reference
  • sum_range The column to add. Leave it out and Excel adds range itself
// Total Sales in the West region
=SUMIF(M:M, "West", R:R)      // $725,458

// Total Sales over $500 — note: no sum_range needed,
// because we test and add the SAME column
=SUMIF(R:R, ">500")

// Total Profit for the Technology category
=SUMIF(O:O, "Technology", U:U)

// Criteria from a cell — the flexible version
=SUMIF(M:M, $X$1, R:R)         // type a region in X1

What it's doing, row by row

Excel walks down the range, and every time the criteria matches, it grabs the same row from sum_range.

🎯 SUMIFS — many conditions

SUMIFS (with the S) accepts up to 127 conditions. Note the argument order flips: the column you're adding comes first.

SUMIFS Add values that meet ALL of several conditions
=SUMIFS( sum_range, range1, criteria1, [range2, criteria2, …] )
  • sum_range The column to add — first, unlike SUMIF
  • range1 First column to test
  • criteria1 What must be true in range1
  • range2, criteria2 Add as many pairs as you need — all must pass
// Sales in the West, for Technology only
=SUMIFS(R:R, M:M, "West", O:O, "Technology")

// Sales in the West, Technology, over $500, no discount
=SUMIFS(R:R,
    M:M, "West",
    O:O, "Technology",
    R:R, ">500",
    T:T, 0)

// Profit for Consumer orders shipped Same Day
=SUMIFS(U:U, H:H, "Consumer", E:E, "Same Day")
Conditions are AND, never OR

Every criteria pair must be true for a row to count. To get OR behaviour ("West or East"), add two SUMIFS together: =SUMIFS(R:R,M:M,"West") + SUMIFS(R:R,M:M,"East").

⚠️ The #1 SUMIFS bug: ranges must be the same size

=SUMIFS(R2:R9995, M2:M5000, "West") returns #VALUE! — the sum range and criteria range have different heights. Use full columns (R:R, M:M) or an Excel Table to make this impossible.

✍️ Writing criteria correctly

Criteria is where almost every mistake happens. Here is every form you'll need.

GoalCriteriaNotes
Exactly equals text"West"Not case-sensitive; "west" works too
Exactly equals a number500No quotes needed for a plain number
Greater / less than">500", "<=0"Operator and value inside one pair of quotes
Not equal to"<>West"Everything except West
Compare to a cell">"&X1Operator in quotes, then &, then the reference
Starts with"Off*"* = any number of characters
Contains"*Chair*"Matches anywhere in the text
Single character"?A-2017*"? = exactly one character
Is blank / not blank"" / "<>"Useful for data-quality checks
A date">="&DATE(2017,1,1)Never type a date as text — see Dates & Time
❌ Wrong
=SUMIF(R:R, >500)

Operators must be inside quotes.

✓ Correct
=SUMIF(R:R, ">500")

The whole condition is one text string.

❌ Wrong
=SUMIF(R:R, ">X1")

Looks for the literal text ">X1".

✓ Correct
=SUMIF(R:R, ">"&X1)

& joins the operator to the cell's value.

👨‍👩‍👧‍👦 The whole family

Same criteria rules, different verb. Learn one, you know them all.

FunctionAnswersExample on Superstore
SUMIFSHow much in total?=SUMIFS(R:R, M:M, "West")
COUNTIFSHow many rows?=COUNTIFS(M:M, "West")
AVERAGEIFSWhat's the typical value?=AVERAGEIFS(R:R, M:M, "West")
MAXIFSWhat's the biggest?=MAXIFS(R:R, M:M, "West")
MINIFSWhat's the smallest?=MINIFS(U:U, M:M, "West")

Argument-order gotcha: the singular forms (SUMIF, COUNTIF, AVERAGEIF) put the criteria range first. The plural …IFS forms put the value range first — except COUNTIFS, which has no value range at all. When in doubt, use the …IFS version for everything; it handles one condition perfectly well.

COUNTIFS deserves a special mention

It's the fastest way to profile a dataset — how many rows are in each bucket?

// How many orders were discounted?
=COUNTIFS(T:T, ">0")

// How many loss-making orders in the West?
=COUNTIFS(M:M, "West", U:U, "<0")

// How many rows are missing a Customer Name? (data quality check)
=COUNTIFS(G:G, "")

// Percentage of orders that lost money
=COUNTIFS(U:U, "<0") / COUNTA(U:U)

📐 The real skill: a summary table you can copy

Typing "West" into a formula works once. The professional move is to put the labels in cells, then write one formula with mixed references that fills the whole grid. This is where absolute vs relative references pays off.

Y3fx=SUMIFS($R:$R, $M:$M, $X3, $O:$O, Y$2)

One formula, written once in Y3 and copied across and down the whole block.

Why the $ signs go exactly there

$X3 — column locked, row free

Copying right keeps pointing at column X (the region labels). Copying down moves to the next region.

Y$2 — row locked, column free

Copying down keeps pointing at row 2 (the category headers). Copying right moves to the next category.

$R:$R and $M:$M — fully locked

The source data never moves, so lock it completely.

✓ The result

A 4×3 cross-tab from a single formula — and it recalculates instantly when the data changes.

Build it in this order

Type the row labels down column X and the column headers across row 2 first. Then write the formula in the top-left cell of the block, check that one number is right, and only then copy it across and down.

⚖️ SUMIFS or a pivot table?

They answer the same questions. The difference is control versus speed.

How analysts actually work

Explore with a pivot table to find what matters, then rebuild the handful of numbers that matter as SUMIFS formulas in your report or dashboard — because those cells stay exactly where you put them and never need refreshing.

🕳️ Traps that waste hours

Trailing spaces

"West " in the data won't match "West". Symptom: a total of 0 that should be large. Fix: TRIM the column first — see Cleaning.

Mismatched range sizes

R2:R9995 against M2:M5000 gives #VALUE!. Use whole columns or a Table.

Numbers stored as text

">500" silently skips text values that look like numbers. Check with =ISNUMBER(R2).

Expecting OR

Extra criteria narrow the result, never widen it. Adding conditions can only make the number smaller.

Wildcards in numbers

* and ? only work on text. On numeric columns they match nothing.

Forgetting the S

SUMIF and SUMIFS take arguments in a different order. Mixing them up returns a plausible-looking wrong number.

Always sanity-check the total

Your region subtotals should add up to =SUM(R:R). If they don't, you have a value that matches none of your criteria — usually a typo or a stray space in the data.

🧠 Key intuitions

  • Read it as a sentence — "add this column, where that column says X"
  • Use the …IFS versions by default — same power, one consistent argument order
  • Criteria go in quotes when they contain an operator — ">500", not >500
  • Join operators to cells with & — ">"&X1
  • All conditions must pass — it's AND; add two formulas for OR
  • Ranges must be the same height — whole columns or Tables avoid the problem entirely
  • Mixed references build whole tables — $X3 and Y$2 from one formula
  • Zero usually means a text mismatch, not "no data"
