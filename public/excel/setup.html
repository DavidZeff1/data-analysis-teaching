Practice Setup

1. Open Google Sheets (Free)

Go to sheets.new to create a new blank spreadsheet instantly.

Google Sheets is completely free and works almost identically to Microsoft Excel. All the formulas you learn here will work in both applications. The main advantages of Google Sheets for learning:

  • Free - No subscription required
  • Cloud-based - Access from any device
  • Auto-save - Never lose your work
  • Easy sharing - Collaborate with others

2. Import the Sample Superstore Dataset

We'll use the famous "Sample Superstore" dataset - a fictional retail company's sales data. This dataset is widely used in data analysis courses because it contains realistic business data with multiple dimensions and measures.

In cell A1, paste this formula and press Enter:

=IMPORTDATA("https://raw.githubusercontent.com/leonism/sample-superstore/master/data/superstore.csv")

Wait 5-10 seconds for the data to load. You'll see ~10,000 rows appear.

3. Understanding the Dataset

The Superstore dataset simulates a retail company selling office supplies, furniture, and technology products across the United States. Each row represents a single item in an order.

ðŸ“‹ Dimension Columns (Categories)

  • Row ID - Unique identifier for each row
  • Order ID - Unique order number (one order can have multiple rows)
  • Order Date - When the order was placed
  • Ship Date - When the order was shipped
  • Ship Mode - Shipping speed (Standard, Second Class, First Class, Same Day)
  • Customer ID - Unique customer identifier
  • Customer Name - Customer's full name
  • Segment - Customer type (Consumer, Corporate, Home Office)
  • Country - Always "United States" in this dataset
  • City, State, Postal Code, Region - Location info
  • Product ID - Unique product identifier
  • Category - Product category (Furniture, Office Supplies, Technology)
  • Sub-Category - Detailed product type (Chairs, Phones, Paper, etc.)
  • Product Name - Full product description

ðŸ“Š Measure Columns (Numbers)

  • Sales - Revenue from the sale (in dollars)
  • Quantity - Number of items sold
  • Discount - Discount applied (0 = no discount, 0.2 = 20% off)
  • Profit - Profit or loss from the sale (can be negative!)

  • Which products are most profitable?
  • Which regions generate the most sales?
  • Do discounts increase or decrease profit?
  • Which customers are most valuable?
  • How do sales trend over time?

4. Verify Your Setup

After the data loads, confirm everything is correct:

  • Row 1 should contain headers (Row ID, Order ID, Order Date, etc.)
  • You should have approximately 9,994 rows of data (plus the header row)
  • Column A should be "Row ID" with numbers 1, 2, 3, etc.
  • Column N should be "Sales" with dollar amounts
  • Scroll right - you should see about 21 columns total
