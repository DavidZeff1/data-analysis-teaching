✨ Dynamic Arrays

One formula that returns many cells. Dynamic arrays replaced a generation of awkward workarounds — no more copying formulas down 10,000 rows, no more Ctrl+Shift+Enter. If you have Excel 365 or Google Sheets, this is the modern way to work.

💧 The big idea: formulas that spill

Traditionally, one formula filled one cell — so to get 500 results you wrote 500 formulas. A dynamic array formula returns a whole block of results from a single cell, and the results spill into the cells below and to the right.

The old way
X
2=UPPER(G2)
3=UPPER(G3)
4=UPPER(G4)
5…9,991 more

9,994 separate formulas to maintain.

The dynamic array way
X
2=UPPER(G2:G9995)
3CLAIRE GUTE
4DARRIN VAN HUFF
5…spills automatically

One formula. Edit it once, everything updates.

🔵 The spill range

Click any spilled cell and you'll see the formula greyed out — only the top-left cell is real. A blue border outlines the whole spill range.

# — the spill operator

Refer to a whole spill range with X2#. It automatically grows and shrinks with the results — no more guessing at range sizes.

🔎 FILTER — the one you'll use most

Returns only the rows that meet a condition. Think of it as a live, formula-driven version of the filter dropdowns.

FILTER Return the rows matching a condition
=FILTER( array, include, [if_empty] )
  • array The rows or columns to return
  • include A condition producing TRUE/FALSE for each row
  • if_empty What to show when nothing matches — always supply this
// Every West-region row, all columns
=FILTER(A2:U9995, M2:M9995="West", "No matches")

// Just the Customer Names of loss-making orders
=FILTER(G2:G9995, U2:U9995<0, "None")

// Two conditions — multiply for AND
=FILTER(A2:U9995, (M2:M9995="West") * (U2:U9995<0), "None")

// Two conditions — add for OR
=FILTER(A2:U9995, (M2:M9995="West") + (M2:M9995="East"), "None")

// Driven by a cell — type a region in X1 and the list rebuilds
=FILTER(A2:U9995, M2:M9995=$X$1, "No matches")
Remember: * is AND, + is OR

Conditions produce TRUE/FALSE, which behave as 1/0. Multiplying gives 1 only when both are true (AND); adding gives 1 or more when either is true (OR). Always wrap each condition in its own parentheses.

The include range must be exactly as tall as the array, or you get #VALUE!. Select both ranges over identical rows.

↕️ SORT and SORTBY

Sorting that updates itself. The original data stays untouched — you're producing a sorted view.

SORT Order an array by one of its own columns
=SORT( array, [sort_index], [order], [by_col] )
  • array What to sort
  • sort_index Which column of the array to sort on (1 = first)
  • order 1 ascending (default), -1 descending
// States and their sales, biggest first
=SORT(X2:Y50, 2, -1)

// Alphabetical list of customers
=SORT(G2:G9995)
SORTBY Order an array by a different range
=SORTBY( array, by_array1, [order1], [by_array2, order2, …] )
// Customer names, ordered by their profit (highest first)
=SORTBY(G2:G9995, U2:U9995, -1)

// Sort by Region A–Z, then by Sales high–low within each region
=SORTBY(A2:U9995, M2:M9995, 1, R2:R9995, -1)

SORT vs SORTBY: use SORT when the sort key is one of the columns you're returning, and SORTBY when it isn't — or when you need several sort levels.

🎯 UNIQUE — the distinct list

Every summary table starts with "what are the distinct values?" This used to take a pivot table or Remove Duplicates on a copy. Now it's one formula that stays live.

// The list of regions, with no repeats
=UNIQUE(M2:M9995)          // Central, East, South, West

// Sorted, which is almost always what you want
=SORT(UNIQUE(M2:M9995))

// Unique combinations across two columns
=UNIQUE(M2:N9995)          // each Region + Product ID pair

// How many distinct customers are there?
=COUNTA(UNIQUE(G2:G9995))

// Values that appear EXACTLY once (the third argument)
=UNIQUE(B2:B9995, FALSE, TRUE)   // Order IDs with a single line item
The pattern that replaces half of your pivot tables

Put =SORT(UNIQUE(M2:M9995)) in X2 to generate the row labels, then =SUMIFS($R:$R, $M:$M, X2#) in Y2 to total each one. Both spill, both resize automatically, and a new region appearing in the data adds a row to your report on its own.

🔢 SEQUENCE, RANDARRAY and the text splitters

// SEQUENCE(rows, [cols], [start], [step])
=SEQUENCE(12)            // 1..12 down a column
=SEQUENCE(1, 12)         // 1..12 across a row
=SEQUENCE(5, 1, 2014)   // 2014..2018

// Twelve month-start dates for 2017 — a whole
// calendar axis from one formula
=DATE(2017, SEQUENCE(12), 1)
// Random values, handy for building test data
=RANDARRAY(10, 1, 1, 100, TRUE)
//   10 rows, 1 col, 1–100, whole numbers

// Split "Claire Gute" into two cells (Excel 365)
=TEXTSPLIT(G2, " ")

// Google Sheets equivalent
=SPLIT(G2, " ")

TAKE, DROP, CHOOSECOLS and CHOOSEROWS (Excel 365, 2022+) slice arrays further — e.g. =TAKE(SORT(X2:Y50,2,-1), 10) gives a live top-10 list.

🧩 Combining them: a live top-10 report

The real power appears when you nest these. Each function takes an array and returns an array, so they stack cleanly.

// Read this from the inside out:
=TAKE( SORT( FILTER(X2:Y50, Y2:Y50>0, "none"), 2, -1 ), 10 )

//   1. FILTER  → keep only profitable states
//   2. SORT    → order by column 2, descending
//   3. TAKE    → keep the first 10 rows

Change the underlying data and this rebuilds itself. No refresh, no re-sorting, no re-copying formulas.

A self-updating filtered view

  1. Add an input cell

    Put a Region name in X1. Give it a dropdown with data validation so only valid values can be typed.

  2. Filter against it

    =FILTER(A2:U9995, M2:M9995=$X$1, "No matches") in X3. The whole matching dataset appears below.

  3. Summarise the spill range

    Point summary formulas at the results, not the source: =SUM(INDEX(X3#, 0, 18)) totals column 18 of whatever spilled.

  4. Change the input

    Pick a different region and the table, the totals and any chart built on them all update at once. That's a dashboard with no macros and no refresh button.

🚧 Errors you'll meet

#SPILL!

Something is in the way. The formula needs empty cells to expand into. Click the cell and Excel outlines the blocked area — clear it, or move the formula somewhere with room.

#CALC!

FILTER matched nothing and you didn't supply if_empty. Always add a third argument.

#VALUE!

Mismatched range heights — the array and the condition cover different numbers of rows.

#NAME?

Your version doesn't have this function. See the compatibility note below.

Don't put anything below a spilling formula

Leave the space beneath a dynamic array empty. If the data grows and the spill range needs more room, anything sitting there turns the result into #SPILL!. Put spilling formulas in their own area of the sheet, or on their own tab.

Quick test: type =SEQUENCE(3) in an empty cell. Three numbers means you're set; #NAME? means you're not.

Sharing files matters

A workbook using FILTER opened in Excel 2019 shows #NAME? everywhere. If colleagues are on older versions, stick to SUMIFS and INDEX/MATCH for anything you send them — both work everywhere and are covered in SUMIFS & COUNTIFS and Lookup.

🧠 Key intuitions

  • One formula, many results — write it once in the top-left cell
  • Only the first cell is real — everything else is spilled output
  • X2# means "the whole spill" — and it resizes itself
  • * is AND, + is OR in FILTER conditions
  • Always give FILTER an if_empty — otherwise no matches means #CALC!
  • They nest — FILTER → SORT → TAKE reads inside-out
  • Leave room to spill — #SPILL! just means "something's in the way"
  • SORT(UNIQUE(...)) + SUMIFS is a live pivot table you fully control
