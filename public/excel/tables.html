🗂️ Tables, Sorting & Filtering

Before you analyse 10,000 rows, you need to be able to move around them, narrow them down, and see what stands out. Excel Tables turn a raw grid into a named, self-expanding object — and they quietly fix a whole class of formula bugs.

⏱️ 25 min ⭐ Core skill 🔗 Needs: Setup, References

🧭 Moving around 10,000 rows

Scrolling is the slowest possible way to navigate a dataset. These four moves will save you more time than any formula on this site.

Ctrl+↓ Jump to the edge

Leaps to the last filled cell in that direction. Ctrl ↓ tells you instantly how many rows you have.

Ctrl+Shift+↓ Select to the edge

Same jump, but selecting everything on the way. This is how you select a whole column of data without touching the mouse.

Ctrl+Shift+* Select the whole block

Grabs the entire contiguous data region around the cursor, headers included.

Ctrl+Home Back to A1

Instant reset. On a Mac: Ctrl Fn ←.

The status bar is a free calculator

Select any range of numbers and look at the bottom-right of the window: count, sum and average appear instantly. Right-click the status bar to add min, max and numerical count. It's the fastest way to sanity-check a total without writing a formula.

❄️ Freeze panes — keep the headers visible

Scroll to row 5,000 in the Superstore data and you'll have no idea which column is which. Freezing pins the header row in place.

  1. Click the cell below and right of what you want frozen

    To freeze just the header row, click A2. To freeze the header row and the first column, click B2.

  2. Apply it

    Excel: View → Freeze Panes → Freeze Panes. Google Sheets: View → Freeze → 1 row.

  3. Check the line

    A thin grey line marks the frozen boundary. Scroll down — the headers should stay put.

Excel Tables (next section) do this automatically: scroll down inside a Table and the column letters A, B, C are replaced by your actual field names.

📦 Excel Tables — press Ctrl T

An "Excel Table" isn't just a range with borders. It's a named object that knows where its data ends — and that one property fixes several problems at once.

A plain range
Order IDRegionSales
CA-2016-152156South261.96
CA-2016-138688West14.62

Formulas point at R2:R9995. Add a row and every formula silently misses it.

→
A Table named Orders
Order ID ▾Region ▾Sales ▾
CA-2016-152156South261.96
CA-2016-138688West14.62

Formulas point at Orders[Sales]. Add a row and it's included automatically.

What you get for free

  • Auto-expanding ranges — new rows join the Table, so formulas, pivots and charts pick them up without editing
  • Filter buttons on every header, added automatically
  • Sticky headers — scroll down and column letters become field names
  • Formulas fill down on their own — type one in a new column and the whole column populates
  • Readable formulas — SUM(Orders[Sales]) instead of SUM(R2:R9995)
  • Banded rows that stay correct when you sort or filter
  • A Total row you can toggle on with Ctrl Shift T
  1. Click any cell in your data

    You don't need to select the whole range — Excel detects it.

  2. Press Ctrl T

    Confirm the detected range and make sure "My table has headers" is ticked. In Google Sheets, the equivalent is Format → Convert to table.

  3. Name it immediately

    Table Design → Table Name, top-left. Call it Orders, not Table1. This name appears in every formula you write from now on.

One Table needs one clean block

No blank rows or columns inside the data, exactly one header row, and no merged cells. If your sheet has a title row above the headers, delete it or move it to another sheet before pressing Ctrl T.

🏷️ Structured references

Once your data is a Table, you can refer to it by name. Formulas stop being cryptic and start reading like sentences.

Structured referenceMeans
Orders[Sales]Every data cell in the Sales column
Orders[[#Headers],[Sales]]Just the header cell
Orders[#All]Headers, data and totals together
Orders[@Sales]Sales on this row — used inside the Table
Orders[[Sales]:[Profit]]A span of adjacent columns
 
// Before: what does R even mean?
=SUMIFS(R2:R9995, M2:M9995, "West")

// After: it reads as English
=SUMIFS(Orders[Sales], Orders[Region], "West")

// A calculated column, typed once in the first row
=Orders[@Profit] / Orders[@Sales]

// Totals that grow with the data
=SUM(Orders[Sales])
=COUNTA(Orders[Order ID])
Why this matters more than it looks

Half the "my formula broke" problems in spreadsheets come from ranges that no longer cover the data — a new month is appended and R2:R9995 quietly excludes it. Structured references cannot have that bug, because the Table always knows its own extent. It also makes the SUMIFS range-size mismatch impossible.

🔠 Sorting

Single column

Click the filter arrow on a header → Sort A→Z or Z→A. On a numeric column, sorting descending is the fastest way to find the biggest values.

Multiple levels

Data → Sort lets you stack rules: Region A→Z, then Sales largest→smallest. The top rule wins; later rules break ties within it.

⚠️ The mistake that corrupts data

If you select a single column and sort it, only that column moves — every row's values get shuffled apart from each other, and the dataset is destroyed in a way that's hard to spot and impossible to undo after saving. Always select the whole data block first, or use a Table, where sorting always moves entire rows.

Sorting changes your data's order permanently. If you only want to look, filter instead — or use SORT(), which produces a sorted view and leaves the source untouched.

🔽 Filtering

Filtering hides rows without deleting anything. Press Ctrl Shift L to toggle filter arrows on any dataset.

☑️ By value

Tick and untick from the list. It shows every distinct value in the column — a fast way to spot inconsistent spellings.

🔢 By condition

Number Filters → Greater Than, Between, Top 10. Text Filters → Contains, Begins With.

🎨 By colour

If you've applied conditional formatting, you can filter to just the highlighted rows.

The filter list is a free data-quality audit

Open the filter on a text column and read the distinct values. "USA", "U.S.A." and "United States" sitting in the same list is a problem you'd otherwise discover much later, when your totals don't add up. See Cleaning & Validation for the fix.

Filtering changes what your formulas should count

This trips people up constantly: SUM ignores filtering entirely and totals the hidden rows too. Use SUBTOTAL when you want the visible rows only.

=SUM(R2:R9995)              // always the full total, filter or no filter
=SUBTOTAL(109, R2:R9995)    // sums only the VISIBLE rows
=SUBTOTAL(103, B2:B9995)    // counts only the visible rows

// The function numbers: add 100 to ignore hidden rows.
//   101 AVERAGE   102 COUNT   103 COUNTA
//   104 MAX       105 MIN     109 SUM

A Table's Total row uses SUBTOTAL automatically, so it always reflects what you're actually looking at. In Google Sheets, the same job is done by =SUBTOTAL(109, R2:R9995) too.

🎨 Conditional formatting — seeing patterns

Colour is the fastest channel your eyes have. Conditional formatting applies it by rule, so the highlighting updates whenever the data does.

🔴 Highlight cells

"Greater than", "Less than", "Text contains", "Duplicate values". The quickest win: highlight every negative Profit in red.

🌡️ Colour scales

A gradient across the range — low values one colour, high another. Turns a column of numbers into a heatmap.

📊 Data bars

An in-cell bar proportional to the value. A bar chart with no chart, right inside the table.

🔺 Icon sets

Arrows or traffic lights by threshold. Use sparingly — three states is usually enough.

Two rules: a colour scale on Sales, and red fill where Profit < 0. The loss-making sub-categories jump straight out.

Custom formula rules — where it gets powerful

Choose "Use a formula to determine which cells to format" and you can highlight based on any logic, including other columns.

// Highlight the ENTIRE row when that row lost money.
// Select A2:U9995 first, then use:
=$U2 < 0
//   $U locks the column so every cell in the row is judged
//   by column U; the free row number lets it walk down.

// Rows that were discounted AND lost money
=AND($T2 > 0, $U2 < 0)

// Duplicate Order IDs
=COUNTIF($B:$B, $B2) > 1

// Anything above the overall average
=$R2 > AVERAGE($R:$R)
The $ rule for whole-row highlighting

Write the formula as if it applies to the top-left cell of your selection. Lock the column with $ and leave the row free — $U2. Get this backwards and you'll get a diagonal stripe of colour instead of highlighted rows.

Keep it restrained

If everything is highlighted, nothing is. Two or three rules per sheet is plenty. Conditional formatting on tens of thousands of rows also slows a workbook noticeably — apply it to the range you're actually reading, not to whole columns.

🧠 Key intuitions

  • Ctrl + arrows is how experienced users move — never scroll to row 9,995
  • Press Ctrl T on every dataset, then name the Table
  • Structured references can't go stale — the Table always knows its own size
  • Sorting rearranges data; filtering only hides it
  • Never sort one column alone — select everything or use a Table
  • SUM ignores filters, SUBTOTAL respects them
  • The filter dropdown lists every distinct value — a free consistency check
  • Conditional formatting is a rule, not a paint stroke — it re-evaluates as data changes
