🔍 Lookup Functions

Look up a value in one place and pull back related information from another — the backbone of connecting datasets and building reports.

⏱️ 30 min ⭐ Core skill 🔗 Needs: References, Logical
On this page What they do VLOOKUP Limitations INDEX + MATCH XLOOKUP Which to use Errors Practice

💡 What they do

Lookup functions search for a value and return matching information. They're essential for:

  • Connecting data — match customer IDs to names, product codes to descriptions
  • Building reports — pull specific values into summary tables
  • Validation — check whether a value exists in a reference list

Think of a book's index: you look up a term (the search value) and find the page number (the return value).

⬇️ VLOOKUP — vertical lookup

The most common lookup. It searches down the first column of a range and returns a value from a chosen column.

=VLOOKUP( search, range, col_num, exact? )
search_value
What you're looking for
range
Where to search — search column must be FIRST
column_number
Which column to return (1 = first)
exact_match
FALSE (or 0) for exact — almost always use this
// Find a customer name by Customer ID
=VLOOKUP("CG-12520", F:G, 2, FALSE)
//   search in F (Customer ID), return from G (Customer Name)

// Find a product's category
=VLOOKUP(N2, N:O, 2, FALSE)

// Reference a cell for the search value
=VLOOKUP(A2, $F$2:$G$9995, 2, FALSE)

🚧 VLOOKUP limitations

⚠️ VLOOKUP can only look right

The search column must be the leftmost column in your range. If you need to search column G and return from column F, VLOOKUP won't work.

  • Search column must be first (leftmost)
  • Column numbers break if you insert/delete columns
  • Returns only the first match found
  • Can be slow on very large datasets

🧩 INDEX + MATCH — the flexible alternative

MATCH — finds a POSITION
=MATCH("CG-12520", F:F, 0)
// row number where it's found
INDEX — returns a VALUE
=INDEX(G:G, 5)
// value in row 5 of column G
// Combined: find customer name by ID
=INDEX(G:G, MATCH("CG-12520", F:F, 0))
// 1) MATCH finds the row, 2) INDEX returns that row's value

// Look LEFT (impossible with VLOOKUP): find ID by name
=INDEX(F:F, MATCH("Claire Gute", G:G, 0))

✨ XLOOKUP — the modern solution

Available in Google Sheets and Excel 365, XLOOKUP combines the best of both:

=XLOOKUP("CG-12520", F:F, G:G)
//   search_value, lookup_range, return_range

// Default value if not found
=XLOOKUP("Unknown", F:F, G:G, "Not Found")

// Look left easily
=XLOOKUP("Claire Gute", G:G, F:F)

// Return multiple columns at once (Name + Segment)
=XLOOKUP("CG-12520", F:F, G:H)

🛡️ Handling errors

Lookups return #N/A when nothing is found. Wrap them in IFERROR:

=IFERROR(VLOOKUP(A2, F:G, 2, FALSE), "Not Found")
=IFERROR(INDEX(G:G, MATCH(A2, F:F, 0)), "Not Found")

➡️ HLOOKUP — horizontal lookup

Like VLOOKUP, but searches across a row instead of down a column. Used when data is arranged horizontally.

=HLOOKUP("Sales", A1:D3, 2, FALSE)
// Search the first ROW for "Sales", return from row 2

🧠 Key intuitions

  • Always use FALSE (exact match) — TRUE gives unexpected results
  • VLOOKUP column numbers are fragile — they break when columns move
  • INDEX/MATCH is robust — references won't break
  • XLOOKUP is the future — simpler syntax, more features
  • #N/A means "not found" — wrap in IFERROR
  • Lookups return the first match — duplicates give you the first one
  • Use absolute references for lookup ranges when copying formulas
