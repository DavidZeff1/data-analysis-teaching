⚙️ Power Query

Everything you did by hand in the cleaning lesson, recorded once and replayed with one click. Power Query is Excel's built-in ETL tool: it connects to data, transforms it through a list of recorded steps, and loads the result — repeatably, on millions of rows.

💡 The problem it solves

You clean a monthly export by hand: delete columns, trim text, fix types, filter, split a column. It takes forty minutes. Next month the new file arrives and you do all forty minutes again.

By hand, every month

40 minutes of clicking · impossible to audit · one missed step changes the numbers · nobody else can reproduce it

→
With Power Query

Set up once · click Refresh · every step is listed and editable · anyone can read exactly what was done

🔁 Repeatable

Steps are recorded and re-run on new data.

📁 Combines files

Point it at a folder and merge fifty CSVs at once.

🔀 Reshapes

Pivot, unpivot, split, merge, join tables.

📦 Scales

Handles millions of rows that would choke a worksheet.

🔍 Auditable

The Applied Steps list is your documentation.

🛡️ Non-destructive

The source file is never modified.

Also called: "Get & Transform" in Excel 2016+, and "Power Query Editor" in Power BI — it's the same engine, so this lesson transfers directly.

🚪 Getting in

✓ Excel for Windows / Mac (desktop)
  1. Select any cell in your data
  2. Data → From Table/Range (or Get Data for external sources)
  3. Confirm the range and headers
  4. The Power Query Editor opens in a new window
✗ Excel Online / Google Sheets

Power Query is not available in either. You need desktop Excel.

Working in Sheets? The closest equivalents are QUERY(), IMPORTRANGE() and Apps Script — different tools, same goals. Read this page anyway: the concepts (recorded steps, group by, merge, unpivot) are how all ETL tools work, and they're identical in Power BI.

Convert your data to a Table with Ctrl T before loading it. Power Query then tracks it by name rather than by cell range, so nothing breaks when the data grows.

📋 Queries (left)

Every query is a separate pipeline. They can reference each other, so you can build modular transformations.

📊 Preview (centre)

The first 1,000 rows. The tiny icon on each header is the data type — the single most important thing to get right.

📝 Applied Steps (right)

Your recorded actions, in order. Click any step to see the data as it looked at that moment. Delete one to undo it.

Applied Steps is the whole idea

Every click you make appends a step. That list is simultaneously your undo history, your documentation, and the program that runs on next month's file. Rename steps to something meaningful (right-click → Rename) and a colleague can read your transformation like a recipe.

🔧 The transformations you'll use daily

Set data typesDo this first, always

Click the type icon in a column header and choose the right type. Text is ABC, whole number 123, decimal 1.2, date 📅.

Getting types right at the start prevents most later problems. A date left as text will not group by month; a number left as text will not sum. Power Query adds an automatic "Changed Type" step — check it rather than trusting it.

Remove columns

Right-click → Remove. Better: select the ones you want to keep → right-click → Remove Other Columns. That way a new column appearing in the source doesn't sneak in.

Filter rows

The ▾ on each header. Same as a worksheet filter, except it's permanent and re-applies on refresh. Text: contains, starts with. Numbers: greater than, between, top N.

Remove duplicates

Select the key columns → right-click → Remove Duplicates. Unlike the worksheet version, this is reversible: just delete the step.

Replace values

Right-click a column → Replace Values. Ideal for standardising variants and swapping null for a real default.

Split column

By delimiter, by character count, or by position. Splitting "CA-2016-152156" on - gives you state, year and order number in three clicks.

Merge columns

Ctrl-click several columns → right-click → Merge Columns, then pick a separator.

Trim & clean

Transform → Format → Trim / Clean / lowercase / Capitalize Each Word. The whole text cleanup lesson, as menu items.

Fill down

Transform → Fill → Down. Fills null cells with the value above — exactly what merged-cell exports need.

Order matters, and you can reorder

Filter out rows you don't need early — everything after that step runs on less data. If you get the order wrong, drag steps up and down in the Applied Steps list. Power Query re-runs the whole pipeline instantly.

➕ Adding columns

Conditional Column — no code

Add Column → Conditional Column. A dialog builds IF/THEN logic for you:

IfSalesis greater than1000
Then"High"
Else ifSalesis greater than100
Then"Medium"
Else"Low"

Custom Column — the M language

Add Column → Custom Column, for anything the dialogs can't express:

// Profit margin
[Profit] / [Sales]

// Concatenate with a separator
[City] & ", " & [State]

// If / then / else — note: all lowercase
if [Sales] > 1000 then "High"
else if [Sales] > 100 then "Medium"
else "Low"

// Guard against divide-by-zero
if [Sales] = 0 then null else [Profit]/[Sales]
M is case-sensitive — worksheet formulas are not

Text.Upper works; text.upper and TEXT.UPPER do not. Keywords like if, then, else and null are always lowercase. Column names go in [square brackets] and must match exactly — use the field list in the dialog to insert them rather than typing.

🔄 Pivot and Unpivot

Unpivot is the one that will save you. Reports arrive wide — one column per month — and nothing analytical works on that shape. Unpivot turns it back into tidy rows.

Wide — how humans write reports
ProductJanFebMar
Chairs102015
Tables5812

A new month means a new column — and every formula and chart breaks.

→
Long — how analysis needs it
ProductMonthValue
ChairsJan10
ChairsFeb20
ChairsMar15
TablesJan5

A new month just adds rows. Pivots and charts keep working.

To unpivot

Select the columns that should stay (Product) → right-click → Unpivot Other Columns. Choosing it this way round means future months are handled automatically.

To pivot

Select the column holding the future headers (Month) → Transform → Pivot Column → choose the values column and an aggregation. Useful for a final presentation layout.

The rule

Unpivot to analyse, pivot to present. Keep your working data long and tidy; reshape it to wide only at the very end, if at all — a pivot table will usually do that for you.

📁 Combine every file in a folder

Power Query's standout feature. Twelve monthly CSVs with the same columns become one table, and next month's file joins automatically.

  1. Put the files in one folder

    Same structure, same column names. Nothing else in the folder.

  2. Data → Get Data → From File → From Folder

    Select the folder. You'll see a list of the files it found, not their contents yet.

  3. Combine & Transform Data

    Pick a sample file so Power Query can learn the structure, then confirm. It builds a helper query and applies it to every file.

  4. Use the Source.Name column

    Every row is tagged with the file it came from — often the only place the month or region is recorded. Split it to extract that.

  5. Drop a new file in and hit Refresh

    That's the entire monthly process from now on.

If one file has a renamed or missing column, that file's rows come through as null. When results look wrong, group by Source.Name and count rows per file — the odd one out is usually obvious.

📤 Loading the result back

Home → Close & Load To…

  • Table — a normal Excel Table on a worksheet
  • PivotTable Report — straight into a pivot
  • Only Create Connection — nothing lands on a sheet

Use Connection Only for intermediate queries that only exist to feed another query — it keeps your workbook small and your sheets uncluttered.

Refreshing

  • Right-click the output table → Refresh
  • Data → Refresh All (Ctrl Alt F5)
  • Query properties → refresh on file open

Every recorded step re-runs against the current source data.

Don't edit the output table by hand

Anything you type into a query's output is wiped on the next refresh. If a value needs changing, change it as a step in the query — that's the whole point. Build your own formulas in columns outside the loaded table, or in a separate sheet that references it.

📖 M function reference

You rarely need to write M by hand, but reading it makes the Applied Steps far less mysterious.

// Text
Text.Upper([Category])
Text.Lower([Category])
Text.Proper([City])
Text.Trim([Region])
Text.Clean([Notes])
Text.Start([Product ID], 3)      // first 3 chars
Text.End([Product ID], 8)        // last 8 chars
Text.Middle([Order ID], 3, 4)     // from position 3
Text.Contains([Sub-Category], "Chair")
Text.Replace([Country], "USA", "US")
Text.Length([Product Name])
// Dates
Date.Year([Order Date])
Date.Month([Order Date])
Date.Day([Order Date])
Date.MonthName([Order Date])     // "November"
Date.DayOfWeekName([Order Date])
Date.QuarterOfYear([Order Date])
Date.StartOfMonth([Order Date])
Duration.Days([Ship Date] - [Order Date])

// Numbers & nulls
Number.Round([Profit], 2)
Number.Abs([Profit])
if [Region] = null then "Unknown" else [Region]

View → Advanced Editor shows the full M script for a query. It's plain text, so you can copy a whole transformation between workbooks — much faster than rebuilding it by clicking.

The professional pattern

Power Query prepares the data; formulas and pivot tables analyse it. Keep all cleaning and shaping in the query so your worksheets contain only analysis. When the source changes, you fix one step in one place instead of hunting through a thousand formulas.

🧠 Key intuitions

  • Every click becomes a step — the step list is the program
  • Click a step to time-travel — see the data exactly as it was there
  • Nothing is destructive — the source file is never touched
  • Set data types first, before any other transformation
  • Filter early so later steps process less data
  • Rename your steps — "Removed test orders" beats "Filtered Rows1"
  • Unpivot to analyse, pivot to present
  • Never type into the output table — refresh will erase it
  • M is case-sensitive, unlike worksheet formulas

🎯 Practice exercises

Desktop Excel required. Load the Superstore data with Data → From Table/Range.

  1. Load the data and keep only Order ID, Order Date, Region, Category, Sales and Profit (use Remove Other Columns)
  2. Check the data types — Sales and Profit as decimal, Order Date as date
  3. Rename the steps you've created so far so they read as plain English
  4. Filter to 2017 orders only
  5. Add a custom column for Profit Margin, guarding against division by zero
  6. Add a conditional column bucketing Sales into High / Medium / Low
  7. Add a column for the order year and another for the month name
  8. Split Order ID on the - delimiter into three columns
  9. Replace "Consumer" with "B2C" in the Segment column
  10. Group By Region to get total Sales, total Profit and an order count
  11. Close & Load the grouped result to a new sheet, then change a value in the source and hit Refresh
  12. Click back through the Applied Steps and confirm you can explain what each one does
🏆 Challenge: a folder pipeline

Split the Superstore data into four CSVs, one per region, and save them in a folder. Then build a query that combines all four, extracts the region from Source.Name, cleans the types, and groups by region and year. Add a fifth file and refresh — if your output grows without you editing anything, you've built a real data pipeline.
