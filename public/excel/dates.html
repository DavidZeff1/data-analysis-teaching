📅 Dates & Time

Almost every business question has a time dimension — this month vs last, year over year, how long did shipping take. Dates look like text but behave like numbers, and understanding that one fact fixes most date problems before they start.

🔑 The secret: every date is just a number

A spreadsheet stores a date as the number of days since 1 January 1900. What you see on screen is only formatting laid over that number.

ABC

21/1/19001day one

411/9/201642,683=A3+1
56:00 AM0.25a quarter of a day

Times are the decimal part: 0.5 is noon, 0.75 is 6pm. A date-time is just both halves together.

✓ Everything that follows makes sense now

  • Subtracting two dates gives you days — it's just subtraction
  • =C2+30 is "30 days later"
  • Sorting dates works because you're sorting numbers
  • > and < mean "after" and "before"

⚠️ And so do the problems

  • A "date" that won't do maths is text, not a date
  • Left-aligned by default = text. Right-aligned = a real date
  • =ISNUMBER(C2) is the definitive test

Today, right now

=TODAY()                // today's date, updates every day
=NOW()                  // date AND time, updates on every recalculation
=TODAY() - C2           // how many days ago was this order placed?

These are volatile — they change on their own. If you need a date frozen in place, type it, or press Ctrl ; to stamp today's date as a fixed value.

🔨 Pulling a date apart, and putting one together

Extract the pieces

// Order Date is in column C
=YEAR(C2)        // 2016
=MONTH(C2)       // 11  (a number, not a name)
=DAY(C2)         // 8
=WEEKDAY(C2)     // 3   (1 = Sunday by default)
=WEEKNUM(C2)     // 46  (week of the year)
=HOUR(C2)        // 0   (no time component here)

Build one from pieces

// DATE(year, month, day)
=DATE(2017, 1, 1)         // 1 Jan 2017

// It handles overflow gracefully
=DATE(2017, 13, 1)        // 1 Jan 2018
=DATE(2017, 1, 0)         // 31 Dec 2016

// First day of the month a date falls in
=DATE(YEAR(C2), MONTH(C2), 1)
Always use DATE() for date criteria

Writing ">1/1/2017" in a SUMIFS is fragile — it depends on your regional date settings. ">="&DATE(2017,1,1) is unambiguous everywhere.

➖ Date arithmetic

Because dates are numbers, you subtract them like numbers. The Superstore data has both an Order Date (C) and a Ship Date (D), which makes shipping lag a one-character formula.

If the result shows as a date instead of a number, the cell inherited date formatting — set it to Number.

// Days between two dates
=D2 - C2                 // 3

// Days ago
=TODAY() - C2

// Add / subtract days
=C2 + 30                 // 30 days after the order
=C2 - 7                  // a week before

// Add months or years properly — EDATE respects month lengths
=EDATE(C2, 3)            // 3 months later
=EDATE(C2, -12)          // same date last year

Don't add 30 to mean "a month" or 365 to mean "a year" — months differ in length and leap years exist. Use EDATE.

DATEDIF — ages and elapsed periods

A hidden legacy function with no autocomplete. It still works, and it's the cleanest way to get whole years or months elapsed.

=DATEDIF(C2, TODAY(), "y")   // complete years elapsed
=DATEDIF(C2, TODAY(), "m")   // complete months elapsed
=DATEDIF(C2, D2, "d")        // days (same as D2-C2)
=DATEDIF(C2, TODAY(), "ym")  // leftover months after whole years

🗓️ Months, quarters and period boundaries

Grouping daily rows into months is the most common date task in analysis. There are two approaches, and they serve different purposes.

A label — for reading
=TEXT(C2, "mmm yyyy")
// "Nov 2016" — but it's TEXT now,
// so it sorts alphabetically: Apr, Aug, Dec…

=TEXT(C2, "yyyy-mm")
// "2016-11" — text, but sorts correctly!
A real date — for grouping & charts
=DATE(YEAR(C2), MONTH(C2), 1)
// 1 Nov 2016 — a genuine date.
// Sorts correctly, charts correctly,
// and you can format it to show "Nov 2016".
Rule of thumb

Use =DATE(YEAR(...), MONTH(...), 1) to group, then apply a mmm yyyy number format to make it read nicely. You get correct sorting and a friendly label. Reach for TEXT only when you need an actual string, and prefer "yyyy-mm" so it still sorts.

Start and end of periods

// Last day of the month C2 falls in
=EOMONTH(C2, 0)             // 30 Nov 2016

// Last day of NEXT month
=EOMONTH(C2, 1)             // 31 Dec 2016

// First day of the month — day after the previous month ends
=EOMONTH(C2, -1) + 1        // 1 Nov 2016

// Quarter number (1–4)
=ROUNDUP(MONTH(C2) / 3, 0)  // 4

// A readable quarter label that sorts correctly
=YEAR(C2) & "-Q" & ROUNDUP(MONTH(C2)/3, 0)   // "2016-Q4"

💼 Working days

"Four days to ship" means something different over a weekend. These two functions skip Saturdays, Sundays, and any holidays you supply.

NETWORKDAYS Count business days between two dates
=NETWORKDAYS( start_date, end_date, [holidays] )
  • start_date First day, counted inclusively
  • end_date Last day, also counted
  • holidays Optional range of dates to also skip
=NETWORKDAYS(C2, D2)              // business days to ship
=NETWORKDAYS(C2, D2, $Z$2:$Z$12)  // minus a holiday list
WORKDAY Find the date N business days from now
=WORKDAY( start_date, days, [holidays] )
=WORKDAY(TODAY(), 5)              // promised delivery date
=WORKDAY(C2, -3)                 // 3 business days before the order

Need a different weekend (say, Friday–Saturday)? Use NETWORKDAYS.INTL and WORKDAY.INTL, which take an extra weekend-pattern argument.

Weekday names, and flagging weekends

=TEXT(C2, "dddd")                    // "Tuesday"
=TEXT(C2, "ddd")                     // "Tue"

// Was it ordered on a weekend?
=IF(WEEKDAY(C2, 2) > 5, "Weekend", "Weekday")
//   the 2 makes Monday=1 … Sunday=7, which is easier to reason about

Formatting vs TEXT() — know which you want

Number formatting keeps the value a date: it still sorts, filters, and charts correctly. =TEXT(C2,"mmm yyyy") produces a real string — good for building labels, fatal if you then try to do date maths on it.

🚑 When dates won't behave

Imported data almost always arrives with some dates as text. Here's how to spot it and fix it.

  1. Diagnose it

    Real dates align right; text aligns left. Confirm with =ISNUMBER(C2) — FALSE means it's text. A quick count of the damage: =SUMPRODUCT(--NOT(ISNUMBER(C2:C9995))).

  2. Try the one-click fix first

    Select the column → Data → Text to Columns → Next → Next → choose Date and the right order (MDY / DMY) → Finish. This converts the whole column in place and solves it most of the time.

  3. Or convert with a formula

    =DATEVALUE(C2) parses a text date into a serial number — then format it as a date. For a text date-time, =DATEVALUE(LEFT(C2,10)) takes just the date part.

  4. Rebuild it if the format is unusual

    For something like 20161108, take it apart and reassemble: =DATE(LEFT(C2,4), MID(C2,5,2), RIGHT(C2,2)).

  5. Verify, then replace the original

    Check the converted column sorts correctly and that MIN/MAX give sensible first and last dates. Then paste-special as values over the original column.

❌ Day/month swapped

03/04/2017 is 3 April in the UK and 4 March in the US. Any row where the day is 13 or higher gives it away — those can only be days. Fix at import with Text to Columns, choosing the correct order.

❌ A date shows as 42682

The value is right, the format was lost. Just apply a date format — nothing is broken.

❌ Two-digit years

Excel reads 00–29 as 2000–2029 and 30–99 as 1930–1999. "65" becomes 1965. Always import four-digit years.

❌ Dates look equal but don't match

One has a time attached. =INT(C2) strips the time and leaves the date.

📈 Putting it to work

Dates only matter because of the questions they let you answer. These combine with SUMIFS.

// Total Sales in 2017
=SUMIFS(R:R, C:C, ">="&DATE(2017,1,1), C:C, "<="&DATE(2017,12,31))

// Total Sales in a single month (Nov 2017)
=SUMIFS(R:R, C:C, ">="&DATE(2017,11,1), C:C, "<="&EOMONTH(DATE(2017,11,1), 0))

// Rolling 90 days from today
=SUMIFS(R:R, C:C, ">="&TODAY()-90, C:C, "<="&TODAY())

// Year-over-year growth, with 2017 in X2 and 2016 in X3
=(X2 - X3) / X3

// Average shipping lag by ship mode
=AVERAGEIFS(V:V, E:E, "Standard Class")   // V holds =D2-C2
Add a date helper block once, reuse it forever

Rather than nesting date functions into every formula, add four helper columns next to your data — Year, Month Start, Quarter, Days to Ship. Every later analysis, pivot table and chart becomes a one-line formula. This is what "prepare your data" actually means in practice.

🧠 Key intuitions

  • A date is a number wearing a costume — the format is the costume
  • Right-aligned = real date, left-aligned = text — the fastest visual check there is
  • Subtracting dates gives days, because it's plain subtraction
  • EDATE for months, EOMONTH for period ends — never +30
  • Group with DATE(YEAR, MONTH, 1), label with a number format
  • DATE() in criteria — typed date strings depend on regional settings
  • TEXT() creates strings — a one-way trip out of date maths
  • Fix broken dates at import, not with formulas layered on top
