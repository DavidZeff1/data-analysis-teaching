Exploratory Data Analysis (EDA) & Data Prep

The crucial first steps before training any Machine Learning model

EDA (Exploratory Data Analysis) is how we "get to know" our dataset. Before we impute missing values, encode categories, or train models, we must understand the shape, distribution, relationships, and anomalies in our data.

1 Initial Inspection & Descriptive Statistics

The very first thing to do when you load a dataset is to look at its fundamental properties: data types, row counts, and summary statistics. 

import pandas as pd

# Load data
df = pd.read_csv('data.csv')

# 1. Quick look at the first and last rows
display(df.head())
display(df.tail())       # Sometimes errors cluster at the end of a file
display(df.sample(5))   # Random sample to spot unexpected patterns

# 2. Shape: how many rows and columns?
print(f"Rows: {df.shape[0]}, Columns: {df.shape[1]}")

# 3. Get data types and non-null counts
# Look out for numeric columns stored as 'object' (strings)
df.info()

# 4. Memory usage — important for large datasets
print(df.memory_usage(deep=True).sum() / 1024**2, "MB")

# 5. Summary statistics for numerical columns
# Checks mean, std, min, max, and quartiles (25%, 50%, 75%)
display(df.describe())

# 6. Summary for categorical columns
display(df.describe(include=['object', 'category']))

🔍 Checking Unique Values & Spotting Issues

# How many unique values per column? Helps identify:
# - IDs (all unique), binary flags (2 unique), low-cardinality categoricals
print(df.nunique())

# Spot columns with suspiciously few unique values
# E.g., a 'gender' column might have: ['Male', 'Female', 'male', 'M', '']
for col in df.select_dtypes(include='object').columns:
    print(f"\n{col}:")
    print(df[col].value_counts(dropna=False))

# Check for columns where numeric data is hidden as strings
# Common culprits: '$100', '5%', '1,000', 'N/A', '-'
for col in df.select_dtypes(include='object').columns:
    sample_vals = df[col].dropna().head(5).tolist()
    print(f"{col}: {sample_vals}")

📊 Missing Data Overview

# Missing values count and percentage per column
missing = df.isnull().sum()
missing_pct = (missing / len(df)) * 100
missing_df = pd.DataFrame({
    'Missing Count': missing,
    'Missing %': missing_pct.round(2)
}).sort_values('Missing %', ascending=False)
display(missing_df[missing_df['Missing Count'] > 0])

# Are missing values random or patterned?
# If two columns are always missing together, that's a pattern
print(df.isnull().corr())  # High correlation = missing together
💡 What to look for: Is the mean vastly different from the median (50%)? → Your data is skewed. Is the max value impossibly high? → Outliers or data entry errors. Are there columns with mostly missing values (>50%)? → Consider dropping them. Does a column have only 1 unique value? → It provides no information, drop it.

2 Handling Duplicates & Data Types

Duplicate rows can bias your model by giving double importance to certain observations. Incorrect data types (like a date stored as text) will break ML algorithms. 

# Check for exact duplicate rows
duplicate_count = df.duplicated().sum()
print(f"Found {duplicate_count} duplicate rows")

# Drop them (keep the first occurrence)
df = df.drop_duplicates()

# Fixing Data Types
# 'price' might have a '$' sign, making it an object. We need to clean it:
df['price'] = df['price'].str.replace('$', '').astype(float)

# Convert to datetime
df['date'] = pd.to_datetime(df['date'])

3 Univariate Analysis (Distributions)

Analyzing one variable at a time using visualizations. This helps identify the shape of the data (normal, skewed, bimodal). 

import matplotlib.pyplot as plt
import seaborn as sns

# 1. Numerical: Histograms and Boxplots
fig, axes = plt.subplots(1, 2, figsize=(12, 4))
sns.histplot(df['income'], kde=True, ax=axes[0])
axes[0].set_title('Income Distribution')

sns.boxplot(x=df['income'], ax=axes[1])
axes[1].set_title('Income Boxplot (detects outliers)')
plt.show()

# 2. Categorical: Countplots
plt.figure(figsize=(8, 4))
sns.countplot(data=df, x='department', order=df['department'].value_counts().index)
plt.xticks(rotation=45)
plt.title('Employee Count by Department')
plt.show()

4 Outlier Detection & Handling

Outliers can severely impact linear models (like Linear/Logistic Regression) and distance-based models (like KNN).

IQR Method (Robust)

Calculates the Interquartile Range (75th percentile - 25th percentile). Any value outside Q1 - 1.5*IQR or Q3 + 1.5*IQR is an outlier. Best for skewed data.

Z-Score Method

Measures how many standard deviations a value is from the mean. Values with a Z-score > 3 or < -3 are often considered outliers. Assumes data is normally distributed.

# Handling outliers via Capping (Winsorization) using IQR
Q1 = df['income'].quantile(0.25)
Q3 = df['income'].quantile(0.75)
IQR = Q3 - Q1

lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR

# Option A: Remove them
df_clean = df[(df['income'] >= lower_bound) & (df['income'] <= upper_bound)]

# Option B: Cap them (Winsorize) - replace extremes with the bounds
df['income_capped'] = df['income'].clip(lower=lower_bound, upper=upper_bound)
Note: For multidimensional or complex outlier detection, you may use algorithms like Isolation Forest (see the Anomaly Detection section).

5 Bivariate Analysis & Correlations

Understanding how features relate to each other and to the target variable. Highly correlated features (multicollinearity) can harm certain linear models. 

# 1. Scatter Plot (Numerical vs Numerical)
plt.figure(figsize=(6, 4))
sns.scatterplot(data=df, x='age', y='income', hue='target_class')
plt.title('Age vs Income (Colored by Target)')
plt.show()

# 2. Correlation Matrix Heatmap (Linear Relationships)
plt.figure(figsize=(8, 6))
corr_matrix = df.select_dtypes(include=['number']).corr()

# sns.heatmap makes it easy to spot strong positive/negative correlations
sns.heatmap(corr_matrix, annot=True, cmap='coolwarm', fmt=".2f")
plt.title('Feature Correlation Matrix')
plt.show()

6 Feature Scaling & Transformation

ML models like SVM, KNN, and Neural Networks require features to be on the same scale (e.g., salary in $100,000s shouldn't dominate age in 10s).

Standardization

StandardScaler shifts data to have Mean=0 and Std=1. Best for normally distributed data.

Normalization

MinMaxScaler scales data between 0 and 1. Good for image pixels or when you know exact bounded limits.

Log Transformation

Applying np.log1p() squashes highly right-skewed data (like incomes or page views) into a more normal shape.

from sklearn.preprocessing import StandardScaler, MinMaxScaler
import numpy as np

# 1. Standardization (Mean=0, Std=1)
scaler = StandardScaler()
df['age_scaled'] = scaler.fit_transform(df[['age']])

# 2. Log Transformation for highly skewed data
# log1p is log(1+x), safe for zero values
df['income_log'] = np.log1p(df['income'])

# Visualizing the fix
fig, axes = plt.subplots(1, 2, figsize=(10, 4))
sns.histplot(df['income'], ax=axes[0]).set_title('Original (Skewed)')
sns.histplot(df['income_log'], ax=axes[1]).set_title('Log Transformed')
plt.show()
