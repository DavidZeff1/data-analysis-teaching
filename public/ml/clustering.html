Clustering

K-Means, Hierarchical, DBSCAN & GMM — Discovering Hidden Structure in Unlabeled Data

What Is Clustering?

Clustering is an unsupervised learning technique that groups data points so that points within the same group (a cluster) are more similar to each other than to those in other groups. There are no labels; the algorithm finds structure on its own.

🎯 Typical Use Cases

  • Customer segmentation for marketing
  • Document / image grouping by topic or content
  • Anomaly detection (points that fit no cluster)
  • Image compression via colour quantization
  • Exploratory analysis — finding natural groupings before modelling

🔑 Core Idea

Every clustering algorithm needs two ingredients:

  • A distance / similarity metric (Euclidean, cosine, Manhattan…)
  • An objective (minimise within-cluster distance, maximise density, etc.)

Different metrics + objectives → very different cluster shapes.

⚠️ Always Scale First

Distance-based algorithms are dominated by features with the largest scale. Apply StandardScaler or MinMaxScaler before clustering, otherwise a feature in dollars will overwhelm a feature in years.

K-Means

K-Means partitions the data into K clusters by repeatedly moving cluster centres (centroids) to minimise the total squared distance between each point and its nearest centroid.

The Algorithm

  1. Pick K initial centroids (usually with k-means++ for good seeding)
  2. Assign each point to its nearest centroid
  3. Update each centroid to the mean of its assigned points
  4. Repeat 2–3 until assignments no longer change (or max iterations hit)

Objective (within-cluster sum of squares, WCSS / inertia):

$J = \sum_{k=1}^{K} \sum_{x_i \in C_k} \|x_i - \mu_k\|^2$

where $\mu_k$ is the centroid of cluster $C_k$.

✓ Strengths

  • Fast and scales to large datasets — $O(n \cdot k \cdot d \cdot i)$
  • Easy to understand and implement
  • Works well when clusters are spherical and similarly sized

✗ Weaknesses

  • Must choose K in advance
  • Assumes spherical, equal-variance clusters — fails on moons, rings, elongated shapes
  • Sensitive to outliers (they pull centroids)
  • Result depends on initial centroid placement (use n_init)
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

# Always scale first for distance-based methods
X_scaled = StandardScaler().fit_transform(X)

kmeans = KMeans(
    n_clusters=4,
    init='k-means++',   # smart seeding
    n_init=10,           # run 10 times, keep best
    max_iter=300,
    random_state=42,
)
labels = kmeans.fit_predict(X_scaled)

print("Inertia (WCSS):", kmeans.inertia_)
print("Centroids:", kmeans.cluster_centers_)

Picking K: the Elbow & Silhouette Methods

Elbow Method

Plot WCSS for $K = 1, 2, \dots, 10$. Pick the $K$ at the "elbow" where adding another cluster stops producing a big drop in WCSS.

Silhouette Score

For each point, compare its average distance to its own cluster ($a$) vs the nearest other cluster ($b$): $s = (b - a) / \max(a, b)$. Ranges from $-1$ to $1$ — higher is better. Pick $K$ with the highest average silhouette. 

from sklearn.metrics import silhouette_score
import matplotlib.pyplot as plt

wcss, sil = [], []
ks = range(2, 11)
for k in ks:
    km = KMeans(n_clusters=k, n_init=10, random_state=42).fit(X_scaled)
    wcss.append(km.inertia_)
    sil.append(silhouette_score(X_scaled, km.labels_))

fig, ax = plt.subplots(1, 2, figsize=(12, 4))
ax[0].plot(ks, wcss, 'o-'); ax[0].set_title("Elbow")
ax[1].plot(ks, sil, 'o-');  ax[1].set_title("Silhouette")
plt.show()

Hierarchical Clustering

Builds a tree of clusters (a dendrogram) instead of a flat partition. You can cut the tree at any height to get any number of clusters — no need to pre-choose $K$.

Agglomerative (bottom-up)

Start with each point as its own cluster, then repeatedly merge the two closest clusters until everything is one big cluster. This is what sklearn implements.

Divisive (top-down)

Start with one cluster containing all points and recursively split. Less common in practice.

Linkage — how distance between clusters is measured

  • Ward — minimises variance increase from merging (default; works well on roughly spherical clusters)
  • Single — distance between closest pair (can produce long "chains")
  • Complete — distance between farthest pair (compact clusters)
  • Average — mean of all pairwise distances (a balance)
from sklearn.cluster import AgglomerativeClustering
from scipy.cluster.hierarchy import dendrogram, linkage
import matplotlib.pyplot as plt

# Build & plot the dendrogram to choose a cut height
Z = linkage(X_scaled, method='ward')
plt.figure(figsize=(10, 4))
dendrogram(Z, truncate_mode='lastp', p=30)
plt.title("Dendrogram"); plt.show()

# Fit a flat clustering once you know how many groups you want
hc = AgglomerativeClustering(n_clusters=4, linkage='ward')
labels = hc.fit_predict(X_scaled)

✓ Strengths

  • No need to pre-specify $K$ — explore via the dendrogram
  • Reveals nested cluster structure
  • Deterministic (no random init)

✗ Weaknesses

  • $O(n^2)$ memory — impractical above ~10k points
  • Greedy merges can't be undone
  • Sensitive to choice of linkage and metric

DBSCAN (Density-Based)

DBSCAN groups together points in dense regions and marks points in low-density regions as noise. It can discover clusters of arbitrary shape and does not need $K$.

Two Key Parameters

  • eps ($\varepsilon$) — radius defining a point's neighbourhood
  • min_samples — minimum points needed within $\varepsilon$ for a "core point"

A point is a core point if at least min_samples other points lie within $\varepsilon$. Clusters are formed by chaining core points and their neighbours. Points not reachable from any core point are labelled noise (label = -1).

💡 Choosing eps

Compute each point's distance to its min_samples-th nearest neighbour, sort, and plot. The "knee" of that curve is a good eps. 

from sklearn.cluster import DBSCAN
from sklearn.neighbors import NearestNeighbors
import numpy as np
import matplotlib.pyplot as plt

# 1. Find a good eps from the k-distance plot
k = 5
dists, _ = NearestNeighbors(n_neighbors=k).fit(X_scaled).kneighbors(X_scaled)
plt.plot(np.sort(dists[:, k - 1])); plt.ylabel("k-distance"); plt.show()

# 2. Fit DBSCAN
db = DBSCAN(eps=0.5, min_samples=5).fit(X_scaled)
labels = db.labels_
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise = list(labels).count(-1)
print(f"Clusters: {n_clusters}, noise points: {n_noise}")

✓ Strengths

  • Finds non-convex clusters (moons, rings)
  • Identifies outliers natively (label -1)
  • No need to choose $K$

✗ Weaknesses

  • Struggles when clusters have very different densities
  • Sensitive to eps — small change → very different result
  • Degrades in high dimensions (distance becomes uniform)

Gaussian Mixture Models (GMM)

A GMM assumes the data is generated by a mixture of $K$ multivariate Gaussian distributions. Each point gets a probability of belonging to each cluster — this is soft clustering, in contrast to K-Means' hard assignments.

Fit with Expectation-Maximisation (EM)

  1. E-step: compute the probability of each point belonging to each Gaussian
  2. M-step: update each Gaussian's mean, covariance and mixing weight using those probabilities
  3. Repeat until the log-likelihood converges
from sklearn.mixture import GaussianMixture

gmm = GaussianMixture(
    n_components=4,
    covariance_type='full',  # 'full', 'tied', 'diag', 'spherical'
    random_state=42,
).fit(X_scaled)

hard_labels = gmm.predict(X_scaled)
probs       = gmm.predict_proba(X_scaled)   # soft assignments

# Pick K with BIC / AIC (lower is better)
print("BIC:", gmm.bic(X_scaled), "AIC:", gmm.aic(X_scaled))

GMM is more flexible than K-Means because clusters can be elongated and have different sizes (via the covariance matrix). Use BIC across a range of $K$ to pick the number of components.

Evaluating a Clustering

Without ground-truth labels you cannot use accuracy. Use internal metrics that score cluster quality from the data itself, or external metrics if labels happen to exist.

Internal (no labels)

  • Silhouette — $[-1, 1]$, higher is better
  • Calinski–Harabasz — ratio of between- to within-cluster dispersion; higher is better
  • Davies–Bouldin — average similarity between each cluster and its most similar; lower is better
  • WCSS / inertia — for the elbow method (K-Means only)

External (labels available)

  • Adjusted Rand Index (ARI) — agreement between predicted and true clusters, corrected for chance
  • Normalised Mutual Information (NMI) — information shared between two clusterings
  • Homogeneity / Completeness / V-measure — purity-style scores

When to Use Which Algorithm

Algorithm Choose When Avoid When
K-Means Large dataset, roughly spherical clusters, you know $K$ Non-convex shapes, many outliers, very different cluster sizes
Hierarchical Small/medium data, want a dendrogram, nested structure Large data (memory blows up); need a fast online model
DBSCAN Arbitrary cluster shapes, want outliers flagged, $K$ unknown Very high dimensions, clusters of very different densities
GMM Need soft assignments, elliptical clusters, probabilistic model Cluster shapes are clearly non-Gaussian; very large $K$

⚠️ Common Pitfalls

  • Forgetting to scale features — distance is meaningless on raw values
  • Choosing $K$ purely by elbow without checking silhouette or business meaning
  • Running clustering on high-dimensional raw data — reduce with PCA / UMAP first
  • Treating cluster labels as ground truth — they are hypotheses to validate
  • Comparing clusterings with different scalings or random seeds without fixing them
