Model Assumptions & Data Transformations

Understanding what your models expect — and how to fix violations

Many machine learning models — especially linear regression, logistic regression, and classical statistical tests — make assumptions about your data. When these assumptions are violated, your model's predictions, coefficients, and p-values can become unreliable. This page covers what those assumptions are, how to diagnose violations, and the transformations you can apply to fix them.

📐 Part 1 — Key Model Assumptions

The rules your data should follow for certain models to work correctly

1 Linearity

Linear regression assumes the relationship between each feature and the target is linear. If the true relationship is curved (quadratic, exponential, etc.), a straight line will systematically miss patterns.

Which models need this? Linear Regression, Logistic Regression. Tree-based models (Random Forest, XGBoost) do not require linearity.

How to Check

import matplotlib.pyplot as plt
import seaborn as sns
import numpy as np

# Scatter plot: feature vs target
# Look for a straight-line pattern (good) vs curves (violation)
fig, axes = plt.subplots(1, 3, figsize=(15, 4))

for i, col in enumerate(['feature_1', 'feature_2', 'feature_3']):
    axes[i].scatter(df[col], df['target'], alpha=0.4)
    axes[i].set_xlabel(col)
    axes[i].set_ylabel('target')
    axes[i].set_title(f'{col} vs target')

plt.tight_layout()
plt.show()
# Residuals vs Fitted plot (after fitting the model)
from sklearn.linear_model import LinearRegression

model = LinearRegression()
model.fit(X_train, y_train)
y_pred = model.predict(X_train)
residuals = y_train - y_pred

plt.figure(figsize=(8, 5))
plt.scatter(y_pred, residuals, alpha=0.4)
plt.axhline(y=0, color='red', linestyle='--')
plt.xlabel('Fitted Values')
plt.ylabel('Residuals')
plt.title('Residuals vs Fitted — look for random scatter')
plt.show()

# ✅ Good: random cloud around 0
# ❌ Bad: U-shape or pattern = non-linear relationship

Fix: Apply a non-linear transformation (log, sqrt, polynomial features) to the feature or target, or switch to a non-linear model.

2 Normality of Residuals / Features

Linear regression assumes the residuals (errors) are normally distributed. Many statistical tests (t-test, ANOVA) also assume the data itself follows a normal distribution. Heavily skewed features can also hurt model performance even for models that don't strictly require normality.

Which models need this? Linear Regression (residuals), statistical tests (t-test, ANOVA). KNN and SVMs can also benefit from normally distributed features. Tree-based models are unaffected.

Visual Checks

import scipy.stats as stats

# 1. Histogram — does it look bell-shaped?
fig, axes = plt.subplots(1, 2, figsize=(12, 4))

axes[0].hist(df['feature'], bins=30, edgecolor='black', alpha=0.7)
axes[0].set_title('Histogram')

# 2. Q-Q Plot — points should fall on the diagonal line
stats.probplot(df['feature'], dist="norm", plot=axes[1])
axes[1].set_title('Q-Q Plot')

plt.tight_layout()
plt.show()

# ✅ Good: points hug the red line
# ❌ Bad: S-curve or banana shape = not normal

Statistical Tests for Normality 

from scipy.stats import shapiro, normaltest

# Shapiro-Wilk test (best for n < 5000)
stat, p_value = shapiro(df['feature'])
print(f"Shapiro-Wilk: stat={stat:.4f}, p={p_value:.4f}")
if p_value > 0.05:
    print("✅ Cannot reject normality (data looks normal)")
else:
    print("❌ Reject normality (data is NOT normal)")

# D'Agostino-Pearson test (better for larger samples)
stat, p_value = normaltest(df['feature'])
print(f"D'Agostino: stat={stat:.4f}, p={p_value:.4f}")

# Skewness and Kurtosis — quick numeric check
print(f"Skewness: {df['feature'].skew():.3f}")   # Near 0 = symmetric
print(f"Kurtosis: {df['feature'].kurtosis():.3f}") # Near 0 = normal tails

Skewness guide:
|skew| < 0.5 → approximately symmetric
0.5–1.0 → moderately skewed
> 1.0 → highly skewed (needs transformation)

Kurtosis guide:
Near 0 → normal tails
> 0 (leptokurtic) → heavy tails, sharp peak
< 0 (platykurtic) → light tails, flat peak

3 Homoscedasticity (Constant Variance)

Linear regression assumes the spread of residuals is constant across all levels of the predicted values. When the spread fans out (or narrows), it's called heteroscedasticity — and it makes your standard errors wrong, which breaks confidence intervals and p-values.

Which models need this? Linear Regression, Logistic Regression. Not required by tree-based models or KNN.

Visual Check: Scale-Location Plot 

# Residuals vs Fitted plot — check for fanning
y_pred = model.predict(X_train)
residuals = y_train - y_pred

plt.figure(figsize=(8, 5))
plt.scatter(y_pred, np.sqrt(np.abs(residuals)), alpha=0.4)
plt.xlabel('Fitted Values')
plt.ylabel('√|Residuals|')
plt.title('Scale-Location Plot')
plt.show()

# ✅ Good: flat, random band
# ❌ Bad: cone/fan shape = heteroscedasticity

Statistical Test: Breusch-Pagan 

import statsmodels.api as sm
from statsmodels.stats.diagnostic import het_breuschpagan

# Fit with statsmodels (adds intercept automatically with add_constant)
X_with_const = sm.add_constant(X_train)
ols_model = sm.OLS(y_train, X_with_const).fit()

# Breusch-Pagan test
bp_stat, bp_pvalue, _, _ = het_breuschpagan(ols_model.resid, X_with_const)
print(f"Breusch-Pagan p-value: {bp_pvalue:.4f}")
if bp_pvalue > 0.05:
    print("✅ Homoscedasticity holds")
else:
    print("❌ Heteroscedasticity detected")

Fix: Apply a log or Box-Cox transform to the target variable, use Weighted Least Squares (WLS), or use robust standard errors.

4 Multicollinearity

When two or more features are highly correlated with each other, the model can't tell which one is actually driving the prediction. This makes individual coefficient estimates unstable and uninterpretable, even though the overall model R² might still look fine.

Which models are affected? Linear Regression and Logistic Regression (coefficients become unreliable). Tree-based models handle collinearity naturally but may produce misleading feature importance scores.

Detection: Correlation Matrix 

# Correlation heatmap — look for values above |0.8|
corr = df.select_dtypes(include='number').corr()

plt.figure(figsize=(10, 8))
sns.heatmap(corr, annot=True, fmt='.2f', cmap='RdBu_r',
            center=0, vmin=-1, vmax=1)
plt.title('Correlation Matrix')
plt.show()

# Flag highly correlated pairs
high_corr = corr.abs().unstack()
high_corr = high_corr[high_corr < 1.0]  # remove self-correlation
high_corr = high_corr[high_corr > 0.8].drop_duplicates()
print("Highly correlated pairs:")
print(high_corr.sort_values(ascending=False))

Detection: Variance Inflation Factor (VIF) 

from statsmodels.stats.outliers_influence import variance_inflation_factor
import pandas as pd

# VIF measures how much a feature is explained by OTHER features
# VIF = 1     → No collinearity
# VIF = 1-5   → Moderate (usually OK)
# VIF = 5-10  → High (concerning)
# VIF > 10    → Severe (must address)

X_numeric = df.select_dtypes(include='number').drop(columns=['target'])
X_with_const = sm.add_constant(X_numeric)

vif_data = pd.DataFrame({
    'Feature': X_with_const.columns,
    'VIF': [variance_inflation_factor(X_with_const.values, i)
           for i in range(X_with_const.shape[1])]
})
print(vif_data.sort_values('VIF', ascending=False))

Fix: Drop one of the correlated features, combine them (PCA or averaging), or use regularization (Ridge/Lasso) which handles collinearity by shrinking coefficients.

5 Independence of Observations

Most models assume each data point is independent — one observation doesn't influence another. This is commonly violated in time-series data (today's stock price depends on yesterday's) or clustered data (students within the same classroom).

Detection: Durbin-Watson Test 

from statsmodels.stats.stattools import durbin_watson

# Durbin-Watson test for autocorrelation in residuals
# Value near 2 → no autocorrelation ✅
# Value near 0 → positive autocorrelation ❌
# Value near 4 → negative autocorrelation ❌

dw_stat = durbin_watson(ols_model.resid)
print(f"Durbin-Watson: {dw_stat:.4f}")
if 1.5 < dw_stat < 2.5:
    print("✅ No significant autocorrelation")
else:
    print("❌ Possible autocorrelation — check data ordering")

Fix: Use time-series specific models (ARIMA), add lag features, or use clustered standard errors for grouped data.

🔧 Part 2 — Data Transformations

Methods to make your data more "well-behaved" when assumptions are violated

6 Log Transformation

The most common transformation for right-skewed data (long tail on the right). It compresses large values and spreads out small values, making the distribution more symmetric.

Best for:
• Income, prices, population
• Counts (page views, sales)
• Any feature with a right skew

Limitations:
• Cannot handle zero values (use log1p)
• Cannot handle negative values
• Doesn't fix left-skewed data 

import numpy as np

# Log transform — use log1p to safely handle zeros
df['income_log'] = np.log1p(df['income'])  # log(1 + x)

# Compare before and after
fig, axes = plt.subplots(1, 2, figsize=(12, 4))

axes[0].hist(df['income'], bins=30, edgecolor='black', alpha=0.7)
axes[0].set_title(f"Original (skew={df['income'].skew():.2f})")

axes[1].hist(df['income_log'], bins=30, edgecolor='black', alpha=0.7, color='green')
axes[1].set_title(f"Log Transformed (skew={df['income_log'].skew():.2f})")

plt.tight_layout()
plt.show()

# To reverse the transformation (for interpreting predictions):
# original = np.expm1(predicted_log)   # inverse of log1p

7 Square Root Transformation

A milder version of the log transform. Good for moderate right skew and particularly common with count data. Unlike log, it can handle zero values directly. 

# Square root transform
df['count_sqrt'] = np.sqrt(df['count'])

# Compare
print(f"Original skew:   {df['count'].skew():.3f}")
print(f"Sqrt skew:       {df['count_sqrt'].skew():.3f}")

# Reverse: original = transformed ** 2

When to use sqrt vs log: Start with log. If log over-corrects (makes data left-skewed), try sqrt instead. If the data has zeros, sqrt is convenient since it doesn't require log1p.

8 Box-Cox & Yeo-Johnson Transforms

Instead of guessing which transform to use, Box-Cox and Yeo-Johnson automatically find the best power parameter (λ) that makes your data as close to normal as possible. Box-Cox requires strictly positive data; Yeo-Johnson works with any data (including zeros and negatives). 

from scipy.stats import boxcox
from sklearn.preprocessing import PowerTransformer

# === Box-Cox (data must be strictly positive) ===
df['feature_boxcox'], best_lambda = boxcox(df['feature'])
print(f"Best λ: {best_lambda:.4f}")
# λ ≈ 0  → log transform
# λ ≈ 0.5 → square root transform
# λ ≈ 1  → no transform needed

# === Yeo-Johnson (handles zeros and negatives) ===
pt = PowerTransformer(method='yeo-johnson')
df['feature_yj'] = pt.fit_transform(df[['feature']])

# Compare all transformations
fig, axes = plt.subplots(1, 3, figsize=(15, 4))

axes[0].hist(df['feature'], bins=30, edgecolor='black')
axes[0].set_title(f"Original (skew={df['feature'].skew():.2f})")

axes[1].hist(df['feature_boxcox'], bins=30, edgecolor='black', color='orange')
axes[1].set_title(f"Box-Cox (skew={df['feature_boxcox'].skew():.2f})")

axes[2].hist(df['feature_yj'], bins=30, edgecolor='black', color='purple')
axes[2].set_title(f"Yeo-Johnson (skew={df['feature_yj'].skew():.2f})")

plt.tight_layout()
plt.show()

Using PowerTransformer in a Pipeline 

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import PowerTransformer
from sklearn.linear_model import LinearRegression

# PowerTransformer in a pipeline — fits and transforms automatically
pipe = Pipeline([
    ('transform', PowerTransformer(method='yeo-johnson')),
    ('model', LinearRegression())
])

pipe.fit(X_train, y_train)
score = pipe.score(X_test, y_test)
print(f"R² with Yeo-Johnson: {score:.4f}")

9 Other Useful Transforms 

# Reciprocal transform — for severe right skew
df['feature_recip'] = 1 / df['feature']  # data must be non-zero

# Cube root — works with negative values too
df['feature_cbrt'] = np.cbrt(df['feature'])

# Polynomial features — capture non-linear relationships
from sklearn.preprocessing import PolynomialFeatures

poly = PolynomialFeatures(degree=2, include_bias=False)
X_poly = poly.fit_transform(X_train)
print(f"Features before: {X_train.shape[1]}, after: {X_poly.shape[1]}")
# Adds x², x³, x₁·x₂ interaction terms

Warning: Polynomial features increase dimensionality quickly. With 10 features and degree=2, you go from 10 to 65 features. Use with feature selection or regularization.

A practical workflow for checking assumptions and applying fixes

11 Transformation Decision Guide

Step 1: Check skewness of each numeric feature

If |skew| > 1.0 → the feature likely needs a transformation

↓

Step 2: Choose the right transform

  • Right-skewed, all positive → try log first, then Box-Cox
  • Right-skewed, has zeros → use log1p or sqrt
  • Right-skewed, has negatives → use Yeo-Johnson
  • Left-skewed → reflect first (max - x), then log; or use Yeo-Johnson
  • Moderate skew → try sqrt
  • Not sure → use Yeo-Johnson (it handles everything)
↓

Step 3: Verify the transformation worked

Re-check skewness, re-plot distributions, re-run normality tests

↓

Step 4: Compare model performance

Train your model with and without the transformation. Keep whichever gives better cross-validation scores.

Complete Workflow Example 

import pandas as pd
import numpy as np
from scipy.stats import shapiro
from sklearn.preprocessing import PowerTransformer
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import cross_val_score

# ---- Step 1: Assess skewness of all numeric features ----
numeric_cols = df.select_dtypes(include='number').columns
skew_report = df[numeric_cols].skew().sort_values(ascending=False)
print("=== Skewness Report ===")
print(skew_report)
print()

# Flag features that need transformation
skewed_features = skew_report[skew_report.abs() > 1.0].index.tolist()
print(f"Features needing transformation: {skewed_features}")

# ---- Step 2: Apply Yeo-Johnson to skewed features ----
pt = PowerTransformer(method='yeo-johnson')
df[skewed_features] = pt.fit_transform(df[skewed_features])

# ---- Step 3: Verify ----
print("=== After Transformation ===")
print(df[skewed_features].skew())

# ---- Step 4: Compare model performance ----
# (run cross-validation before and after to decide)
X = df.drop(columns=['target'])
y = df['target']

scores = cross_val_score(LinearRegression(), X, y, cv=5, scoring='r2')
print(f"R² after transform: {scores.mean():.4f} ± {scores.std():.4f}")

12 Transforming the Target Variable

When the target is skewed, transforming it can improve both normality of residuals and homoscedasticity in one step. But you must reverse the transform when making predictions. 

from sklearn.compose import TransformedTargetRegressor

# TransformedTargetRegressor handles the transform + inverse automatically
model = TransformedTargetRegressor(
    regressor=LinearRegression(),
    func=np.log1p,       # transform applied to y before fitting
    inverse_func=np.expm1 # reverse applied to predictions
)

model.fit(X_train, y_train)
y_pred = model.predict(X_test)  # predictions are in original scale!

# Or use PowerTransformer for automatic lambda selection
model_auto = TransformedTargetRegressor(
    regressor=LinearRegression(),
    transformer=PowerTransformer(method='yeo-johnson')
)
model_auto.fit(X_train, y_train)

Common mistake: Manually transforming y with log, fitting a model, predicting, and then forgetting to apply np.expm1() to reverse it. Use TransformedTargetRegressor to avoid this.

Key Takeaway

Not every model needs every assumption. Tree-based models are the most forgiving. Linear models are the most demanding. Always check assumptions after fitting, and compare model performance with and without transformations to decide if they actually help.
