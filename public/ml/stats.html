Statistical Tests in Machine Learning

Selecting the right statistical test for feature selection, evaluation, and hypothesis testing

Statistical tests are essential in Machine Learning for Feature Selection, comparing group differences, and understanding relationships between variables before training models. Below is a comprehensive guide based on your data types and objectives.

💡 Why Do We Need Statistical Tests in ML?

Machine Learning models are powerful, but they can easily learn noise or false patterns if we feed them garbage data. Statistical tests provide the mathematical proof we need to make rigorous decisions before we train any models.

1. Feature Selection

Not all features are useful. If a statistical test proves a feature has no significant relationship with the target variable, we can safely drop it. This reduces noise, speeds up training, and prevents overfitting.

2. Avoiding Multicollinearity

If two independent features are highly correlated (proven by Pearson/Spearman correlation), they provide duplicate information, which can destabilize models like Logistic Regression. Testing helps us remove redundant features.

3. Proving A/B Test Results

When deploying a model, we often run A/B tests (e.g., Model A vs. Model B). Statistical tests (like T-Tests) tell us if Model B's higher conversion rate is actually better, or just a result of random luck.

4. Validating Anomalies

Statistical tests help us understand if a data point is a true outlier mathematically, or if it naturally falls within the expected variance of the data distribution.

1 Comparing Groups (Hypothesis Testing)

These tests help you determine if changes or differences across categories are statistically significant, or if they are likely just due to random chance.

Test What it Checks When to Use (Data Types) Non-Parametric Alternative
t-test Checks if there is a significant difference between the means of two groups. Evaluates if the average difference is large enough relative to the internal variance (standard deviation) to claim it is not due to luck. Dependent: Continuous
Explanatory: Dichotomous (2 groups)		 Mann-Whitney U test
ANOVA Compares means of more than two groups simultaneously. Checks the variance within the groups versus the variance between the groups. Dependent: Continuous
Explanatory: Categorical (≥3 levels)		 Kruskal-Wallis test
Chi-Square (χ²) Checks the fit or independence between categories. Answers: "Is there a dependency between these two categorical variables?" Feature 1: Categorical (Nominal/Ordinal)
Feature 2: Categorical		 Fisher's exact test

2 Measuring Correlation

Correlation helps us identify how features interact together linearly or non-linearly. In ML, highly correlated independent features represent multicollinearity and one can typically be removed.

Feature / Aspect Pearson (r) Spearman (ρ)
What is measured? Linear relationship between two continuous variables. Monotonic relationship (strictly increasing/decreasing) between variables.
Suitable Data Types Continuous, approximately normally distributed. Continuous or ordinal variables.
Value Range -1 ≤ r ≤ 1 -1 ≤ ρ ≤ 1
Interpretation
  • r = 1 Perfect linear positive
  • r = 0 No linear relationship
  • r = -1 Perfect linear negative
  • ρ = 1 Perfect monotonic positive
  • ρ = 0 No monotonic relationship
  • ρ = -1 Perfect monotonic negative
Sensitivity to Outliers High (Very affected by outliers) Lower (Robust and stable against outliers)
Common Use Cases Scores, physical measurements, heights, weights. Rankings, surveys, non-normal distributions.

3 Python Implementation Guide 

import pandas as pd
from scipy import stats

# ──────────────────────────────────────────
# 1. T-Test Example (Continuous Target vs Binary Feature)
# Evaluating if there's a significant salary difference between Genders
male_salaries = df[df['Gender'] == 'M']['Salary']
female_salaries = df[df['Gender'] == 'F']['Salary']

t_stat, p_value = stats.ttest_ind(male_salaries, female_salaries)
print(f"T-Test p-value: {p_value:.4f}")

# ✅ What to do with the result:
if p_value < 0.05:
    print("✅ Significant difference → Keep 'Gender' as a feature")
else:
    print("❌ No significant difference → Consider dropping 'Gender'")

# WHY:
# p < 0.05 means there's less than a 5% chance the observed salary gap
# is just random noise. The two groups are genuinely drawn from
# different distributions, so Gender carries real predictive signal.
#
# p >= 0.05 means the difference we see could easily be due to chance
# alone. Gender doesn't meaningfully separate the salary values,
# so it won't help a model and may just add noise.


# ──────────────────────────────────────────
# 2. ANOVA Example (Continuous Target vs Categorical >=3 Levels)
# Checking if Region affects Sales significantly
ny_sales = df[df['Region'] == 'NY']['Sales']
tx_sales = df[df['Region'] == 'TX']['Sales']
ca_sales = df[df['Region'] == 'CA']['Sales']

f_stat, p_value = stats.f_oneway(ny_sales, tx_sales, ca_sales)
print(f"ANOVA p-value: {p_value:.4f}")

# ✅ What to do with the result:
# p < 0.05  → At least one region's sales distribution is different. Keep 'Region'.
# p >= 0.05 → All regions have similar sales. Region doesn't add predictive value.
#
# WHY:
# ANOVA compares the variance *between* groups to the variance *within* groups.
# A high F-statistic (low p) means the group means are far apart relative
# to the spread inside each group — i.e., Region actually explains some of
# the variation in Sales, not just random fluctuation within each region.
#
# ⚠️ ANOVA only tells you *that* groups differ, not *which* ones.
# Run a post-hoc Tukey test to find out which pairs are actually different:
from statsmodels.stats.multicomp import pairwise_tukeyhsd
tukey = pairwise_tukeyhsd(df['Sales'], df['Region'])
print(tukey.summary())
# WHY Tukey: Running multiple T-Tests between every pair would inflate your
# false positive rate (e.g. 3 pairs = 3 chances to get a fluke p < 0.05).
# Tukey corrects for this by adjusting the threshold for each comparison.


# ──────────────────────────────────────────
# 3. Chi-Square Example (Categorical vs Categorical)
# Checking if Product Preference depends on Gender
contingency_table = pd.crosstab(df['Gender'], df['Preference'])
chi2_stat, p_val, dof, expected = stats.chi2_contingency(contingency_table)
print(f"Chi-Square p-value: {p_val:.4f}")

# ✅ What to do with the result:
# p < 0.05  → Association exists → Keep both features, or engineer an interaction term.
# p >= 0.05 → No association → The two variables are statistically independent.
#
# WHY:
# Chi-Square compares the *observed* counts in each cell of the table to the
# *expected* counts if the two variables had zero relationship.
# A large chi2_stat means the real distribution looks very different from
# the "no relationship" baseline — i.e., knowing someone's Gender genuinely
# shifts the probability distribution of their Preference.
#
# ⚠️ Check that all expected frequencies are >= 5.
# WHY: Chi-Square is an approximation that breaks down with very small cell
# counts. If any cell is < 5, use Fisher's Exact Test instead.
print("Expected frequencies:\n", expected)


# ──────────────────────────────────────────
# 4. Pearson vs Spearman Correlation
# Pearson (Linear, sensitive to outliers)
pearson_corr, pearson_p = stats.pearsonr(df['Age'], df['Income'])

# Spearman (Monotonic, robust to outliers)
spearman_corr, spearman_p = stats.spearmanr(df['Age'], df['Income'])

print(f"Pearson:  r={pearson_corr:.3f},  p={pearson_p:.4f}")
print(f"Spearman: r={spearman_corr:.3f}, p={spearman_p:.4f}")

# ✅ What to do with the result:
# |r| > 0.8  → Very strong → Multicollinearity risk, consider dropping one feature.
# |r| 0.5–0.8 → Moderate → Useful signal, worth keeping.
# |r| < 0.3  → Weak → Low value for linear models (may still help tree-based models).
#
# WHY multicollinearity matters:
# If two features move together (r > 0.8), they're telling the model the same
# story twice. In linear models this inflates coefficient variance and makes
# the model unstable. One of them is essentially redundant.
#
# Pearson ≈ Spearman → Relationship is linear. Pearson is reliable here.
# Pearson >> Spearman → A few outliers are pulling the linear correlation up.
# WHY: Pearson uses raw values, so extreme points have outsized weight.
# Spearman uses ranks, so a single extreme value only moves one rank.
# If they diverge, trust Spearman and investigate the outliers.

# Apply across entire DataFrame:
df.corr(method='pearson')
df.corr(method='spearman')


# ──────────────────────────────────────────
# 5. Point-Biserial Correlation (Continuous vs Binary)
# How strongly does a binary flag correlate with a continuous variable?
# e.g. Does being a Premium member correlate with spending?
df['IsPremium_num'] = df['IsPremium'].map({'Yes': 1, 'No': 0})
corr, p_value = stats.pointbiserialr(df['IsPremium_num'], df['Spending'])
print(f"Point-Biserial r={corr:.3f}, p={p_value:.4f}")

# ✅ What to do with the result:
# Interpret r just like Pearson — direction and magnitude both matter.
# Positive r → The "Yes" group has higher Spending on average.
# Negative r → The "Yes" group has lower Spending on average.
# p < 0.05   → This relationship is statistically real, not a fluke.
# p >= 0.05  → The spending difference between groups could just be random.
#
# WHY use Point-Biserial instead of just Pearson?
# Mathematically they're equivalent when one variable is binary (0/1).
# Using pointbiserialr() is just more explicit — it makes your intent
# clear to anyone reading the code, and avoids accidentally using Pearson
# on a non-binary categorical that was label-encoded as numbers.


# ──────────────────────────────────────────
# 6. Cramér's V (Categorical vs Categorical — Strength of Association)
# Chi-Square tells you IF an association exists. Cramér's V tells you HOW STRONG.
import numpy as np

contingency_table = pd.crosstab(df['Region'], df['ProductCategory'])
chi2, p, dof, _ = stats.chi2_contingency(contingency_table)
n = contingency_table.sum().sum()
cramers_v = np.sqrt(chi2 / (n * (min(contingency_table.shape) - 1)))
print(f"Cramér's V: {cramers_v:.3f}")

# ✅ What to do with the result:
# V ~0.1  → Weak   → Features are mostly independent, safe to use both.
# V ~0.3  → Moderate → Some overlap, but both features still add unique info.
# V ~0.5+ → Strong → Features are heavily redundant. Using both in a model
#                     risks feeding it the same information twice.
#
# WHY not just use Chi-Square for strength?
# Chi-Square is sensitive to sample size — a huge dataset will produce a
# significant p-value even for a trivially weak association.
# Cramér's V normalizes the chi2 statistic by sample size and table dimensions,
# giving you a 0–1 score that's comparable across datasets of any size.

# 7. Full EDA Correlation Heatmap (Quick Visual Summary)
import seaborn as sns
import matplotlib.pyplot as plt

corr_matrix = df.select_dtypes(include='number').corr()

plt.figure(figsize=(10, 8))
sns.heatmap(corr_matrix, annot=True, fmt=".2f", cmap='coolwarm',
            center=0, square=True, linewidths=0.5)
plt.title("Correlation Heatmap")
plt.tight_layout()
plt.show()

# ✅ What to do with the result:
# Dark red  (r → +1) → Strong positive: both features rise together.
# Dark blue (r → -1) → Strong negative: one rises as the other falls.
# Near white (r → 0) → No linear relationship.
#
# Feature pairs with |r| > 0.8 → Flag for multicollinearity (see #4 above).
# Features with high |r| to your target column → Prioritize these in modeling.
# Features with |r| ≈ 0 to target → May still be useful in non-linear models;
#   don't drop them based on correlation alone.
#
# WHY use a heatmap instead of just reading the numbers?
# With 10+ features the correlation matrix becomes a wall of decimals.
# The color encoding lets you instantly spot clusters of correlated features
# and strong target relationships that would take minutes to find manually.
