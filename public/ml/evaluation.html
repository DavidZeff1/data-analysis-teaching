Evaluating Classification Models

When predicting categories (e.g., fraud vs. not fraud, spam vs. not spam), simply looking at overall accuracy is often misleading. We need more nuanced metrics to understand how our model makes errors, especially when classes are imbalanced or when false positives and false negatives have different real-world costs.

Confusion Matrix, Precision, and Recall

The Confusion Matrix:

A table showing actual vs. predicted values. This represents the foundation of all classification metrics.

  • True Positives (TP): Model predicted positive, and it actually is positive.
  • True Negatives (TN): Model predicted negative, and it actually is negative.
  • False Positives (FP): Model predicted positive, but it's actually negative (Type I Error).
  • False Negatives (FN): Model predicted negative, but it's actually positive (Type II Error).

Precision

"Out of all the instances I flagged as positive, how many were actually positive?"

$$Precision = \frac{TP}{TP + FP}$$

Focus on minimizing False Positives (e.g., Spam filter - don't put regular emails into the spam folder).

Recall (aka Sensitivity)

"Out of all the actual positive instances, how many did I successfully flag?"

$$Recall = \frac{TP}{TP + FN}$$

Focus on minimizing False Negatives (e.g., Cancer screening - make sure you don't miss a sick patient).

Cutoff (Decision Threshold)

Most classifiers output a probability (e.g., 80% chance of fraud) rather than a hard label. By default, the cutoff is usually 0.5 (50%). However, you can change this cutoff to prioritize Precision over Recall or vice versa.

💡 Adjusting the Cutoff:

  • Higher cutoff (e.g., 0.8): Model is more conservative. It only predicts positive if it's very sure. Results in higher Precision, lower Recall.
  • Lower cutoff (e.g., 0.2): Model flags easily. Results in higher Recall, lower Precision.

ROC Curve and Area Under the Curve (AUC)

Visualizing Trade-offs

  • ROC (Receiver Operating Characteristic) Curve: A plot showing the True Positive Rate (Recall) vs False Positive Rate at different classification cutoffs.
  • AUC (Area Under the Curve): A single metric summarizing the entire ROC curve. It tells you the probability that a randomly chosen positive example is ranked more highly than a randomly chosen negative example.

AUC = 1.0: Perfect model.

AUC = 0.5: Meaningless model (equivalent to random guessing).

Expected Value (EV)

Machine Learning logic doesn't exist in a vacuum; business value is what matters. Expected Value assigns financial costs and benefits to your confusion matrix to determine the actual business impact of your model.

The EV Formula:

$$EV = P(TP) \times Cost(TP) + P(FP) \times Cost(FP) + P(TN) \times Cost(TN) + P(FN) \times Cost(FN)$$

By calculating the Expected Value at different cutoffs, you can find the optimal threshold that maximizes profit (or minimizes loss) rather than just maximizing abstract machine learning metrics.

Jupyter Notebook Examples

Below is an example showing how to calculate and visualize these metrics using Python and Scikit-Learn.

# Import necessary libraries
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import (confusion_matrix, classification_report, 
                             roc_curve, roc_auc_score, precision_score, recall_score)

# 1. Train a model and get predictions/probabilities
# Assuming X_train, X_test, y_train, y_test already exist
clf = RandomForestClassifier(random_state=42)
clf.fit(X_train, y_train)

# Predictions using the default 0.5 probability cutoff
y_pred = clf.predict(X_test)

# Probabilities (needed for custom cutoffs, ROC, and AUC)
y_prob = clf.predict_proba(X_test)[:, 1] # Probability of the positive class

# 2. Confusion Matrix & Basic Metrics
print("Confusion Matrix:\n", confusion_matrix(y_test, y_pred))
print("\nClassification Report:\n", classification_report(y_test, y_pred))

# 3. Changing the Cutoff
custom_cutoff = 0.3  # More eager to flag positives, increasing recall
y_pred_custom = (y_prob >= custom_cutoff).astype(int)

print(f"Recall at 0.5 cutoff: {recall_score(y_test, y_pred):.3f}")
print(f"Recall at {custom_cutoff} cutoff: {recall_score(y_test, y_pred_custom):.3f}")

# 4. Plotting the Confusion Matrix nicely
cm = confusion_matrix(y_test, y_pred_custom)
sns.heatmap(cm, annot=True, fmt="d", cmap="Blues")
plt.title(f"Confusion Matrix (Cutoff={custom_cutoff})")
plt.xlabel("Predicted Label")
plt.ylabel("True Label")
plt.show()

# 5. ROC Curve and AUC
auc = roc_auc_score(y_test, y_prob)
fpr, tpr, thresholds = roc_curve(y_test, y_prob)

plt.figure(figsize=(8, 6))
plt.plot(fpr, tpr, label=f"ROC Curve (AUC = {auc:.3f})")
plt.plot([0, 1], [0, 1], 'k--', label="Random Guessing (AUC = 0.5)")
plt.xlabel("False Positive Rate")
plt.ylabel("True Positive Rate (Recall)")
plt.title("Receiver Operating Characteristic (ROC)")
plt.legend()
plt.show()

# 6. Expected Value Calculation Example
# Suppose detecting fraud correctly saves $100 but investigating a false alarm costs $10
profit_matrix = np.array([
    [0, -10],  # True Negatives (TN) = $0 profit, False Positives (FP) = loss of $10
    [0,  100]  # False Negatives (FN) = $0 profit, True Positives (TP) = save $100
])

# Calculate Expected Value (EV) for the 0.3 cutoff
total_profit = np.sum(cm * profit_matrix)
print(f"Total Expected Profit at 0.3 cutoff: ${total_profit}")
