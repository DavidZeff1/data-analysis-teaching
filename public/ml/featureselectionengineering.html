1. The High-Level Workflow

Feature engineering and selection are not one-shot steps — they are an iterative loop that runs alongside model development. The order matters because some steps cause data leakage if done wrong.

  1. 1
    Understand the problem & target. Classification vs regression, business metric, class imbalance, time component.
  2. 2
    EDA. Distributions, missingness, cardinality, outliers, target relationship.
  3. 3
    Split the data first. Train/val/test (or CV folds) before any transformation that learns from data.
  4. 4
    Clean & impute. Decide a strategy per column; fit on train only.
  5. 5
    Engineer features. Encode categoricals, transform numerics, extract from text/datetime, build interactions.
  6. 6
    Select features. Filter → embedded → wrapper, in roughly that order of cost.
  7. 7
    Train, evaluate, interpret. Use CV, look at SHAP/permutation importance, sanity-check.
  8. 8
    Iterate. Errors and importances suggest new features; loop back to step 5.

Golden rule: any transformation whose parameters depend on the data (mean, std, target stats, vocabulary, learned encoder weights) must be fit on the training set only and then transform applied to validation/test. Use Pipeline to make this automatic.

2. Exploratory Data Analysis

Before touching any transformer, profile the data. You're answering: what types are these columns, how missing are they, how skewed, how do they relate to the target? 

import pandas as pd
import numpy as np

df = pd.read_csv("data.csv")

# Shape and dtypes
print(df.shape)
print(df.dtypes.value_counts())

# Missingness per column, sorted
miss = df.isna().mean().sort_values(ascending=False)
print(miss[miss > 0])

# Cardinality of object/categorical columns
cat_cols = df.select_dtypes(include=["object", "category"]).columns
print(df[cat_cols].nunique().sort_values(ascending=False))

# Numeric stats + skewness
num_cols = df.select_dtypes(include=np.number).columns
print(df[num_cols].describe().T)
print(df[num_cols].skew().sort_values(ascending=False))

# Target relationship (regression: corr; classification: groupby mean)

# print(df.groupby("target")[num_cols].mean())

3. Handling Missing Values

There is no universal best answer — the right strategy depends on why the value is missing (MCAR, MAR, MNAR) and how much is missing.

Strategy When to use Watch out
drop rows Few rows missing & MCAR Loses data, biases sample
drop column >60–80% missing or low signal Discarding info; consider a "is_missing" flag instead
mean / median Numeric, MCAR, low missingness Shrinks variance, distorts distribution
most_frequent / "Missing" Categorical Concentrates rare-category mass
KNNImputer Mixed numeric, modest dataset Slow, sensitive to scaling
IterativeImputer Missing values themselves predictable Experimental, costly
model handles it XGBoost, LightGBM, CatBoost Still add a missing indicator if missingness is informative 
from sklearn.impute import SimpleImputer, KNNImputer
from sklearn.experimental import enable_iterative_imputer  # noqa
from sklearn.impute import IterativeImputer

num_imputer = SimpleImputer(strategy="median", add_indicator=True)
cat_imputer = SimpleImputer(strategy="constant", fill_value="Missing")

# add_indicator=True appends a 0/1 column flagging the imputed rows —
# extremely useful when missingness is itself a signal.
X_train_num = num_imputer.fit_transform(X_train[num_cols])
X_test_num  = num_imputer.transform(X_test[num_cols])

4. Encoding Categorical Features

The right encoder depends on the cardinality, whether the categories are ordinal, and the model family.

One-Hot Encoding

Default for low-cardinality nominal features (≲ 15 levels) with linear / distance-based models. 

from sklearn.preprocessing import OneHotEncoder
ohe = OneHotEncoder(
    handle_unknown="ignore",
    sparse_output=False,
    drop="if_binary",
)

Ordinal Encoding

When order is meaningful (sizes, ratings) or when feeding tree-based models. 

from sklearn.preprocessing import OrdinalEncoder
oe = OrdinalEncoder(
    categories=[["S","M","L","XL"]],
    handle_unknown="use_encoded_value",
    unknown_value=-1,
)

Target / Mean Encoding

High-cardinality features (zip codes, item ids). Replace each level with a smoothed estimate of the target. High leakage risk — use CV-fold encoding. 

from category_encoders import TargetEncoder
te = TargetEncoder(smoothing=10.0)
te.fit(X_train[["zip"]], y_train)
X_train_enc = te.transform(X_train[["zip"]])

Frequency / Count Encoding

Replace each category with how often it appears. Cheap, handles new categories at zero, surprisingly effective for trees. 

freq = X_train["city"].value_counts(normalize=True)
X_train["city_freq"] = X_train["city"].map(freq)
X_test["city_freq"]  = X_test["city"].map(freq).fillna(0)

Hashing Encoder

Massive cardinality, online/streaming setting. Trades collisions for fixed memory. 

from sklearn.feature_extraction import FeatureHasher
fh = FeatureHasher(n_features=64, input_type="string")

Native handling

CatBoost handles categoricals internally with ordered target stats. LightGBM accepts a categorical_feature argument. Skip encoding entirely when possible.

Avoid: label-encoding nominal categories before feeding a linear / KNN / NN model — you're inventing an order that doesn't exist.

5. Scaling & Normalization

Required for linear models, SVMs, KNN, k-means, PCA, and neural nets. Not needed for tree-based models (RF, GBM, XGBoost, LightGBM).

Scaler Output Best for
StandardScaler mean 0, std 1 Roughly Gaussian features
MinMaxScaler [0, 1] Bounded features, NN inputs
RobustScaler median 0, IQR 1 Heavy outliers
MaxAbsScaler [-1, 1], preserves zeros Sparse data
Normalizer unit row norm Cosine-similarity setups, text 
from sklearn.preprocessing import StandardScaler, RobustScaler

scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)   # never refit on test

6. Numerical Transformations

Log / Box-Cox / Yeo-Johnson

Tame right-skewed distributions (incomes, counts, prices). log1p handles zeros; Yeo-Johnson handles negatives. 

from sklearn.preprocessing import PowerTransformer
pt = PowerTransformer(method="yeo-johnson", standardize=True)
X_train_t = pt.fit_transform(X_train[skewed_cols])

Binning / Discretization

Convert a continuous feature into ordinal buckets. Helps linear models capture non-monotone effects; usually pointless for trees. 

from sklearn.preprocessing import KBinsDiscretizer
binner = KBinsDiscretizer(n_bins=5, encode="ordinal", strategy="quantile")
X_train["age_bin"] = binner.fit_transform(X_train[["age"]])

Outlier handling

Cap (winsorize) extreme tails or remove rows beyond IQR or z-score thresholds. Don't blindly delete — check whether outliers are signal. 

q_lo, q_hi = X_train["price"].quantile([0.01, 0.99])
X_train["price"] = X_train["price"].clip(q_lo, q_hi)
X_test["price"]  = X_test["price"].clip(q_lo, q_hi)   # use train bounds

7. Datetime Features

A raw timestamp is rarely useful. Decompose it, and remember that cyclical components (hour, weekday, month) should be encoded as sin/cos pairs so the model learns that 23:00 is close to 00:00. 

import numpy as np

df["ts"] = pd.to_datetime(df["ts"])

df["year"]      = df["ts"].dt.year
df["month"]     = df["ts"].dt.month
df["day"]       = df["ts"].dt.day
df["dayofweek"] = df["ts"].dt.dayofweek
df["hour"]      = df["ts"].dt.hour
df["is_weekend"]= (df["dayofweek"] >= 5).astype(int)
df["quarter"]   = df["ts"].dt.quarter
df["days_since_signup"] = (df["ts"] - df["signup_ts"]).dt.days

# Cyclical encoding
df["hour_sin"] = np.sin(2*np.pi*df["hour"]/24)
df["hour_cos"] = np.cos(2*np.pi*df["hour"]/24)
df["mon_sin"]  = np.sin(2*np.pi*df["month"]/12)
df["mon_cos"]  = np.cos(2*np.pi*df["month"]/12)

# Time-series specific (per group)
df = df.sort_values(["user_id", "ts"])
df["amount_lag1"]   = df.groupby("user_id")["amount"].shift(1)
df["amount_roll7"]  = df.groupby("user_id")["amount"].shift(1).rolling(7).mean()
Time-series leak alert: always .shift(1) before rolling/expanding aggregations so a row never sees its own target. Split chronologically with TimeSeriesSplit, never random.

9. Interactions, Ratios & Polynomials

Linear models benefit hugely from manually crafted interactions (trees discover them automatically). Domain knowledge wins here. 

# Domain-driven (always try these first)
df["price_per_sqm"]   = df["price"] / df["area_sqm"]
df["debt_to_income"]  = df["debt"] / (df["income"] + 1)
df["age_x_income"]    = df["age"] * df["income"]

# Brute-force polynomial expansion (use sparingly)
from sklearn.preprocessing import PolynomialFeatures
poly = PolynomialFeatures(degree=2, interaction_only=True, include_bias=False)
X_poly = poly.fit_transform(X_train[["x1","x2","x3"]])
# columns explode quickly — pair with feature selection

Group-level aggregations are another rich source: per-customer mean spend, per-zip default rate (computed on train only and merged in).

10. Feature Selection Methods

Three families, used in roughly increasing order of cost and accuracy.

Filter

Cheap · Model-agnostic

Score each feature by a univariate statistic and drop the bottom N.

Embedded

Built into the model

L1 regularization, tree feature importance — selection happens during training.

Wrapper

Expensive · Most accurate

Train a model on different subsets and pick the best. RFE, sequential search.

10.1 Filter methods 

from sklearn.feature_selection import (
    VarianceThreshold, SelectKBest,
    f_classif, f_regression,
    chi2, mutual_info_classif, mutual_info_regression,
)

# Drop near-constant features
vt = VarianceThreshold(threshold=0.01)
X_train_v = vt.fit_transform(X_train)

# ANOVA F-test (numeric features, classification target)
skb = SelectKBest(score_func=f_classif, k=30)
X_train_k = skb.fit_transform(X_train, y_train)

# Mutual information — captures non-linear dependence
mi = mutual_info_classif(X_train, y_train, random_state=0)

# Drop highly correlated pairs (numeric only)
import numpy as np
corr = X_train.corr().abs()
upper = corr.where(np.triu(np.ones(corr.shape), k=1).astype(bool))
to_drop = [c for c in upper.columns if any(upper[c] > 0.95)]
X_train = X_train.drop(columns=to_drop)

10.2 Embedded methods 

from sklearn.linear_model import LassoCV, LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.feature_selection import SelectFromModel

# L1 regularization shrinks irrelevant coefficients to zero
lasso = LassoCV(cv=5, random_state=0).fit(X_train_s, y_train)
selector = SelectFromModel(lasso, prefit=True, threshold="median")
X_train_sel = selector.transform(X_train_s)

# Tree-based importance
rf = RandomForestClassifier(n_estimators=500, n_jobs=-1, random_state=0)
rf.fit(X_train, y_train)
importances = pd.Series(rf.feature_importances_, index=X_train.columns)
importances.sort_values(ascending=False).head(20)

10.3 Wrapper methods 

from sklearn.feature_selection import RFECV, SequentialFeatureSelector
from sklearn.linear_model import LogisticRegression

# Recursive feature elimination with cross-validation
rfecv = RFECV(
    estimator=LogisticRegression(max_iter=1000),
    step=1, cv=5, scoring="roc_auc", n_jobs=-1,
)
rfecv.fit(X_train_s, y_train)
print("optimal #features:", rfecv.n_features_)

# Sequential forward selection (greedy)
sfs = SequentialFeatureSelector(
    LogisticRegression(max_iter=1000),
    n_features_to_select=20,
    direction="forward",
    scoring="roc_auc", cv=5, n_jobs=-1,
)
sfs.fit(X_train_s, y_train)

10.4 Permutation importance & SHAP

After fitting your candidate model, these give a model-faithful view of what actually matters. Permutation importance measures the drop in score when a feature is shuffled; SHAP decomposes individual predictions. 

from sklearn.inspection import permutation_importance
result = permutation_importance(
    model, X_val, y_val,
    n_repeats=10, random_state=0, n_jobs=-1, scoring="roc_auc",
)
imp = pd.Series(result.importances_mean, index=X_val.columns).sort_values()

Practical recipe: drop near-constants and highly correlated columns (filter), fit a regularized linear model and a tree-based model and look at both rankings (embedded), confirm with permutation importance on a validation set. Wrapper methods only when you need the last 1–2% of accuracy.

11. Putting It in a Pipeline

A scikit-learn Pipeline + ColumnTransformer is the only sane way to keep all of this leak-free, reproducible, and deployable. 

import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.feature_selection import SelectFromModel
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import GradientBoostingClassifier

# 1. Split first
X = df.drop(columns="target")
y = df["target"]
X_tr, X_te, y_tr, y_te = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

num_cols = X.select_dtypes(include=np.number).columns.tolist()
cat_cols = X.select_dtypes(include=["object", "category"]).columns.tolist()

# 2. Per-column-type preprocessing
num_pipe = Pipeline([
    ("impute", SimpleImputer(strategy="median", add_indicator=True)),
    ("scale",  StandardScaler()),
])
cat_pipe = Pipeline([
    ("impute", SimpleImputer(strategy="constant", fill_value="Missing")),
    ("ohe",    OneHotEncoder(handle_unknown="ignore", sparse_output=False)),
])

preprocess = ColumnTransformer([
    ("num", num_pipe, num_cols),
    ("cat", cat_pipe, cat_cols),
], remainder="drop")

# 3. Selection + model in the same pipeline
clf = Pipeline([
    ("prep",   preprocess),
    ("select", SelectFromModel(
        LogisticRegression(penalty="l1", solver="liblinear", C=0.5),
    )),
    ("model",  GradientBoostingClassifier(random_state=0)),
])

# 4. Cross-validate — the whole pipeline is refit per fold
scores = cross_val_score(clf, X_tr, y_tr, cv=5, scoring="roc_auc", n_jobs=-1)
print(f"CV ROC-AUC: {scores.mean():.3f} ± {scores.std():.3f}")

# 5. Fit on all training data, evaluate once on the held-out test
clf.fit(X_tr, y_tr)
print("Test ROC-AUC:", clf.score(X_te, y_te))

Tune everything jointly with GridSearchCV / RandomizedSearchCV — including imputer strategy, scaler, number of selected features, and model hyperparameters. Reference them as "prep__num__impute__strategy", "select__threshold", etc.

12. Common Pitfalls & Data Leakage

⚠️ Fitting on the full dataset before splitting

Calling scaler.fit(X) before train_test_split leaks test statistics into training. Always split first.

⚠️ Target encoding without out-of-fold

Plain mean encoding leaks the target. Use K-fold target encoding (encode each fold using stats computed on the other folds) or category_encoders.TargetEncoder with smoothing inside a CV pipeline.

⚠️ Selecting features on the entire dataset

Running SelectKBest on all data and then doing CV over what's left is a classic leak. Put selection inside the pipeline.

⚠️ Using future information in time series

Rolling means, group statistics, or random splits on time-ordered data all leak the future. Use TimeSeriesSplit and lag/shift before any rolling op.

⚠️ Imputing with target-derived statistics

Don't fill missing values using group means computed from the target on the full dataset.

⚠️ Overengineering

Hundreds of polynomial / interaction features can overfit and slow training. Always validate that a new feature actually moves CV score.

⚠️ Ignoring train/test distribution drift

Categories present only in test crash a fitted OneHotEncoder unless you set handle_unknown="ignore". Validate that key feature distributions match.

13. Final Checklist

  • ✓ Data was split before any fitting.
  • ✓ Every transformer is wrapped in a Pipeline / ColumnTransformer.
  • ✓ Missing values handled with a documented strategy per column type.
  • ✓ Categorical encoder chosen for the model family (OHE for linear/NN, ordinal/native for trees, target for high-cardinality).
  • ✓ Numeric features scaled (linear/SVM/KNN/NN) — skipped (trees).
  • ✓ Skewed features transformed (log / Yeo-Johnson) when relevant.
  • ✓ Datetime decomposed and cyclical components encoded as sin/cos.
  • ✓ Domain-driven interactions / ratios attempted before brute-force polynomials.
  • ✓ Near-constant and highly correlated features removed.
  • ✓ Feature selection cross-validated, importance verified with permutation / SHAP.
  • ✓ No target leakage; CV scheme matches the production setting (time-aware if temporal).
  • ✓ The full pipeline can be pickled and reapplied to new raw data with one call.
