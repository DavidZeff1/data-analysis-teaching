Python Basics: Data Types & Operations

Python's Core Data Types

Understanding Python's built-in data types is fundamental. Each type has different characteristics that determine when and how to use them.

Data Type Comparison

Type Ordered Mutable Duplicates Hashable Empty Use Case
list ‚úì ‚úì ‚úì ‚úó [] General-purpose, mutable sequence
tuple ‚úì ‚úó ‚úì ‚úì* () Fixed data, dict keys, function returns
dict ‚úì** ‚úì Keys: ‚úó ‚úó {} Key-value mapping, fast lookup
set ‚úó ‚úì ‚úó ‚úó set() Unique items, membership testing
str ‚úì ‚úó ‚úì ‚úì "" Text data

* Tuple is hashable only if all its elements are hashable. ** Dict maintains insertion order (Python 3.7+)

üí° Key Concepts:

  • Ordered: Elements maintain their position/sequence
  • Mutable: Can be changed after creation (add, remove, modify)
  • Duplicates: Can contain the same value multiple times
  • Hashable: Can be used as dictionary keys or in sets (requires immutability)

1. List

Ordered, mutable sequence. The most versatile and commonly used data structure.

Creating Lists
# Empty list
empty = []
empty = list()

# With values
nums = [1, 2, 3, 4, 5]
mixed = [1, "hello", 3.14, True]
nested = [[1, 2], [3, 4]]

# From other iterables
from_string = list("hello")        # ['h', 'e', 'l', 'l', 'o']
from_range = list(range(5))        # [0, 1, 2, 3, 4]
from_tuple = list((1, 2, 3))       # [1, 2, 3]

# List comprehension
squares = [x**2 for x in range(5)]  # [0, 1, 4, 9, 16]

# Repeat elements
zeros = [0] * 5                   # [0, 0, 0, 0, 0]

2. Tuple

Ordered, immutable sequence. Use for fixed collections, function returns, and dictionary keys.

Creating Tuples
# Empty tuple
empty = ()
empty = tuple()

# With values (parentheses optional)
point = (10, 20)
point = 10, 20           # Same thing
rgb = (255, 128, 0)

# Single element (need comma!)
single = (42,)            # tuple
not_tuple = (42)          # just int 42!

# From other iterables
from_list = tuple([1, 2, 3])
from_string = tuple("abc")   # ('a', 'b', 'c')

# Tuple unpacking
x, y = (10, 20)
a, b, c = 1, 2, 3

# Swap values
a, b = b, a
Tuple Methods (only 2!)
Method Description Returns Example
count(x) Count occurrences of x int (1,2,2,3).count(2) ‚Üí 2
index(x) Find index of first x int (1,2,3).index(2) ‚Üí 1

Tuples are immutable, so no methods for adding/removing elements. Use all the same operations as list (indexing, slicing, len, in, +, etc.)

When to use tuple instead of list:

  • ‚Ä¢ Data shouldn't change: coordinates, RGB colors, database records
  • ‚Ä¢ Dictionary keys: tuples are hashable (if elements are)
  • ‚Ä¢ Function returns: return multiple values
  • ‚Ä¢ Slight performance: tuples are faster and use less memory

3. Dictionary (dict)

Key-value pairs with O(1) lookup. The most important data structure for real-world programming.

Creating Dictionaries
# Empty dict
empty = {}
empty = dict()

# With values
person = {"name": "Alice", "age": 30}
scores = {"math": 95, "english": 88}

# From list of tuples
pairs = [("a", 1), ("b", 2)]
d = dict(pairs)              # {'a': 1, 'b': 2}

# From zip
keys = ["a", "b", "c"]
vals = [1, 2, 3]
d = dict(zip(keys, vals))    # {'a': 1, 'b': 2, 'c': 3}

# Dict comprehension
squares = {x: x**2 for x in range(5)}  # {0:0, 1:1, 2:4, 3:9, 4:16}

# From keys with default value
d = dict.fromkeys(["a", "b"], 0)  # {'a': 0, 'b': 0}
Accessing Values
d = {"name": "Alice", "age": 30}

# Direct access (raises KeyError if missing)
d["name"]             # "Alice"
d["city"]             # KeyError!

# Safe access with .get() (returns None or default)
d.get("name")          # "Alice"
d.get("city")          # None
d.get("city", "Unknown")  # "Unknown"

# Set value
d["city"] = "NYC"     # Add new key
d["age"] = 31         # Update existing

Common Patterns
# Iterate

for key, val in d.items(): # key-value pairs

# Membership (checks keys)
"name" in d               # True

# Merge dicts (Python 3.9+)
merged = d1 | d2            # d2 values override d1

# Merge dicts (Python 3.5+)
merged = {**d1, **d2}

# Count items
from collections import Counter
counts = Counter(["a", "b", "a"])  # {'a': 2, 'b': 1}

# Default values
from collections import defaultdict
dd = defaultdict(list)
dd["key"].append(1)        # No KeyError!

4. Set

Unordered collection of unique elements. Fast membership testing and set operations.

Creating Sets
# Empty set (NOT {} - that's a dict!)
empty = set()

# With values
nums = {1, 2, 3}
unique = {1, 2, 2, 3}      # {1, 2, 3} - duplicates removed

# From other iterables
from_list = set([1, 2, 2, 3])  # {1, 2, 3}
from_string = set("hello")     # {'h', 'e', 'l', 'o'}

# Set comprehension
evens = {x for x in range(10) if x % 2 == 0}

# Frozen set (immutable, hashable)
fs = frozenset([1, 2, 3])

Indexing and Slicing

Works on list, tuple, and string. Powerful way to access elements.

Index Reference

s = "Python"

P y t h o n

s = "Python"
lst = [0, 1, 2, 3, 4, 5]

# INDEXING: Get single element
s[0]      # 'P' - first
s[-1]     # 'n' - last
s[-2]     # 'o' - second to last

# SLICING: [start:stop:step] - stop is exclusive!
s[0:3]    # 'Pyt' - indices 0, 1, 2
s[:3]     # 'Pyt' - from start to 3
s[3:]     # 'hon' - from 3 to end
s[:]      # 'Python' - copy entire

# Negative indices in slicing
s[-3:]    # 'hon' - last 3
s[:-2]    # 'Pyth' - all except last 2
s[1:-1]   # 'ytho' - exclude first and last

# Step
s[::2]    # 'Pto' - every 2nd character
s[::-1]   # 'nohtyP' - REVERSE!
lst[::-1] # [5, 4, 3, 2, 1, 0]

# Modify list via slicing (lists only, not strings)
lst[1:3] = [10, 20]  # [0, 10, 20, 3, 4, 5]
lst[1:3] = []        # Delete: [0, 3, 4, 5]

Comprehensions

Concise way to create lists, dicts, and sets. More Pythonic than loops.

# LIST COMPREHENSION
[x**2 for x in range(5)]              # [0, 1, 4, 9, 16]
[x for x in range(10) if x % 2 == 0]  # [0, 2, 4, 6, 8] - with filter
[x.upper() for x in ["a", "b"]]      # ['A', 'B']

# Nested loops
[(x, y) for x in [1,2] for y in [3,4]]  # [(1,3),(1,4),(2,3),(2,4)]

# Flatten nested list
nested = [[1,2], [3,4]]
[x for sublist in nested for x in sublist]  # [1, 2, 3, 4]

# If-else (ternary)
["even" if x%2==0 else "odd" for x in range(4)]

# DICT COMPREHENSION
{x: x**2 for x in range(5)}          # {0:0, 1:1, 2:4, 3:9, 4:16}
{k: v for k, v in zip(keys, vals)}  # from two lists
{k: v for k, v in d.items() if v > 0} # filter dict

# SET COMPREHENSION
{x % 3 for x in range(10)}           # {0, 1, 2}

# GENERATOR EXPRESSION (lazy, memory efficient)
gen = (x**2 for x in range(1000000))  # doesn't create list in memory
sum(x**2 for x in range(10))         # use directly in functions
