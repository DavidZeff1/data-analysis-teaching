Series & Initialization

Pandas has two core types: a Series (a single column of data with an index) and a DataFrame (a table made of multiple Series). Think of a Series as one column in a spreadsheet, and a DataFrame as the entire spreadsheet.

  Series (1D)              DataFrame (2D)
  ┌───────┐              ┌──────┬─────┬───────┐
  │ Alice │  ← one       │ name │ age │ city  │
  │ Bob   │    column    ├──────┼─────┼───────┤
  │ Carol │    of data   │ Alice│  25 │ NYC   │
  └───────┘              │ Bob  │  30 │ LA    │
    ↑ index              │ Carol│  35 │ Chicago│
    is on the            └──────┴─────┴───────┘
    left side              ↑ multiple Series side by side

Creating a Series

# From a list
s = pd.Series([10, 20, 30])
# 0    10
# 1    20
# 2    30

# From a list with custom index
s = pd.Series([10, 20, 30], index=['a', 'b', 'c'])
# a    10
# b    20
# c    30

# From a dictionary (keys become the index)
s = pd.Series({'apples': 3, 'bananas': 5, 'oranges': 2})

# From a scalar (broadcasts to all index values)
s = pd.Series(0, index=['a', 'b', 'c'])
# a    0
# b    0
# c    0

# Name your Series (useful inside DataFrames)
s = pd.Series([10, 20, 30], name='scores')

Creating a DataFrame

# From a dictionary of lists (most common)
df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Carol'],
    'age': [25, 30, 35],
    'city': ['NYC', 'LA', 'Chicago']
})

# From a list of dictionaries (each dict = one row)
df = pd.DataFrame([
    {'name': 'Alice', 'age': 25},
    {'name': 'Bob', 'age': 30},
    {'name': 'Carol', 'age': 35}
])

# From a numpy array
df = pd.DataFrame(
    np.array([[1, 2], [3, 4], [5, 6]]),
    columns=['A', 'B']
)

# Empty DataFrame with defined columns
df = pd.DataFrame(columns=['name', 'age', 'city'])

Reading Data from Files

# CSV (most common)
df = pd.read_csv('data.csv')
df = pd.read_csv('data.csv', sep=';')              # custom delimiter
df = pd.read_csv('data.csv', encoding='latin1')       # encoding issues
df = pd.read_csv('data.csv', parse_dates=['date'])    # auto-parse dates
df = pd.read_csv('data.csv', nrows=100)               # read only first 100 rows

# Excel
df = pd.read_excel('data.xlsx')
df = pd.read_excel('data.xlsx', sheet_name='Sheet2')

# JSON
df = pd.read_json('data.json')

# From clipboard (paste from Excel/Google Sheets)
df = pd.read_clipboard()

# Save back to file
df.to_csv('output.csv', index=False)     # index=False avoids extra column
df.to_excel('output.xlsx', index=False)

DataFrame Structure (Index, Columns, Values)

Every DataFrame has 3 parts: column names (top), index (side), and values (the data).

                COLUMNS (df.columns)
                   ↓    ↓    ↓
                 name   age  city
               +------+-----+-------+
INDEX    0     | Alice|  25 | NYC   |  ← VALUES
(df.index)     +------+-----+-------+
         1     | Bob  |  30 | LA    |  ← (df.values)
               +------+-----+-------+
         2     | Carol|  35 | Chicago|
               +------+-----+-------+
# Access the 3 parts
df.columns              # Index(['name', 'age', 'city'])
df.index                # RangeIndex(0, 1, 2) or custom labels
df.values               # 2D numpy array of data

# Access data by COLUMN (most common)
df['name']               # get column → returns Series
df[['name', 'age']]      # get multiple columns → returns DataFrame

# Access data by INDEX (row label)
df.loc[0]               # get row by label → returns Series
df.loc[[0, 2]]          # get multiple rows → returns DataFrame

# Access specific cell
df.loc[0, 'name']       # row 0, column 'name' → 'Alice'

# Access by position (not label)
df.iloc[0]              # first row
df.iloc[0, 1]           # first row, second column
Method Accesses by Example
df['col'] Column name df['age']
df.loc[label] Index label df.loc[0] or df.loc['row_a']
df.iloc[pos] Integer position df.iloc[0] (always first row)
df.loc[row, col] Both df.loc[0, 'name'] 
# After .corr().stack() → MultiIndex Series
corr_matrix = df.corr()
#          A     B
#     A   1.0   0.5
#     B   0.5   1.0

stacked = corr_matrix.stack()
# A  A    1.0   ← index is now (row, col) tuple
#    B    0.5
# B  A    0.5
#    B    1.0

stacked.index              # MultiIndex with 2 levels
stacked[('A', 'B')]        # access by tuple → 0.5
stacked[stacked != 1]      # filter by value
💡 .loc[] uses labels, .iloc[] uses positions. When index is 0,1,2... they look the same, but with custom index labels they differ!

Selecting Columns

Think of this as choosing which columns from a spreadsheet you care about. You're narrowing your view to just the relevant data — like picking only the columns you'd include in a report.

# Select specific columns
df[['col1', 'col2']]

# Select single column (returns Series)
df['col']

# Select single column (returns DataFrame)
df[['col']]

# Rename columns
df.rename(columns={'old': 'new'})

# Get unique values
df['col'].unique()              # returns array
df[['col']].drop_duplicates()   # returns DataFrame
💡 Single brackets df['col'] → Series. Double brackets df[['col']] → DataFrame.

Filtering Rows

Filtering is like a bouncer at the door — you set the rules and only the rows that pass get through. This is equivalent to SQL's WHERE clause. You create a True/False mask and pandas keeps only the True rows.

# Simple filter
df[df['Total'] > 10]

# Multiple conditions (AND) - use &
df[(df['Total'] > 10) & (df['Country'] == 'USA')]

# Multiple conditions (OR) - use |
df[(df['Country'] == 'USA') | (df['Country'] == 'Canada')]

# Check if value in list
df[df['Country'].isin(['USA', 'Canada'])]

# Range check
df[df['Total'].between(5, 15)]

# Pattern matching
df[df['Email'].str.contains('gmail.com')]
df[df['Name'].str.startswith('A')]

# Null checks
df[df['Company'].isna()]       # is null
df[df['Company'].notna()]     # is not null

# Negation - use ~
df[~df['Country'].isin(['USA', 'Canada'])]
💡 Always wrap each condition in parentheses when combining with & or |. Use & for AND, | for OR, ~ for NOT.

Sorting & Limiting

Sorting rearranges your rows by value (like SQL's ORDER BY), and limiting lets you grab just the first/last N rows. Together they answer questions like "what are the top 10 highest sales?"

# Sort ascending
df.sort_values('Total')

# Sort descending
df.sort_values('Total', ascending=False)

# Sort by multiple columns
df.sort_values(['Country', 'Total'], ascending=[True, False])

# Get first/last N rows
df.head(10)
df.tail(10)

# Get top/bottom N by value
df.nlargest(10, 'Total')
df.nsmallest(10, 'Total')

# Slice rows by position
df.iloc[5:15]
💡 .nlargest() and .nsmallest() are optimized for "top N" queries.

Aggregation (groupby)

Groupby is "split-apply-combine": split your data into groups (by country, by category), apply a calculation to each group (sum, mean, count), then combine the results into a summary table. It's SQL's GROUP BY equivalent.

# Basic groupby with named aggregations
df.groupby('Country').agg(
    Count=('Total', 'count'),
    TotalSales=('Total', 'sum'),
    AvgSale=('Total', 'mean')
).reset_index()

# Group by multiple columns
df.groupby(['Country', 'City']).agg(
    AvgSale=('Total', 'mean')
).reset_index()

# Quick aggregations
df.groupby('Country')['Total'].sum()
df.groupby('Country')['Total'].mean()

# Multiple aggregations on same column
df.agg({'Total': ['count', 'sum', 'mean', 'min', 'max']})
💡 .reset_index() converts the grouped result back to a regular DataFrame. Without it, group columns become the index.

Counting Methods

"How many?" is one of the most common questions in data analysis, but there are many flavors: how many rows? How many non-null values? How many unique values? How many of each? Each method answers a different version of that question — and they behave differently with nulls.

# .size() - counts ALL rows including nulls
df.groupby('class').size()

# .count() - counts non-null values per column
df.groupby('class').count()

# .nunique() - counts unique values
df.groupby('class')['student'].nunique()

# .value_counts() - frequency of each value
df['Country'].value_counts()
df.groupby('class')['score'].value_counts()
Method Question it answers Counts nulls?
.size() How many rows? Yes
.count() How many non-null values per column? No
.nunique() How many unique values? No
.value_counts() How many of each specific value? No

Filtering Groups

Sometimes you don't just want to filter individual rows — you want to filter entire groups. For example: "show me only customers who have placed 5+ orders." This is SQL's HAVING clause. You aggregate first, then apply a condition to the summary.

# Filter after grouping (aggregate first, then filter)
grouped = df.groupby('Country').agg(
    TotalSales=('Total', 'sum')
).reset_index()
grouped[grouped['TotalSales'] > 100]

# Filter groups by count
grouped = df.groupby('CustomerId').agg(
    Orders=('InvoiceId', 'count')
).reset_index()
grouped[grouped['Orders'] >= 5]

# One-liner with .filter() - returns original rows
df.groupby('CustomerId').filter(lambda x: len(x) >= 5)
💡 .filter() returns the original rows (not aggregated), while aggregating then filtering returns summary rows.

Transform (Group Calculations Back to Rows)

transform() applies a function to groups but returns results aligned with the original index — same shape in, same shape out.

# Add group average to each row
df['CountryAvg'] = df.groupby('Country')['Total'].transform('mean')

# Percent of group total
df['PctOfCountry'] = df['Total'] / df.groupby('Country')['Total'].transform('sum')

# Filter: keep only groups with 5+ members
df[df.groupby('Country')['Total'].transform('count') >= 5]

# Normalize within groups (z-score)
df['Normalized'] = df.groupby('Country')['Total'].transform(
    lambda x: (x - x.mean()) / x.std()
)

# Fill NaN with group mean
df['Total'] = df.groupby('Country')['Total'].transform(
    lambda x: x.fillna(x.mean())
)
💡 agg() collapses groups into summary rows. transform() broadcasts group results back to every original row.

Binning (cut & qcut)

Binning converts continuous numbers into discrete categories — like turning exact ages into age groups (0-17, 18-35, 36-60, 60+). Think of it as putting values into labeled buckets. cut makes buckets of equal width, qcut makes buckets with equal number of items.

# cut: Equal-width bins (by value range)
df['PriceTier'] = pd.cut(
    df['Total'],
    bins=[0, 10, 20, float('inf')],
    labels=['Low', 'Medium', 'High']
)

# cut: Auto-create n equal-width bins
df['PriceTier'] = pd.cut(df['Total'], bins=3, labels=['Low', 'Medium', 'High'])

# qcut: Equal-frequency bins (by count)
df['Quartile'] = pd.qcut(df['Total'], q=4, labels=['Q1', 'Q2', 'Q3', 'Q4'])

# qcut: Custom percentiles
df['Tier'] = pd.qcut(
    df['Total'],
    q=[0, 0.25, 0.75, 1],
    labels=['Bottom 25%', 'Middle', 'Top 25%']
)
pd.cut() pd.qcut()
Bins by Value range (equal width) Data distribution (equal count)
Use when Boundaries matter (age groups, price ranges) Even groups matter (percentiles, quartiles)

Merging DataFrames

Merging is how you combine two tables that share a common column — exactly like SQL JOINs. Think of it as "looking up" information from one table and attaching it to another. For example, merging a customers table with an orders table using customer ID.

# Inner join (only matching rows)
pd.merge(df1, df2, on='key', how='inner')

# Left join (keep all from left)
pd.merge(df1, df2, on='key', how='left')

# Right join (keep all from right)
pd.merge(df1, df2, on='key', how='right')

# Outer join (keep all from both)
pd.merge(df1, df2, on='key', how='outer')

# Multiple join keys
pd.merge(df1, df2, on=['key1', 'key2'])

# Different column names
pd.merge(df1, df2, left_on='cust_id', right_on='customer_id')
💡 Default is inner join. Use how= to specify: 'inner', 'left', 'right', 'outer'.

Concatenating DataFrames

Unlike merging (which matches rows by a key), concatenating is simple stacking — gluing tables together either vertically (more rows) or horizontally (more columns). Think: "I have January data and February data in separate tables, and I want to combine them into one."

# Stack vertically (add rows)
pd.concat([df1, df2])

# Stack vertically and remove duplicates
pd.concat([df1, df2]).drop_duplicates()

# Stack horizontally (add columns)
pd.concat([df1, df2], axis=1)

Conditional Logic

This is how you create new columns based on rules — like SQL's CASE WHEN. "If total > 10, label it 'High', otherwise 'Low'." Use np.where for simple if/else and np.select when you have multiple conditions.

# Simple if/else (2 outcomes)
df['Category'] = np.where(df['Total'] > 10, 'High', 'Low')

# Multiple conditions
conditions = [
    df['Total'] > 15,
    df['Total'] > 5
]
choices = ['High', 'Medium']
df['Category'] = np.select(conditions, choices, default='Low')

# Alternative: pd.cut() for ranges
df['Category'] = pd.cut(
    df['Total'],
    bins=[0, 5, 15, float('inf')],
    labels=['Low', 'Medium', 'High']
)
💡 np.where() for simple if/else. np.select() for multiple conditions. pd.cut() for numeric ranges.

Renaming Columns

Real-world datasets often come with messy column names — spaces, inconsistent casing, abbreviations. Renaming columns is one of the first things you do to make your data easier to work with. Think of it as putting clear labels on every column so your code reads like plain English.

# Rename specific columns (dict mapping)
df.rename(columns={'old_name': 'new_name'})
df.rename(columns={
    'CustomerID': 'customer_id',
    'Total Price': 'total_price'
})

# Rename ALL columns at once (provide list in same order)
df.columns = ['id', 'name', 'age', 'city']

# String operations on ALL column names
df.columns = df.columns.str.strip()              # Remove whitespace
df.columns = df.columns.str.lower()              # Lowercase
df.columns = df.columns.str.upper()              # Uppercase
df.columns = df.columns.str.replace(' ', '_')     # Replace spaces with underscores

# Chain multiple transformations
df.columns = df.columns.str.strip().str.lower().str.replace(' ', '_')

# Remove special characters
df.columns = df.columns.str.replace(r'[^a-zA-Z0-9_]', '', regex=True)

# Add prefix/suffix to all columns
df.add_prefix('col_')                        # col_A, col_B, col_C
df.add_suffix('_2024')                      # A_2024, B_2024, C_2024

# Rename using a function
df.rename(columns=lambda x: x.strip().lower())
df.rename(columns=str.upper)

# Rename by index position
df.rename(columns={df.columns[0]: 'first_col'})

# Rename inplace (modifies original DataFrame)
df.rename(columns={'old': 'new'}, inplace=True)
💡 Common cleaning pattern: 
# Clean messy column names in one go
df.columns = (df.columns
              .str.strip()                # Remove leading/trailing spaces
              .str.lower()                # Lowercase everything
              .str.replace(' ', '_')      # Spaces → underscores
              .str.replace(r'[^\w]', '', regex=True))  # Remove special chars
Method Use case Example
.rename(columns={...}) Rename specific columns {'old': 'new'}
df.columns = [...] Replace ALL column names ['a', 'b', 'c']
.columns.str.method() Transform all column names .str.lower()

Handling Duplicates

Duplicate rows can silently inflate your counts, sums, and averages. Before any analysis, you should check "do I have repeated data?" Think of this as quality control — making sure every row represents a unique observation.

# Check if duplicates exist
df.duplicated().any()                    # True if any duplicates
df.duplicated().sum()                    # Count duplicate rows

# See which rows are duplicates
df.duplicated()                          # Boolean Series (True = duplicate)
df[df.duplicated()]                      # Show duplicate rows (excludes first occurrence)
df[df.duplicated(keep=False)]            # Show ALL duplicates (including first)

# Remove duplicates
df.drop_duplicates()                     # Keep first occurrence
df.drop_duplicates(keep='last')          # Keep last occurrence
df.drop_duplicates(keep=False)            # Remove all duplicates

# Check duplicates in specific columns
df.duplicated(subset=['Name', 'Email'])
df.drop_duplicates(subset=['ID'])       # Remove rows with duplicate IDs

# Find duplicate values in a single column
df[df['Email'].duplicated()]           # Rows where email appears more than once
df['Email'].value_counts()              # Count occurrences of each value

Index Management & reset_index()

Many operations (groupby, set_index, filtering) can create messy indexes. Use reset_index() to fix them.

# Basic reset - moves index to a column
df.reset_index()                         # Index becomes column 'index'
df.reset_index(drop=True)                  # Discard old index, create new 0,1,2...

# Common after groupby
result = df.groupby('Category').agg({'Sales': 'sum'})
# result has 'Category' as index - often not what you want!
result.reset_index()                     # Now 'Category' is a regular column

# After filtering - indexes become non-consecutive
filtered = df[df['Age'] > 30]          # Index: [2, 5, 7, 12...]
filtered.reset_index(drop=True)           # Index: [0, 1, 2, 3...]

# Set a column as index (opposite of reset_index)
df.set_index('CustomerID')             # Use CustomerID as index
df.set_index(['Year', 'Month'])        # Multi-level index
⚠️ Common gotcha: After groupby().agg(), the grouped column becomes the index! 
# ❌ Won't work - 'Country' is the index, not a column
grouped = df.groupby('Country').agg({'Sales': 'sum'})
grouped['Country']  # KeyError!

# ✅ Fix with reset_index()
grouped = df.groupby('Country').agg({'Sales': 'sum'}).reset_index()
grouped['Country']  # Works!
Operation Index behavior Need reset_index?
groupby().agg() Grouped columns become index ✅ Almost always
df[condition] Keeps original (non-sequential) ⚠️ Optional (cosmetic)
set_index() Column moves to index ❌ Intentional
sort_values() Reorders but keeps values ⚠️ Optional
concat() May have duplicates ✅ Often needed

Ranking & Window Functions

Ranking assigns a position number to each row based on a value (1st place, 2nd place, etc.). Window functions like running totals and lag/lead let you reference other rows — the previous row's value, the cumulative sum so far, etc. These are the pandas equivalents of SQL's ROW_NUMBER(), RANK(), and LAG().

# Row number (unique rank for each row)
df['RowNum'] = df['Total'].rank(method='first', ascending=False).astype(int)

# Rank (ties get same rank, gaps after)
df['Rank'] = df['Total'].rank(method='min', ascending=False).astype(int)

# Dense rank (ties get same rank, no gaps)
df['DenseRank'] = df['Total'].rank(method='dense', ascending=False).astype(int)

# Rank within groups
df['CountryRank'] = df.groupby('Country')['Total'].rank(
    method='first', ascending=False
).astype(int)

# Running total
df = df.sort_values('Date')
df['RunningTotal'] = df['Total'].cumsum()

# Previous/Next row values
df['PrevTotal'] = df['Total'].shift(1)   # LAG
df['NextTotal'] = df['Total'].shift(-1)  # LEAD
💡 method parameter: 'first'=row number, 'min'=rank, 'dense'=dense rank.

Common Operations

These are the everyday one-liners you'll use constantly — filling nulls, concatenating strings, extracting date parts, filtering by averages, and cross-referencing between DataFrames. Think of this as your toolbox of frequently needed recipes.

# Count distinct
df['Country'].nunique()

# Fill null values
df['Company'].fillna('N/A')

# String concatenation
df['FullName'] = df['FirstName'] + ' ' + df['LastName']

# Date extraction
df['Date'] = pd.to_datetime(df['Date'])
df['Year'] = df['Date'].dt.year
df['Month'] = df['Date'].dt.month

# Filter above average
avg = df['Total'].mean()
df[df['Total'] > avg]

# Filter using another DataFrame's values
high_spenders = df[df['Total'] > 20]['CustomerId']
customers[customers['CustomerId'].isin(high_spenders)]

.str Accessor (String Methods)

The .str accessor lets you run Python string methods on an entire column at once — no loops needed. Think of it as applying .lower(), .split(), .contains() to every row simultaneously.

# Case conversion
df['Name'].str.lower()
df['Name'].str.upper()
df['Name'].str.title()

# Whitespace
df['Name'].str.strip()

# Search & Match
df['Email'].str.contains('gmail')
df['Email'].str.startswith('a')
df['Email'].str.endswith('.com')

# Replace
df['Phone'].str.replace(r'\D', '', regex=True)  # remove non-digits

# Split
df['Name'].str.split(' ')                    # into list
df['Name'].str.split(' ', expand=True)       # into columns
df['Name'].str.split(' ').str[0]             # first part

# Length & Slicing
df['Name'].str.len()
df['Name'].str[:3]                          # first 3 chars

# Padding
df['ID'].str.zfill(5)                       # '42' → '00042'

# Boolean checks
df['Code'].str.isdigit()
df['Code'].str.isalpha()

.dt Accessor (Datetime Methods)

Once a column is datetime type, .dt gives you instant access to components like year, month, day of week, and quarter — no parsing or string slicing needed. This is essential for any time-based analysis (trends, seasonality, grouping by month).

# First, ensure datetime type
df['Date'] = pd.to_datetime(df['Date'])

# Extract components
df['Date'].dt.year
df['Date'].dt.month
df['Date'].dt.day
df['Date'].dt.hour
df['Date'].dt.dayofweek           # 0=Mon, 6=Sun
df['Date'].dt.day_name()          # 'Monday', etc.
df['Date'].dt.quarter             # 1-4

# Boolean checks
df['Date'].dt.is_month_start
df['Date'].dt.is_month_end
df['Date'].dt.is_weekend          # Sat/Sun

# Period conversions (for grouping)
df['Date'].dt.to_period('M')      # monthly
df['Date'].dt.to_period('Q')      # quarterly

# Formatting
df['Date'].dt.strftime('%Y-%m-%d')
df['Date'].dt.strftime('%B %d, %Y')  # 'March 15, 2024'

.cat Accessor (Categorical Methods)

Categories let you tell pandas that a column has a fixed set of values with an optional order (like S < M < L < XL). This saves memory, enables meaningful sorting, and allows comparison operators to work on non-numeric data.

# Convert to categorical
df['Size'] = df['Size'].astype('category')

# Create ordered categorical
df['Size'] = pd.Categorical(
    df['Size'], 
    categories=['S', 'M', 'L', 'XL'], 
    ordered=True
)

# View categories
df['Size'].cat.categories
df['Size'].cat.codes              # numeric codes (0, 1, 2, 3)

# With ordered categorical, comparisons work!
df[df['Size'] > 'M']               # rows where Size is L or XL

Apply & Map

Apply custom functions to your data. Use vectorized operations when possible - they're faster.

.map() — Value Translation (Dictionary Lookup)

Think of .map() as a lookup table. You give it a dictionary and it translates every value in the column. It's the simplest way to say "whenever you see X, replace it with Y."

# Basic dict mapping — translate values
df['Grade'].map({'A': 4, 'B': 3, 'C': 2, 'D': 1, 'F': 0})

# Country codes → full names
country_map = {'US': 'United States', 'UK': 'United Kingdom', 'CA': 'Canada'}
df['CountryName'] = df['CountryCode'].map(country_map)

# Boolean-like cleanup → real booleans
df['Active'] = df['Status'].map({'yes': True, 'no': False, 'Y': True, 'N': False})

# Numeric scores → letter grades
def score_to_grade(score):
    if score >= 90: return 'A'
    elif score >= 80: return 'B'
    elif score >= 70: return 'C'
    elif score >= 60: return 'D'
    else: return 'F'

df['LetterGrade'] = df['Score'].map(score_to_grade)
⚠️ NaN behavior: .map() with a dict sets unmapped values to NaN. If your dict doesn't cover every value, you'll get NaN for the rest. 
# ⚠️ Problem: values not in dict become NaN
df['Size'].map({'S': 'Small', 'M': 'Medium'})
# If any row has 'L' or 'XL' → becomes NaN!

# ✅ Fix 1: Fill NaN with the original value
df['Size'].map({'S': 'Small', 'M': 'Medium'}).fillna(df['Size'])

# ✅ Fix 2: Use .replace() instead (keeps unmapped values as-is)
df['Size'].replace({'S': 'Small', 'M': 'Medium'})
# 'L' stays 'L', 'XL' stays 'XL'

# ✅ Fix 3: Use a defaultdict
from collections import defaultdict
safe_map = defaultdict(lambda: 'Unknown', {'S': 'Small', 'M': 'Medium'})
df['Size'].map(safe_map)

.map() with a Series (Cross-DataFrame Lookup)

You can map from one DataFrame's column using another DataFrame as a lookup table. Create a Series with the lookup index and map onto it — like a VLOOKUP in Excel.

# Build a lookup Series from another DataFrame
# departments has columns: dept_id, dept_name
dept_lookup = departments.set_index('dept_id')['dept_name']
# dept_lookup is now:   101 → 'Sales'
#                       102 → 'Marketing'
#                       103 → 'Engineering'

# Map employee dept_id → dept_name
employees['Department'] = employees['dept_id'].map(dept_lookup)

# Map with a lambda function
df['Name'].map(lambda x: x.upper())           # function
df['Name'].map(lambda x: x.strip().title())    # clean + capitalize
df['Email'].map(lambda x: x.split('@')[1])     # extract domain
df['Price'].map('${:.2f}'.format)              # format as currency

.map() vs .replace() — When to Use Which

.map() .replace()
Unmapped values Become NaN Stay unchanged
Accepts Dict, Series, or function Dict, list, scalar, regex
Works on Series only (or DataFrame element-wise) Series or DataFrame
Use when Complete translation (every value must map) Partial cleanup (fix some values, leave the rest)
Regex support ❌ No ✅ Yes 
# .replace() — partial substitution (unmatched stay the same)
df['Status'].replace({'Active': 'A', 'Inactive': 'I'})

# .replace() with regex — fix messy text patterns
df['Phone'].replace(r'\D', '', regex=True)       # strip non-digits
df['Name'].replace(r'\s+', ' ', regex=True)       # collapse whitespace

# .replace() across entire DataFrame
df.replace({'N/A': np.nan, 'n/a': np.nan, '': np.nan})  # clean nulls

# Chaining .map() for multi-step transforms
df['Region'] = (df['State']
    .map(state_to_region)        # state → region
    .fillna('Unknown')             # handle unmapped states
)
💡 Rule of thumb:
  • .map(dict) — "Translate ALL values. Anything I didn't list should be NaN."
  • .replace(dict) — "Fix SOME values. Leave everything else alone."
  • .map(function) — "Run this function on every single value."
  • np.where / np.select — "Create values based on conditions (if/else logic)."

.apply() — Custom Functions

When built-in methods and simple mapping aren't enough, .apply() runs any function on every element, row, or column. It's the most flexible tool — but also the slowest.

# .apply() on Series - element-wise with more flexibility
df['Total'].apply(lambda x: x * 1.1)
df['Name'].apply(lambda x: x.split()[0] if ' ' in x else x)

# .apply() on DataFrame - apply to each column (default)
df.apply(lambda col: col.max() - col.min())    # range per column

# .apply() on DataFrame - apply to each row
df.apply(lambda row: row['Price'] * row['Qty'], axis=1)

# .apply() with custom function
def categorize(value):
    if value > 100: return 'High'
    elif value > 50: return 'Medium'
    else: return 'Low'

df['Category'] = df['Total'].apply(categorize)

# .apply() with groupby - apply function to each group
df.groupby('Country').apply(lambda g: g.nlargest(3, 'Total'))

# .map() on DataFrame - element-wise (replaces deprecated .applymap())
df[['A', 'B']].map(lambda x: f'${x:.2f}')      # format all cells
Method Works on Use case
Series.map(dict) Each element Translate values via lookup table (unmapped → NaN)
Series.map(func) Each element Simple one-liner transforms on every value
Series.map(Series) Each element Cross-DataFrame lookups (like VLOOKUP)
Series.replace() Each element Fix some values, leave the rest unchanged
Series.apply() Each element Complex logic, multiple operations
DataFrame.apply(axis=0) Each column Column-wise aggregations
DataFrame.apply(axis=1) Each row Row-wise calculations using multiple columns
DataFrame.map() Each element Format all cells, element-wise transforms
💡 Prefer vectorized operations (df['col'] * 2) over apply - they're 10-100x faster. Use apply only when vectorization isn't possible.

GeoPandas (Spatial Data)

GeoPandas extends pandas to handle geographic data — points, lines, and polygons. If your data has locations (latitude/longitude, boundaries, shapes), GeoPandas lets you do spatial filtering ("which points are inside this area?"), spatial joins, and mapping — all with familiar pandas syntax.

import geopandas as gpd
from shapely.geometry import Point

# Load spatial data
gdf = gpd.read_file('data.geojson')

# Create from DataFrame with coordinates
gdf = gpd.GeoDataFrame(
    df, 
    geometry=gpd.points_from_xy(df['lon'], df['lat']),
    crs='EPSG:4326'
)
# Spatial filtering
gdf[gdf.within(polygon)]
gdf[gdf.intersects(other_geom)]
gdf[gdf.contains(point)]

# Spatial join
joined = gpd.sjoin(points, neighborhoods, predicate='within')

# Nearest neighbor join
joined = gpd.sjoin_nearest(points, stations, distance_col='dist')

# Geometric operations
gdf['buffered'] = gdf.geometry.buffer(100)
gdf['centroid'] = gdf.geometry.centroid
gdf['distance'] = gdf.geometry.distance(target_point)

# Dissolve (merge geometries by group)
dissolved = gdf.dissolve(by='region', aggfunc={'population': 'sum'})

# Overlay operations
gpd.overlay(gdf1, gdf2, how='intersection')
gpd.overlay(gdf1, gdf2, how='difference')
gpd.overlay(gdf1, gdf2, how='union')

# CRS (Coordinate Reference System)
gdf.crs                              # check current CRS
gdf = gdf.set_crs('EPSG:4326')       # set CRS
gdf = gdf.to_crs('EPSG:3857')        # reproject

# ⚠️ Always project to meters before calculating area/distance!
gdf_m = gdf.to_crs('EPSG:3857')
gdf['area_m2'] = gdf_m.geometry.area

# Quick plot
gdf.plot(column='population', cmap='viridis', legend=True)
gdf.explore()  # interactive map
💡 Common CRS: EPSG:4326 (GPS lat/lon), EPSG:3857 (Web Mercator, meters).

Key Things to Remember

  • Pandas is 0-indexed
  • Use & | ~ for AND, OR, NOT (not the words)
  • Always wrap conditions in parentheses when combining
  • Operations return new DataFrames - assign to save changes
  • Use .reset_index() after groupby to get a clean DataFrame
  • Chain operations for readable, fluent code
