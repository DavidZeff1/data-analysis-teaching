🔽 Filters

Filters decide what data reaches your view. That sounds simple until you meet Tableau's order of operations — where a Top 10 filter and a Region filter interact in a way that produces confidently wrong answers.

⏱️ 30 min ⭐ Core skill 🔗 Needs: Interface, Chart Types

🗂️ Six kinds of filter

They look similar on the shelf but behave very differently — especially in what order they run.

Dimension filters

Keep or exclude category values. "Only East and West."

Measure filters

Filter on an aggregated number. "Only sub-categories with Sales over $100K." Runs after aggregation.

Date filters

Absolute ranges, relative ranges ("last 12 months"), or discrete periods.

Context filters

Promoted to run first, creating a temporary subset everything else works from.

Data source filters

Set on the Data Source page. Apply to every sheet in the workbook and are easy to forget.

Extract filters

Limit what gets pulled into an extract at all. The most efficient filter, because the data never arrives.

➕ Three ways to add one

// 1. Drag to the Filters shelf — full control
Drag [Region] from the Data pane onto Filters
→ a dialog opens for choosing values

// 2. From the view — fastest for exploring
Right-click a bar or mark → Keep Only
Right-click a bar or mark → Exclude

// 3. Show Filter — makes it interactive
Right-click [Region] in the Data pane → Show Filter
→ a filter card appears beside the view
"Exclude" quietly becomes a trap

Excluding specific members creates a rule that says "everything except these". When new values appear in the data next quarter, they're included automatically — possibly without anyone noticing. Prefer explicitly keeping what you want, or a condition that re-evaluates itself.

🎛️ The filter dialog's four tabs

TabWhat it doesUpdates with new data?
GeneralTick and untick specific valuesNo — a fixed list
WildcardMatch text patterns — contains, starts with, ends withYes
ConditionFilter by a formula, e.g. SUM([Sales]) > 100000Yes
TopTop or bottom N by a measureYes
Prefer Condition and Top over General

The General tab produces a hard-coded list of members. Condition and Top define rules, so they stay correct as data changes. A dashboard that silently stops including a new region six months from now is a real and common failure — and it always traces back to a General-tab filter.

The Top tab can be driven by a parameter, letting viewers choose Top 5, 10 or 20 themselves. That's the standard way to make a "Top N" dashboard flexible without building three sheets.

🎚️ Interactive filter cards

Right-click a filter pill → Show Filter. Then right-click the card itself to choose how it behaves.

Card styles

  • Single value (list) — radio buttons; one at a time
  • Single value (dropdown) — compact, good for many values
  • Multiple values (list) — checkboxes
  • Multiple values (dropdown) — compact multi-select
  • Wildcard match — a search box
  • Slider — for numbers and dates

Options worth setting

  • Show "All" value — give viewers a way back
  • Only Relevant Values — the card only lists values that survive other filters. Prevents empty results.
  • Apply button — waits for the viewer to finish selecting before re-querying. Essential on slow data.
  • Customize → hide the title when the label is obvious
"Only Relevant Values" prevents a common frustration

Without it, filtering to the West region still shows every state in the country in your State filter — most of which produce a blank chart. With it, only the states actually in the West are listed. Right-click the card → Only Relevant Values. It costs one click and removes a whole class of confused user.

🪜 Order of operations — the thing to actually learn

Tableau applies filters in a strict sequence. Almost every "why is this number wrong?" question in Tableau is answered somewhere in this ladder.

  1. Extract filters
    Applied when the extract was built — the data isn't even there
  2. Data source filters
    Workbook-wide, set on the Data Source page
  3. Context filters
    Creates a temporary subset. Everything below sees only this.
  4. FIXED LOD expressions
    Computed before dimension filters — which is why they seem to ignore them
  5. Dimension filters
    Ordinary filter cards. Top N filters run here.
  6. INCLUDE / EXCLUDE LODs
    These do respect dimension filters
  7. Measure filters
    Filters on aggregated values, e.g. SUM(Sales) > 100K
  8. Table calculations & their filters
    Run last, on the finished view — see Table Calculations

🎯 Context filters — the classic Top N bug

The situation

You want the top 10 products in the East region. You add a Top 10 filter on Product Name and a Region filter set to East. You get fewer than 10 products — sometimes only three or four — and they look wrong.

What actually happens

Both filters sit at level 5, and Tableau computes Top 10 across all regions first. It then filters those ten to East — and only the ones that happen to be in East survive.

Result: "the East-region members of the national top 10". Not what you asked for.

→
The fix

Right-click the Region filter → Add to Context. It turns grey and moves to level 3, so the Top 10 is computed within East.

Result: the actual top 10 products in the East region. Always 10 of them.

When to reach for a context filter

  • Top N within a subset — the case above, and by far the most common
  • Percent of total within a subset — same logic
  • Making FIXED LODs respect a filter — context runs before them
  • Performance on very large data — occasionally, by shrinking the working set early
Don't add everything to context

Context filters build a temporary table, which costs time on each change. Use them where the order genuinely matters, not as a default. Two or three per workbook is plenty.

🔗 Applying a filter to several sheets

By default a filter affects only its own worksheet. On a dashboard that's rarely what you want.

Right-click the filter pill → Apply to Worksheets →

  Only This Worksheet          // the default
  All Using This Data Source   // every sheet on this connection
  Selected Worksheets…         // pick exactly which — usually best
  All Using Related Data Sources
Prefer "Selected Worksheets"

"All Using This Data Source" also hits sheets you build later, including ones that shouldn't be filtered — a KPI card showing company-wide totals, for instance. Choosing explicitly takes ten extra seconds and prevents a confusing bug much later.

A filter applied to multiple sheets shows a small icon on the pill. On a dashboard, that one filter card now drives every connected chart — which is exactly how you build coordinated interactivity.

⚡ Filters and performance

Filters are the biggest single lever on how fast a workbook feels, and the differences between them are large.

ApproachCostNotes
Extract filterCheapestThe rows never enter the workbook
Data source filterVery cheapApplied before anything else
Dimension filter on a listCheapEfficient, especially on indexed columns
Context filterModerateBuilds a temporary table on each change
Measure filterModerateRequires aggregating first
Filter on a table calculationExpensiveRuns last, after everything else
"Only Relevant Values" on many cardsExpensiveEach card re-queries when any other changes
Filter early, filter once

If a dashboard only ever shows the last two years, exclude older data at the extract level rather than filtering it in every view. The general principle: remove data as early in the pipeline as you can, because everything downstream then has less work to do.

🧠 Key intuitions

  • Order of operations explains most wrong numbers — learn the ladder
  • Top N + another filter = context filter, every time
  • Condition and Top beat the General tab — rules don't go stale
  • "Exclude" silently admits new values — prefer keeping what you want
  • Show Filter is what makes a dashboard interactive
  • Only Relevant Values stops viewers picking combinations with no data
  • Apply to Selected Worksheets, not "All Using This Data Source"
  • Filter as early in the pipeline as possible

🎯 Practice exercises

  1. Build Sales by Category, then filter to the Consumer segment only
  2. Add an interactive Region filter and try each card style
  3. Filter to 2017 orders only using a date filter
  4. Use a relative date filter for "last 12 months" and explain why it might show nothing
  5. Add a measure filter for sub-categories with Sales over $100K
  6. Create a Top 10 Customers by Sales filter
  7. Now add a Region filter set to East — count the products, and explain the result
  8. Fix it with Add to Context and confirm you get exactly 10
  9. Use a wildcard filter to find products containing "Chair"
  10. Turn on Only Relevant Values on a State filter and observe the difference
  11. Apply a Category filter to selected worksheets across three sheets
  12. Create a parameter-driven Top N filter that viewers can set to 5, 10 or 20
🏆 Challenge: prove the order of operations to yourself

Build one sheet showing the top 5 sub-categories by Sales, with a Region filter card. Duplicate it, and add the Region filter to context on the copy. Put both on a dashboard side by side with the same filter driving them, then click through each region. The two charts will disagree — sometimes dramatically. Being able to explain why, out loud, means you understand Tableau's evaluation order better than most people who use it daily.
