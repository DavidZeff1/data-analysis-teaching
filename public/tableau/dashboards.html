🏁 Dashboards

Everything in this section, assembled into one interactive screen. A dashboard isn't a collection of charts — it's an argument made visually, where the viewer can follow up on their own questions by clicking. This is the section capstone.

📋 The brief

Your task

Build a one-screen sales dashboard that lets a regional manager answer, without asking you: how are we doing, where is it working, what's going wrong, and what should I look at next?

The constraint — one screen, no scrolling — is the exercise. What you leave off is as much a decision as what you include, and it's the harder one.

The test every element must pass

"What would someone do differently after seeing this?" If the honest answer is "nothing", it doesn't belong. Most weak dashboards fail by addition — every chart individually defensible, the whole thing unreadable.

✏️ Plan on paper first

Ten minutes of sketching saves an hour of dragging. Decide the question, the charts that answer it, and the layout — before opening Tableau.

  1. Write the question in one sentence

    "How is regional sales performance trending, and which products are hurting margin?" If you can't write it, the dashboard doesn't have a point yet.

  2. List the four or five things that answer it

    Headline numbers · trend over time · breakdown by region · the exception chart. Anything beyond five, cut.

  3. Sketch the layout

    Top-left gets the most attention, bottom-right the least. Put the headline where the eye lands first.

  4. Build each worksheet separately

    Get every chart working and formatted on its own sheet first. Assembling half-finished charts into a dashboard is much harder than assembling finished ones.

  5. Then assemble, and only then add interactivity

    Layout, then actions. Adding filter actions to a layout you're still rearranging wastes time.

Fixed size

Exact pixels. Predictable everywhere — the safe choice, and what to use for anything you publish.

Automatic

Fills the browser window. Flexible, but layouts can break at sizes you never tested.

Range

Flexes between a minimum and maximum. A reasonable compromise.

📐 Layout: tiled, floating and containers

Tiled — use this

Objects snap into a grid and resize together. Predictable, and it survives being viewed at a different size.

Floating — sparingly

Free positioning, layered on top. Useful for a logo or an overlay; a maintenance problem if you build the whole layout this way.

Containers are the actual skill

A Horizontal container arranges its contents side by side; a Vertical container stacks them. Nesting them gives you a real layout system rather than a pile of dragged objects.

  • Build the containers first, then drop sheets in — retrofitting containers is painful
  • Use the Layout pane (top left, next to Dashboard) to see the nesting tree and select precisely
  • Select a container → Distribute Contents Evenly for instant equal sizing
  • Set inner padding on containers rather than adding Blank objects for spacing
  • Blank objects are for deliberate gaps, not for pushing things around
Selecting the right thing

Clicking a chart selects the chart; clicking its thin border selects its container. When you can't grab what you want, use the Layout pane's tree — click the item there and Tableau selects it on the canvas. This is the single most useful thing to know about dashboard building, and it's not discoverable.

⚡ Actions — what makes it a dashboard

Without actions you have a poster. With them, viewers answer their own follow-up questions.

What a filter action actually does

A Filter action removes the non-matching rows, as above. A Highlight action instead dims them while keeping everything visible — usually better when the comparison is the point.

Adding one

⚡ Quick way

Select a sheet on the dashboard → click the funnel icon in its top-right corner. That sheet now filters every other sheet using the same data source.

🎛️ Controlled way

Dashboard → Actions → Add Action. Choose source and target sheets, the trigger, and — importantly — what happens on deselect.

Always set "Clearing the selection will…"

Three options, and the default causes most action-related confusion:

  • Show all values — deselecting returns everything. Usually what you want.
  • Exclude all values — the target goes blank until something is selected. Deliberate, occasionally right.
  • Leave the filter — the selection sticks. Viewers get stuck in a filtered state with no obvious way out.
Prefer Highlight over Filter more often than you think

Filtering removes context — click West and you can no longer see how it compares to East. Highlighting keeps everything visible while drawing attention. For comparison dashboards, highlight is usually the better interaction.

🧱 Dashboard objects

Text

Titles, framing, instructions. Use it to state the finding, not just to label.

Image

Logos and icons. Can be given a target URL.

Blank

Deliberate whitespace. Prefer container padding where you can.

Navigation

A styled button to another dashboard. Much clearer than a Go-to-Sheet action.

Download

One-click PDF or image export for viewers.

Extension

Third-party add-ins. Powerful, but a dependency — check before relying on one.

Ask Data / Show Me

Natural-language querying, where your Tableau version supports it.

Web Page

Embeds a live URL, which can respond to a URL action.

Write the title as a finding

"Sales Dashboard" labels the object. "West leads on revenue but Central converts it best" tells the reader what you concluded and gives everything below it a purpose. Use a Text object, make it the largest thing on the screen, and update it when the story changes.

📱 Device layouts

Dashboard → Device Preview → Add Layout creates a separate arrangement for phone or tablet from the same sheets. Tableau serves whichever matches the viewer's screen.

  • Same data and worksheets — only the arrangement differs
  • Sheets can be hidden entirely on smaller layouts
  • Phone layouts should stack vertically; side-by-side never works at that width
  • Cut to the two or three most important charts rather than shrinking all six

If your dashboard will mostly be viewed on a laptop, a phone layout is optional. If it's going to executives, build one — that's exactly the audience checking it on a phone between meetings.

📤 Publishing to Tableau Public

  1. Check what you're about to make public

    Tableau Public is publicly visible and downloadable, including the underlying data. Never publish anything confidential. This is not reversible in any meaningful sense.

  2. Server → Tableau Public → Save to Tableau Public

    Sign in and give the workbook a clear name — this becomes its public title.

  3. Set the thumbnail

    Whatever sheet is active when you save becomes the preview image in your profile. Make it the dashboard, not a working sheet.

  4. Write a description

    On the Tableau Public site, add a note on what the dashboard shows and what you found. A portfolio piece with a written finding reads very differently from a bare chart.

This is your portfolio

Tableau Public profiles are one of the few places you can show real analytical work publicly, and hiring managers do look at them. Three well-finished dashboards with written findings beat twenty half-built experiments.

✅ Self-review

Correctness

  • Orders counted with COUNTD, not COUNT
  • Ratios use SUM(a)/SUM(b)
  • Any Top N combined with a filter uses a context filter
  • Diverging colour palettes centred on zero
  • No unknown locations on the map
  • Totals reconcile against a plain SUM(Sales)

Communication

  • Fits on one screen with no scrolling
  • The title states a finding
  • Every chart title says what it shows
  • Gridlines and borders removed
  • Numbers formatted as $0K, not raw decimals
  • Every action tested, including deselect
The five-second test

Show it to someone who hasn't seen the data and ask what they take away. If it isn't roughly your intended message, the dashboard needs changing — not the viewer. Then ask them to find one specific number; if they can't, your layout is wrong.

🎯 Practice exercises

  1. Build four worksheets: KPI text, monthly trend, sales by region, profit by sub-category
  2. Create a dashboard at fixed 1200 × 800 and add all four
  3. Arrange them with horizontal and vertical containers, not free dragging
  4. Add a Text title stating a finding rather than a label
  5. Make the region chart filter the others via the funnel icon
  6. Set "Clearing the selection will show all values" and test deselecting
  7. Change that action to Highlight instead and decide which you prefer
  8. Add an interactive Category filter applied to selected worksheets
  9. Turn on Only Relevant Values on that filter
  10. Add a map and make clicking a state filter everything else
  11. Remove all gridlines and borders, and format every number as $0K
  12. Create a phone layout showing only the KPIs and the trend
🏆 Challenge: the discount dashboard

Build a dashboard answering one question: "Is our discounting policy costing us money, and where?" Include the headline margin impact, a scatter of discount against profit ratio by sub-category with a trend line, a map of margin by state, and a sorted bar of the worst offenders. Make selecting a sub-category filter the map and vice versa. Then add a Text object with your recommendation and the dollar figure attached to it.

That last step is what separates a dashboard from an analysis. Anyone can show the data; the value is in saying what to do about it — and being willing to put a number on it.
