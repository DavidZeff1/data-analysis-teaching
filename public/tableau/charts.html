📊 Chart Types

Every chart in Tableau is a specific arrangement of pills on shelves. Once you can read a chart as "this pill here, that pill there", you can build any of them from memory — and Show Me stops being a crutch.

🧾 Every chart is a recipe

Charts in Tableau aren't types you select — they're what emerges from where you put things. The same three pills in different positions produce different charts.

The pattern behind the pattern

Two dimensions → a table or crosstab. One dimension + one measure → a bar chart. A date + a measure → a line. Two measures → a scatter. Learn those four and you can predict what Tableau will build before you drop the pill.

Sort it, every time

Tableau sorts alphabetically by default. Click the sort icon on the axis, or right-click the dimension → Sort → By Field for something that survives being put on a dashboard. See Sorting, Groups & Sets.

A dimension on Color splits one line into several. Keep it under about five, or the chart becomes spaghetti.

// Changing the date granularity
Right-click the date pill → Year / Quarter / Month / Week / Day

// Blue (discrete) vs green (continuous) matters here:
//   Blue MONTH   → 12 buckets, all years combined (seasonality)
//   Green MONTH  → one continuous timeline (trend)

// Both at once — a small-multiples grid:
Columns: MONTH(Order Date) // green, continuous
Rows:    SUM(Sales)
Rows:    YEAR(Order Date)  // blue, discrete → one row per year
Too many lines? Use small multiples

Instead of seventeen coloured lines, drag the dimension to Rows so each gets its own miniature chart on a shared scale. Far easier to read, and Tableau builds it as fast as the tangled version.

🧱 Composition — stacked bars and pies

Stacked bar

Columns: SUM([Sales])
Rows:    [Region]
Color:   [Category]

// For 100% stacked, add a table calc:
Right-click SUM(Sales) →
  Quick Table Calculation → Percent of Total
  Compute Using → Table (across)

Shows totals and their mix at once. Only the bottom segment sits on a common baseline, so keep it to three or four categories.

Pie chart

Marks type: Pie
Angle:  SUM([Sales])
Color:  [Region]
Label:  [Region], SUM([Sales])

// Size is on the Size slider, not a shelf —
// pies default to tiny. Drag it up.

Two to five slices with clearly different sizes, or use a bar chart. Nobody can rank eight similar angles — see the Chart Decision Tree.

⚖️ Dual axis — two measures, one view

  1. Build the first measure

    MONTH(Order Date) on Columns, SUM(Sales) on Rows.

  2. Add the second to the same shelf

    Drag SUM(Profit) onto Rows next to Sales. You get two stacked charts.

  3. Turn on Dual Axis

    Right-click the second pill → Dual Axis. The two charts overlay, and the Marks card splits into one section per measure.

  4. Set different mark types

    Click SUM(Sales) in the Marks card → Bar. Click SUM(Profit) → Line. Now you have bars with a line over them.

  5. Decide about synchronising

    Right-click the right axis → Synchronize Axis to put both on one scale.

⚠️ Unsynchronised axes can lie

With independent scales, Tableau fits each measure to the available height. Profit ($286K) and Sales ($2.3M) will appear to track each other closely even when they don't — because the axes were stretched to make them fit.

Synchronise when the measures share units (Sales and Profit are both dollars). Don't when they don't (Sales and Profit Ratio) — but then label both axes clearly, because the reader has no other way to tell.

🎓 Charts Show Me won't build for you

Bar-in-bar

Actual vs target in one bar. Dual axis with both as Bars, synchronised, then narrow the front bar with the Size slider.

Lollipop chart

Dual axis on the same measure: one as Bar sized very thin, one as Circle. Cleaner than bars when you have many categories.

Bullet chart

Show Me has one, but building it manually — a bar plus a reference line for target plus distribution bands — gives far more control.

Pareto chart

Sorted bars with a cumulative-percentage line on a dual axis. Needs two table calculations and is worth the effort.

Small multiples

Put a dimension on Rows and Columns alongside your measures. One small chart per combination, shared scale.

Donut chart

Two pies on a dual axis — the front one white and smaller. Fiddly, but it lets you put a KPI in the middle.

Dual axis is the universal escape hatch

Most "how did they build that?" Tableau charts are a dual axis with different mark types and some size trickery. When a chart you want isn't in Show Me, the answer is usually two measures on one axis.

💡 Show Me — useful, with a caveat

Select fields, open Show Me (top right), and available chart types light up while impossible ones grey out. Hovering any option tells you what fields it needs — genuinely the fastest way to learn the requirements of each chart.

✓ Good for

  • Learning what each chart needs
  • Quickly trying alternatives
  • Getting unstuck

⚠️ Limits

  • Rearranges your shelves, sometimes destructively
  • Only knows standard charts
  • Optimises for "valid", not "clear"

Use it while you're learning; build by hand once you know the recipes. Placing pills deliberately gives you a view you can reason about and fix — Show Me's arrangement is sometimes surprising.

🧠 Key intuitions

  • Charts are recipes, not types you pick
  • Dimension + measure = bar; date + measure = line; two measures = scatter
  • Scatter plots need something on Detail — otherwise one dot
  • Always sort bar charts by value
  • Horizontal bars for long labels
  • Blue vs green dates decide seasonality vs trend
  • Dual axis: synchronise when units match, label clearly when they don't
  • Too many lines → small multiples
  • Dual axis is how the fancy charts are made

🎯 Practice exercises

  1. Bar chart: Profit by Sub-Category, sorted descending — how many lose money?
  2. Line chart: Sales by Month with a separate line per Region
  3. Switch that date pill between blue and green Month and explain both results
  4. Scatter: Discount vs Profit Ratio, coloured by Category, with Sub-Category on Detail
  5. Dual axis: Sales as bars and Profit as a line over time — synchronise the axes and explain whether you should
  6. Highlight table: Sales by State and Category using Square marks
  7. Stacked bar: Sales by Region split by Segment
  8. Convert that to 100% stacked with a Percent of Total table calculation
  9. Pie chart: Sales by Region — then rebuild it as a bar chart and say which reads better
  10. Small multiples: monthly sales trend with one row per Region
  11. Bar-in-bar: Sales vs Profit by Category on a dual axis, both as bars
  12. Select Category and Sales, then click through every enabled option in Show Me
🏆 Challenge: the same question, four ways

Take one question — "how do our categories perform across regions?" — and answer it with a grouped bar chart, a stacked bar, a highlight table and small multiples. Put all four side by side and decide which you'd actually present, and why. The answer depends on whether the audience needs precise values, overall pattern, or a single comparison, and being able to articulate that choice is more valuable than knowing how to build any one of them.
