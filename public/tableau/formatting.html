🎛️ Formatting & Custom Charts

Tableau's defaults are functional, not finished — and almost every part of a view is editable once you know where the setting lives. This is the map: the formatting hierarchy, a labelled tour of every region of a chart, the panels that sit beside it, and the techniques for building charts Show Me has never heard of.

🪜 The formatting hierarchy

Before any specific setting: Tableau formats at four nested levels, and the most specific one wins. Knowing which level you're editing is the difference between changing one thing and changing everything.

An inner box always beats the box containing it. Set the outer ones first and you'll rarely need the inner ones.

Work outside-in, not inside-out

Set fonts and the base palette at workbook level once. Then only reach inwards to make a deliberate exception. The alternative — formatting each sheet individually — produces a dashboard where ten charts are each slightly different and nobody can say why.

Undoing: the Format pane has a Clear button (an eraser) at the top that resets the sheet to default. Right-clicking any individual element also offers Reset. Both beat hunting for the one setting you changed an hour ago.

🗂️ The Format pane — and the Header/Pane split

Open it with Format → Font, or right-click in the view → Format. It replaces the Data pane on the left. Three tabs — Sheet, Rows, Columns — and inside each, settings split into two columns that catch people out constantly.

Changing the font under Header and wondering why the numbers didn't change is the standard first experience of this pane. There's a third Total column when totals are showing, so you can style those separately — usually bold.

📏 Axes — where a chart starts telling the truth or not

Right-click any axis → Edit Axis. Two of these settings decide whether your chart is honest.

Include zero

✓ Axis includes zero
Q1 100 · Q2 103 · Q3 107 · Q4 110. A 10% rise over the year, and it looks like one.
✗ Axis starts at 95
Identical data. Q4 now looks four times Q1. Nothing was falsified — the baseline was just moved.
Bars encode by length, so the baseline is part of the number

Cut it and every comparison is exaggerated. Bar and column charts must include zero, always. Line charts may start elsewhere when the variation is genuinely small — but say so on the axis.

Uniform vs independent ranges

✓ Uniform axis range
Every panel on one scale, so South visibly is the smallest. Comparison works.
✗ Independent ranges
Same data. Each panel rescaled to fit, so South looks the size of West. The differences vanish.

Right-click any axis or header → untick Show Header to hide it entirely. Essential for the placeholder-axis trick further down, and for hiding the redundant duplicate axis on a dual-axis chart.

🧹 The 60-second cleanup

Six removals, no additions. Do this on every sheet before showing anyone.

Tableau's default
→
After the cleanup
  1. Grid lines → None

    Format → Lines → Grid Lines → None

  2. Row and column dividers → None

    Format → Borders. Keep dividers only between groups on a table.

  3. Hide field labels

    Right-click the grey field bar → Hide Field Labels for Rows and for Columns.

  4. Label the marks, delete the axis

    Values on the bars mean the axis is redundant — right-click it → untick Show Header.

  5. Colour one thing, grey the rest

    The bar you're talking about gets the accent colour. Everything else goes neutral.

  6. Write the title as a finding

    "West leads on revenue", not "Sheet 1" or "Sales by Region".

📎 The panels beside your chart

Tableau parks a stack of cards to the right of the view. Each is a separate object you can show, hide, retitle, restyle and — on a dashboard — move anywhere.

The Highlighter is the underused one

Typing in it dims everything that doesn't match while keeping it visible — so a viewer can find one product line among fifty without losing the context of the rest. On a tangled multi-line chart it beats a filter, because the comparison survives.

On a dashboard these don't come across automatically. Select the sheet, open its ▾ → Legends / Filters / Parameters and add the ones you want, or drag them from the Dashboard pane. Tile them into the layout or float them on top.

🗺️ Legends in detail

The colour-coded key beside the chart — and an object with more settings than people expect.

Discrete legend — a key

A dimension on Color gives one swatch per member. Every member is independently recolourable and the order is editable by dragging.

Continuous legend — a scale

A measure on Color gives a gradient with end values. Readers have to estimate from it, which is why stepped colour is often clearer.

🌈 Colour, in depth

Palette types

Categorical

Unordered groups. Stop at about seven before colours stop being distinguishable.

Sequential

One direction, low to high. Sales, volume, count.

Diverging

A meaningful midpoint. Profit and loss, above and below target.

Stepped colour — the conditional-formatting equivalent

A continuous gradient asks the reader to judge shades. Stepped Color gives discrete bands instead, so equal values look equal and the legend becomes a set of ranges you can name in a meeting.

Continuous gradient

Ten shades, none of them nameable. Are cells 3 and 4 different, or is that the eye?

→
Stepped, 4 bands

Four bands you can label: under 100K, 100–300K, 300–600K, 600K+.

Edit Colors → Stepped Color → set the number of steps.

⚠️ Always set the diverging centre by hand

Tableau centres a diverging palette on the data's midpoint, not on zero. Colour states by Profit when every state is profitable and the weakest ones still render red — the chart claims losses that don't exist.

Fix: Edit Colors → Advanced → tick Center and enter 0. Fix Start and End too if you want the scale stable as filters change.

Beyond the palette

Opacity

The slider on the Color card. 60–70% on scatter plots reveals density where points overlap. 0% hides a mark while it still occupies space — the basis of several custom-chart tricks.

Borders

Color card → Effects → Border. A thin white border between stacked segments or dense squares makes boundaries readable without gridlines.

Halo

Color card → Effects → Halo. A ring around each mark so dark points stay visible on a dark map background.

Custom palettes

For brand colours, edit Preferences.tps in My Tableau Repository, define a palette with hex values, and restart Tableau. It then appears in every workbook.

Colour one thing, grey the rest

The highest-impact colour decision in any chart — you can see it in the cleanup example above. A chart where all twelve bars are different colours has no focal point, so the reader has to work out what matters. Assign the greys explicitly in Edit Colors.

Accessibility: roughly 1 in 12 men can't reliably separate red from green. Never let that pair be the only signal — pair colour with position, a label or a shape. Tableau's "Color Blind" categorical palette is a safe default.

📐 Size and shape

Size

The slider sets a uniform size; dropping a measure on Size makes it vary by value.

  • Bars — width, which is how bar-in-bar charts are made
  • Circles — area, not radius, so the mapping is perceptually honest
  • Lines — thickness

Right-click the size legend → Edit Sizes to set the value range sizes map to, and how wide the smallest-to-largest spread is.

Shape

Available on Circle/Shape mark types. Click Shape on the Marks card to pick a palette — filled, open, arrows, KPI symbols.

  • Assign a specific shape per member, like colours
  • Custom shapes: drop PNGs into My Tableau Repository → Shapes → <your folder>, then click Reload Shapes

Shape is a weak channel — people reliably distinguish four or five. Use it for a secondary dimension, never the main comparison.

🔖 Labels — and the setting that replaces a legend

Click Label on the Marks card. Beyond the on/off checkbox, Marks to Label decides whether labels help or clutter. Same chart, three settings:

Marks to Label: All
Twelve numbers competing with the lines. Precise, unreadable.
Min / Max
Start and end only. Often exactly the right amount of labelling.
✓ Line Ends
Delete the legend. Each line names itself where it finishes — the eye never leaves the data.
OptionWhat it does
TextAn editor — combine fields, static text and line breaks: "<SUM(Sales)> in <Region>"
Selected / HighlightedLabels appear only on interaction — clean by default, informative on demand
AlignmentPosition relative to the mark, plus rotation and wrapping
Allow labels to overlapOff by default — which is why some labels silently disappear when space is tight

If a label you expected isn't showing, it's almost always the overlap rule. Either allow overlap, make the cells bigger, or label fewer marks — the third is usually right.

💬 Tooltips

Every mark already has a hover surface. It costs no space on the chart and it's the cheapest place to add depth.

Repeats the axis, with pointless precision.

✓ Written deliberately
<b>West</b>
Sales   $725K   (31.6% of total)
Profit  $108K   (14.9% margin)
Orders  1,611

<i>Click to filter the dashboard</i>

Context, share, margin, and an instruction.

The editor

  • Insert menu — any field on the sheet, a parameter, the sheet name, or All Fields
  • Formatting — font, size, colour, bold and italic per line
  • Responsive / On Hover / On Click — On Click is better on touch devices
  • Include command buttons — the Keep Only / Exclude controls. Turn off on published dashboards where viewers shouldn't restructure the view
  • Allow selection by category — clicking a tooltip value selects all marks sharing it
Viz in Tooltip — the best value feature in Tableau

Insert → Sheets → pick another worksheet. Hovering a region now pops up a small chart of that region's monthly trend, filtered automatically to what you're hovering. A whole second level of detail for zero dashboard space.

Keep the embedded sheet small and simple — it renders on every hover — and set maxwidth and maxheight in the inserted tag.

✍️ Annotations

Right-click in the view → Annotate. Three kinds, differing in what they're anchored to.

Mark

Attached to a specific data point. Moves with it as data changes, and disappears if that mark is filtered out.

Point

Attached to a fixed spot on the axes. Stays put regardless of the data.

Area

A shaded region. Good for calling out a whole period — "supply chain disruption".

Annotations can carry live values

In the editor, right-click → Insert and pick a field: "Peak of <SUM(Sales)> in <MONTH(Order Date)>". Format the box, line and shading by right-clicking the annotation → Format.

One annotation pinned to the anomaly does more than a paragraph underneath, because the reader's eye is already there. One or two per view, not six.

📰 Titles and captions — with live text

Double-click the title to edit. The Insert menu is the part people miss: titles can contain live values.

InsertProduces
<Sheet Name>The worksheet's name — renaming the sheet renames the title
<Parameters.Select Measure>The current value of a parameter
<Region>The current filter selection for that field
<Page Name>The current value on the Pages shelf
 
// A title that describes itself as the viewer filters
<Parameters.Select Measure> by Sub-Category — <Region>

// Renders as:
Profit by Sub-Category — West
Profit by Sub-Category — All
Dynamic titles solve the screenshot problem

Someone filters to one region and screenshots it. Without a dynamic title that image says "Sales by Sub-Category" and nothing about which region — and it ends up in a deck with no way to tell. Putting the filter value in the title makes every screenshot self-describing.

Captions (Worksheet → Show Caption) sit below the view and auto-generate a description you can overwrite. Use them for definitions, data-as-of dates and caveats — things that belong near the chart but shouldn't compete with it.

The four-part structure

A custom format string has up to four sections separated by semicolons:

positive ; negative ; zero ; text

So ▲0.0%;▼0.0%;— means positive gets an up triangle, negative a down triangle, zero a dash. Combine that with conditional colour and you have a KPI change indicator with no calculated field at all.

Formatting is display only. Showing $1.2M doesn't round the underlying value — sorting, filtering and calculations still use the full number. That's usually right, but it means two rows can both display "$1.2M" and sort in an order that looks wrong. If exactness matters at the displayed precision, round in a calculation.

🛠️ Building genuinely custom charts

Almost every "how did they build that?" Tableau chart comes from a small set of techniques.

The placeholder axisThe single most important custom-chart trick

Put a constant on a shelf to create a blank layer you can draw anything on. Double-click the empty part of a shelf and type:

MIN(0)     // a flat axis at zero
AVG(0)     // same thing, different aggregation
MIN(1)     // a flat axis at one

Add a second placeholder, make it a dual axis, then hide both headers. You now have two independent Marks cards stacked in the same space — one for shape, one for labels or a second encoding. That's how KPI tiles, custom bullet charts and most "impossible" layouts get made.

Worked example: a KPI tile from nothing

The rest of the toolkit

Dual axis + different marks

The universal escape hatch. Bars behind a line, circles on bars, two map layers. Synchronise if units match; hide the duplicate axis.

Size slider tricks

Bar-in-bar is a dual axis with both as bars, synchronised, front bar's Size dragged narrow. Lollipop is a very thin bar plus a circle.

Transparent marks

Opacity 0% keeps a mark's position and labels while hiding the mark — floating text at a computed position.

Ad-hoc calculations

Double-click an empty area of any shelf and type a formula. It becomes a pill without cluttering the Data pane.

Gantt marks

A Gantt bar starts at its value and extends by whatever is on Size — the right mark for ranges, durations and deviation bars.

Reference lines as chart parts

A bullet chart is a bar plus a reference line for target plus distribution bands. The "chart" is mostly reference lines.

How to reverse-engineer any chart you see

Three questions in order: What's the mark type? How many axes are there? What's on Size, Color and Label? Nearly every custom Tableau chart resolves into a dual axis, an unusual mark type, and something clever on Size. On Tableau Public you can download most published workbooks and open them — the fastest way to learn a technique nobody documented.

🧠 Key intuitions

  • Four formatting levels — workbook, worksheet, field, element. Most specific wins
  • Set fonts and palette at workbook level first, then make deliberate exceptions
  • Header vs Pane — labels vs data area. The reason your font change did nothing
  • Most formatting is deleting — gridlines, dividers, field labels, legends
  • Bar charts include zero; small multiples use a uniform axis
  • Centre diverging palettes on zero manually, every time
  • Stepped Color when readers need to read values, not just see a pattern
  • Marks to Label → Line Ends replaces a legend on line charts
  • Dynamic titles make filtered screenshots self-describing
  • Custom number formats have four parts — positive; negative; zero; text
  • MIN(0) placeholder axes are how custom layouts get built
  • Colour one thing, grey the rest

🎯 Practice exercises

  1. Set fonts and a base colour at Format → Workbook and confirm every sheet picks them up
  2. Open the Format pane and change a font under Header, then under Pane — describe what moved
  3. Run the 60-second cleanup on an existing chart, following all six steps
  4. Take a bar chart, uncheck Include zero, and screenshot before and after
  5. Build small multiples per region and switch between Uniform and Independent ranges
  6. Add row banding to a text table with a band size matching a nested group
  7. Colour a map by Profit, then fix it with Advanced → Center = 0
  8. Switch that map to Stepped Color with 4 steps and say which you'd publish
  9. Build a multi-line chart and try all three Marks to Label settings, then delete the legend
  10. Write a custom tooltip with sales, share of total, margin and an instruction line
  11. Add a Viz in Tooltip showing each region's trend on hover
  12. Add a mark annotation on the biggest month, including the live value
  13. Write a dynamic title containing a parameter and a filter value, then change both
  14. Apply ▲0.0%;▼0.0%;— to a growth measure
  15. Build a KPI tile from a MIN(0) placeholder axis, following the four steps

🏆 Challenge: rebuild one chart to production standard

Take any chart from this section and finish it properly. Every axis deliberate, gridlines gone, numbers in the right units, one accent colour with everything else grey, direct labels instead of a legend, a written tooltip, one annotation on the thing that matters, a dynamic title, and a caption with the data's as-of date.

Then put before and after side by side, as in the cleanup example above. The data is identical — every difference is a formatting decision. That comparison is the most convincing argument there is for spending time on this.
