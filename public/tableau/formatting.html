🎛️ Formatting & Custom Charts

Tableau's defaults are functional, not finished — and almost every part of a view is editable if you know where the setting lives. This page is the map: the formatting hierarchy, every region of a chart and what controls it, the panels that sit beside it, and the techniques for building charts Show Me has never heard of.

🪜 The formatting hierarchy

Before any specific setting: Tableau formats at four levels, and the most specific one wins. Knowing which level you're editing is the difference between changing one thing and changing everything.

  1. Workbook Format → Workbook
    Fonts and colours for every sheet, present and future. Set this first on any real project.
  2. Worksheet Format → Font / Lines / Shading…
    Everything in this one sheet. The Format pane on the left.
  3. Field right-click a pill → Format
    That field wherever it appears in this sheet — its headers and its panes.
  4. Single element right-click the thing itself → Format
    One axis, one header, one annotation. Overrides everything above.
Work top-down, not bottom-up

Set fonts and the base palette at workbook level once. Then only reach for lower levels to make a deliberate exception. The alternative — formatting each sheet individually — produces a dashboard where ten charts are each slightly different, and nobody can say why.

Undoing formatting: the Format pane has a Clear button (an eraser) at the top that resets the current sheet's formatting to default. Right-clicking any individual element also offers Reset. Both are safer than hunting for the one setting you changed an hour ago.

🗂️ The Format pane and its two-column logic

Open it with Format → Font (or right-click in the view → Format). It replaces the Data pane on the left. At the top are three tabs — Sheet, Rows, Columns — and inside each, settings are split into two columns that catch people out constantly.

Header

The labels — row and column headers, the field names down the side and across the top. Text, not data.

Pane

The data area — where the marks and numbers live. Everything inside the chart itself.

Changing the font under "Header" and wondering why your numbers didn't change is the standard first experience of this pane. There's also a Total column when totals are showing, so you can style them separately — usually bold, which reads well.

📏 Axes — full control

Right-click any axis → Edit Axis. This dialog decides whether your chart tells the truth.

SettingWhat it doesWhen to change it
AutomaticTableau picks the range from the data in each paneThe default; fine for a single pane
Uniform axis rangeEvery pane shares one rangeEssential for small multiples — otherwise each panel is scaled differently and comparison is meaningless
Independent rangesEach pane scales to its own dataOnly when you care about shape, not magnitude — and say so

ReversedFlips the directionRankings, where 1 should be at the top
Titles & subtitlesAxis text, or blank to removeDelete when the chart title already says it

⚠️ The two axis settings that mislead

A bar chart that doesn't start at zero. Bars say "this is twice as long, so it's twice as big". Cut the baseline and a 3% difference looks like 300%. Lines may start elsewhere when variation is small — bars may not, ever.

Independent ranges on small multiples. Six panels that each look similar because each was scaled to fit. Set Uniform and the real differences appear.

Dual axes: right-click the secondary axis → Synchronize Axis to force both onto one scale. Do that whenever the two measures share units. When they don't, leave them independent but make sure both axes are clearly titled — the reader has no other way to know which scale belongs to which series.

🏷️ Headers, field labels and cell size

Hide the field labels

The grey bar reading "Region" above your row headers is a field label, separate from the headers themselves. Right-click it → Hide Field Labels for Rows. Removing these is usually the second-biggest visual improvement after gridlines.

Hide a header entirely

Right-click any axis or header → untick Show Header. Essential for the placeholder-axis trick further down, and for hiding a redundant duplicate axis on a dual-axis chart.

Rotate long labels

Right-click a column header → Rotate Label, or Format pane → Alignment → Direction. Better still: switch to horizontal bars so the labels read straight.

Cell size

Drag the edge of a header, or use Format → Cell Size for exact control. Taller rows in a dense text table make it far more readable.

Rename without changing the data

Double-click a pill in the view (or right-click → Edit Alias for a dimension member) to rename what's displayed. Aliases change the label only — the underlying field and every calculation referencing it are untouched. This is how "Office Supplies" becomes "Office" when the column is too narrow.

The structural furniture of a view. Most improvements here are removals.

The 60-second cleanup that works on any chart
  1. Grid lines → None
  2. Row and column dividers → None
  3. Hide field labels for rows and columns
  4. Delete axis titles the chart title already covers
  5. Format numbers as $0K rather than raw decimals
  6. Remove the legend if there's only one series

Six removals, no additions. Do them on every sheet before you show anyone.

📎 The panels beside your chart

Tableau parks a stack of cards to the right of the view. Each is a separate object you can show, hide, retitle, restyle and — on a dashboard — move anywhere. Knowing the full list is how you decide what a viewer needs within reach.

The Highlighter is the underused one

Right-click a dimension → Show Highlighter and you get a search box beside the chart. Typing dims everything that doesn't match while keeping it visible — so a viewer can find one product line among fifty without losing the context of the rest. On a tangled multi-line chart it's better than a filter, because the comparison survives.

On a dashboard, these don't come across automatically. Select the sheet, open its ▾ → Legends / Filters / Parameters and add the ones you want — or drag them in from the Dashboard pane. They can be tiled into the layout or floated on top.

🗺️ Legends in detail

A legend is the "colour-coded key beside the chart". It's also an object with a surprising number of settings.

Discrete legend — a key

When a dimension is on Color you get one swatch per member. Every member is independently recolourable, and the order is editable by dragging.

Continuous legend — a scale

When a measure is on Color you get a gradient with end values. Readers estimate from it rather than look things up — which is why stepped colour is often clearer.

A single-series chart needs no legend. A two-series chart is usually better with direct labels on the lines — the reader's eye doesn't have to travel to a key and back. Keep legends for three or more categories, or when colour encodes a scale rather than an identity.

🌈 Colour, in depth

Palette types

Categorical

Unordered groups. Stop at about seven before colours stop being distinguishable.

Sequential

One direction, low to high. Sales, volume, count.

Diverging

A meaningful midpoint. Profit and loss, above and below target.

The Edit Colors dialog

Click Color on the Marks card → Edit Colors. What you see depends on whether the field is discrete or continuous.

Discrete field
  • Pick a palette, then Assign Palette to apply it to all members at once
  • Or select one member and click a swatch to set it individually
  • Double-click a swatch for a full colour picker with hex entry
Continuous field
  • Reversed — flip the direction
  • Stepped Color — discrete bins instead of a gradient
  • Include Totals — whether totals participate in the scale
  • Advanced — fix the Start, End and Center values
Stepped Color is the Excel-conditional-formatting equivalent

A continuous gradient asks the reader to judge shades. Stepped Color with 5 steps gives you five discrete bands, so the legend becomes a set of ranges rather than a smear — much easier to read a value out of, and much easier to describe in a meeting.

Five steps. Every cell in a band is identical, so equal values look equal.

⚠️ Always set the diverging centre by hand

Tableau centres a diverging palette on the data's midpoint, not on zero. Colour states by Profit when every state is profitable and the weakest ones still render red — the chart claims losses that don't exist.

Fix: Edit Colors → Advanced → tick Center and enter 0. Fix Start and End too if you want the scale stable as filters change.

Beyond the palette

Opacity

The slider on the Color card. Drop to 60–70% on scatter plots so overlapping points reveal density. Set to 0% to make marks invisible while still occupying space — the basis of several custom-chart tricks.

Borders

Color card → Effects → Border. A thin white border between stacked bar segments or dense squares makes boundaries readable without adding gridlines.

Halo

Color card → Effects → Halo. A ring around each mark, so dark points stay visible on a dark map background.

Custom palettes

For brand colours, edit Preferences.tps in My Tableau Repository, define a palette with hex values, and restart Tableau. It then appears in the palette dropdown on every workbook.

Colour one thing, grey the rest

The highest-impact colour decision in any chart. If one category is the story, give it your accent colour and make everything else a neutral grey. A chart where all twelve bars are different colours has no focal point — the reader has to work out what matters. Assign the greys explicitly in Edit Colors; it takes a minute and transforms the chart.

Accessibility: roughly 1 in 12 men can't reliably separate red from green. Never let that pair be the only signal — pair colour with position, a label, or a shape. Tableau's "Color Blind" categorical palette is a safe default for qualitative data.

📐 Size and shape

Size

The slider sets a uniform size; dropping a measure on Size makes it vary by value.

  • Bars — the slider controls width, which is how bar-in-bar charts are made
  • Circles — area, not radius, so the mapping is perceptually honest
  • Lines — thickness

Right-click the size legend → Edit Sizes to set the value range the sizes map to, and how wide the smallest-to-largest spread is.

Shape

Available on Circle/Shape mark types. Click Shape on the Marks card to pick a palette — filled, open, arrows, KPI symbols.

  • Assign a specific shape per member, like colours
  • Custom shapes: drop PNGs into My Tableau Repository → Shapes → <your folder>, then click Reload Shapes

Shape is a weak channel — people distinguish maybe four or five reliably. Use it for a second, secondary dimension, never for the main comparison.

🔖 Labels

Click Label on the Marks card. Beyond the on/off checkbox there's a set of options that decide whether labels help or clutter.

OptionWhat it does
TextOpens an editor — combine fields, static text and line breaks. "<SUM(Sales)> in <Region>"
Marks to Label: AllEvery mark. Fine on a bar chart with eight bars, terrible on a scatter with two hundred points
Min/MaxOnly the extremes — often exactly the right amount of labelling
Line EndsLabels at the end of each line. This is how you replace a legend on a multi-line chart
Selected / HighlightedLabels appear only on interaction — clean by default, informative on demand
AlignmentPosition relative to the mark, plus rotation and wrapping
Allow labels to overlapOff by default, which is why some labels silently disappear when space is tight
Line Ends is the setting worth remembering

On a chart with four lines, set Marks to Label → Line Ends and delete the colour legend. Each line is now named where it finishes, so the reader identifies it without moving their eyes off the data. It's a small change that makes multi-series line charts dramatically easier to read.

If a label you expected isn't showing, it's almost always the overlap rule. Either allow overlap, make the cells bigger, or reduce how many marks you're labelling — the third option is usually right.

💬 Tooltips

Every mark already has a hover surface. It costs no space on the chart and it's the cheapest place to add depth.

<b>West</b>
Sales   $725K   (31.6% of total)
Profit  $108K   (14.9% margin)
Orders  1,611

<i>Click to filter the dashboard</i>

Context, share, margin, and an instruction.

The editor

  • Insert menu — drop in any field on the sheet, a parameter, the sheet name, or All Fields
  • Formatting — font, size, colour, bold and italic per line, like a tiny word processor
  • Show tooltips — uncheck to disable entirely on decorative sheets
  • Responsive / On Hover / On Click — On Click is better on touch devices
  • Include command buttons — the Keep Only / Exclude / sort controls. Turn off for published dashboards where you don't want viewers restructuring the view
  • Allow selection by category — clicking a tooltip value selects all marks sharing it
Viz in Tooltip — the best value feature in Tableau

In the tooltip editor, Insert → Sheets → pick another worksheet. Hovering a region now pops up a small chart of that region's monthly trend, filtered automatically to what you're hovering.

You get a whole second level of detail without spending a pixel of dashboard space. Keep the embedded sheet small and simple — it renders on every hover — and set maxwidth and maxheight in the inserted tag to control its size.

✍️ Annotations

Right-click in the view → Annotate. Three kinds, and the difference is what they're anchored to.

Mark

Attached to a specific data point. Moves with it when the data changes, and disappears if that mark is filtered out.

Point

Attached to a fixed spot on the axes. Stays put regardless of the data.

Area

A shaded region of the view. Good for calling out a whole period — "supply chain disruption".

Annotations can carry live values

In the annotation editor, right-click → Insert and pick a field. The text then updates with the data: "Peak of <SUM(Sales)> in <MONTH(Order Date)>". Format the box, line and shading by right-clicking the annotation → Format.

One annotation pinned to the anomaly does more than a paragraph underneath the chart, because the reader's eye is already there. Use one or two per view, not six.

📰 Titles and captions — with live text

Double-click the title to edit it. The Insert menu in that editor is the part people miss: titles can contain live values.

InsertProduces

<Parameters.Select Measure>The current value of a parameter
<Region>The current filter selection for that field
<Page Name>The current value on the Pages shelf
 
// A title that describes itself as the viewer filters
<Parameters.Select Measure> by Sub-Category — <Region>

// Renders as:
Profit by Sub-Category — West
Profit by Sub-Category — All
Dynamic titles solve the screenshot problem

Someone filters a dashboard to one region and screenshots it. Without a dynamic title, that image says "Sales by Sub-Category" and nothing about which region — and it will end up in a deck with no way to tell. Putting the filter value in the title makes every screenshot self-describing.

Captions (Worksheet → Show Caption) sit below the view and auto-generate a description you can overwrite. Use them for definitions, data-as-of dates and caveats — the things that belong near the chart but shouldn't compete with it.

🔢 Number formats, including custom strings

Right-click a field → Format → Numbers for this sheet, or Default Properties → Number Format to set it on the field everywhere. Always prefer the second.

The built-in options cover Number, Currency, Percentage and Scientific. Custom takes a format string, and that's where the control is.

The four-part structure

A custom format string has up to four sections separated by semicolons:

positive ; negative ; zero ; text

So ▲0.0%;▼0.0%;— means: positive gets an up triangle, negative gets a down triangle, zero shows a dash. Combine that with conditional colour on the Marks card and you have a KPI change indicator with no calculated field at all.

Formatting is display only. Showing $1.2M doesn't round the underlying value — sorting, filtering and calculations still use the full number. That's usually what you want, but it does mean two rows can display as "$1.2M" and sort in an order that looks wrong. If exactness matters at the displayed precision, round in a calculation instead.

🛠️ Building genuinely custom charts

Almost every "how did they build that?" Tableau chart comes from a small set of techniques. Here they are.

The placeholder axisThe single most important custom-chart trick

Put a constant on a shelf to create a blank layer you can draw anything on.

// Type these directly onto Rows or Columns
// (double-click the empty part of the shelf and type)
MIN(0)     // a flat axis at zero
AVG(0)     // same thing, different aggregation
MIN(1)     // a flat axis at one

Add a second placeholder, make it a dual axis, then hide both headers. You now have two independent Marks cards stacked in the same space — one for the shape, one for labels or a second encoding. That's how KPI tiles, custom bullet charts and most "impossible" layouts are made.

Dual axis + different marks

The universal escape hatch. Bars behind a line, circles on top of bars, two layers of a map. Synchronise if the units match; hide the duplicate axis with Show Header.

Size slider tricks

Bar-in-bar is one dual axis where both are bars, synchronised, with the front bar's Size slider dragged narrow. Lollipop is a very thin bar plus a circle.

Transparent marks

Opacity 0% keeps a mark's position and labels while hiding the mark. Useful for floating text at a computed position.

Ad-hoc calculations

Double-click an empty area of any shelf and type a formula. It becomes a pill without cluttering the Data pane — ideal for experiments and placeholders.

Gantt marks

A Gantt bar starts at its value and extends by whatever is on Size. That makes it the right mark for ranges, durations and deviation bars.

Reference lines as chart parts

A bullet chart is a bar plus a reference line for target plus distribution bands for context. The "chart" is mostly reference lines.

Worked example: a KPI tile

  1. Create the blank canvas

    New sheet. Type MIN(0) onto Columns. You get a single flat axis and one mark.

  2. Put the number on it

    Set the mark type to Text, drag SUM(Sales) to Label, and format it large and bold.

  3. Add the comparison underneath

    In the Label editor, add a second line with your year-over-year table calculation, formatted with ▲0.0%;▼0.0%;— and coloured conditionally.

  4. Strip everything

    Right-click the axis → untick Show Header. Format → Lines → all None. Format → Borders → all None. Hide the title if the dashboard already labels it.

  5. Repeat and assemble

    Duplicate the sheet for each metric, then place four of them in a horizontal container on the dashboard. That's the KPI row from the dashboard project — and it's four placeholder axes.

How to reverse-engineer any chart you see

Ask three questions in order: What's the mark type? How many axes are there? What's on Size, Color and Label? Nearly every custom Tableau chart resolves into a dual axis, an unusual mark type, and something clever on Size. On Tableau Public you can download most published workbooks and open them — which is the fastest way to learn a technique that isn't documented anywhere.

🧠 Key intuitions

  • Four formatting levels — workbook, worksheet, field, element. Most specific wins
  • Set fonts and palette at workbook level first, then make deliberate exceptions
  • Header vs Pane — labels vs data area. The reason your font change did nothing
  • Most formatting is deleting — gridlines, dividers, field labels, legends
  • Bar charts include zero; small multiples use a uniform axis
  • Centre diverging palettes on zero manually, every time
  • Stepped Color when readers need to read values, not just see a pattern
  • Marks to Label → Line Ends replaces a legend on line charts
  • Dynamic titles make filtered screenshots self-describing
  • Custom number formats have four parts — positive; negative; zero; text
  • MIN(0) placeholder axes are how custom layouts get built
  • Colour one thing, grey the rest

🎯 Practice exercises

  1. Set fonts and a base colour at Format → Workbook and confirm every sheet picks them up
  2. Run the 60-second cleanup on an existing chart — six removals, nothing added
  3. Build small multiples of monthly sales per region and switch the axis between Uniform and Independent; explain which is honest
  4. Take a bar chart, uncheck Include zero, and describe how much the story changes
  5. Add row banding to a text table and set the band size to match a nested group
  6. Colour a map by Profit, then fix the palette with Advanced → Center = 0
  7. Switch that same map to Stepped Color with 5 steps and say which you'd publish
  8. Build a four-line chart, set Marks to Label → Line Ends, and delete the legend
  9. Write a custom tooltip with sales, share of total, margin and an instruction line
  10. Add a Viz in Tooltip showing each region's trend on hover
  11. Add a mark annotation on the biggest month that includes the live value
  12. Write a dynamic title containing a parameter and a filter value, then change both
  13. Apply the custom format ▲0.0%;▼0.0%;— to a growth measure
  14. Build a KPI tile from a MIN(0) placeholder axis and strip it completely bare

🏆 Challenge: rebuild one chart to production standard

Take any chart you've built in this section and finish it properly. Every axis deliberate, gridlines gone, numbers formatted in the right units, one accent colour with everything else grey, direct labels instead of a legend, a written tooltip, one annotation on the thing that matters, a dynamic title, and a caption stating the data's as-of date.

Then put the before and after side by side. The data is identical — every difference is a formatting decision. That comparison is the most convincing argument there is for spending time on this, and it's the difference between a chart that answers a question and one someone will actually act on.
