🗺️ Maps

Tableau knows where places are. Double-click a State field and you get a map — no coordinates, no setup. That convenience is also the danger: maps are the easiest chart to build and one of the easiest to mislead with.

📖 Terms on this page

Defined before they're used. Every term on the site is also in the glossary.

Geographic role
The location type Tableau assigns a field so it knows how to place it.
Example: State, City, Postal Code. Shown by a small globe icon.
📍 Right-click a field → Geographic Role
Generated field
A field Tableau creates that isn't in your data.
Example: Latitude and Longitude appear automatically once a field has a geographic role.
Geocoding
Looking up where a place name sits on the map.
Example: Tableau does this from a built-in database — no coordinates needed.
Choropleth / filled map
Regions shaded by value.
Example: States coloured by profit margin. Best for rates, since area distorts raw counts.
Symbol map
Circles placed at locations, usually sized by a measure.
Example: Cities sized by sales. Better than a filled map for raw totals.

🤔 When a map is actually the right chart

A map answers where. If your real question is how much, a sorted bar chart will answer it faster and more precisely.

✓ Use a map when
  • Spatial position is part of the insight — clusters, adjacency, distance
  • The audience thinks geographically (territory managers)
  • You're showing coverage or gaps
  • Routes or flows between places matter
✗ Use a bar chart when
  • You want a ranking — which states sell most
  • Precise comparison matters
  • Only a handful of places are involved
  • The pattern isn't geographic at all
Area is not value

On a US map, Montana is enormous and Rhode Island is a speck — regardless of their sales. A filled map inherently over-weights large, often sparsely-populated regions. That's fine when you're showing where something happens, and misleading when you're showing how much.

🌐 Geographic fields

Tableau auto-detects common location types and marks them with a globe icon. Those fields carry hidden Latitude and Longitude values it generates for you.

Auto-detected roles
🌐 Country / Region
🌐 State / Province
🌐 City
🌐 Postal Code / ZIP
🌐 County
🌐 Area Code
🌐 CBSA / MSA

If Tableau doesn't recognise a field

Right-click it → Geographic Role → pick the correct type. Common causes:

  • The column is named something unexpected
  • Codes rather than names ("CA" instead of "California")
  • It was read as a number — postcodes especially

Superstore's Country, State, City, Postal Code and Region are all detected automatically.

Have real coordinates? Set one field's role to Latitude and another's to Longitude, then put them on Rows and Columns as continuous measures. That's how you map anything Tableau doesn't have built-in geocoding for.

🎨 Map types

Filled map (choropleth)
Regions shaded by value. Best for rates and ratios — profit margin, sales per capita.
Symbol map
Circles sized by value. Best for raw totals — area doesn't distort the comparison.
Dual-axis map
Filled states plus city bubbles. Region context and point detail in one view.
Density map
Marks type → Density. For thousands of overlapping points where individual dots are unreadable.
The rule that keeps choropleths honest

Filled maps for rates, symbol maps for counts. Shading California dark because it has the most sales mostly tells you California is populous. Shading it by profit margin tells you something you didn't already know.

🌈 Colour on maps

Sequential

One direction, low to high. Sales, quantity, population.

Diverging

A meaningful midpoint. Profit and loss, above and below target.

Categorical

Unordered groups. Region, segment, territory owner.

⚠️ Centre the diverging palette on zero — it doesn't happen by default

Colouring states by Profit, Tableau centres the palette on the data's midpoint. If every state is profitable, the least profitable ones still render red — visually claiming losses that don't exist.

Fix: Color → Edit Colors → Advanced → tick Center and set it to 0. Do this on every profit map you ever build.

🗂️ Layers and styling

// Background style
Map → Background Maps → Light / Dark / Streets / Satellite

// Turn map features on and off
Map → Map Layers → Coastlines, Borders, City names,
                   Streets, Terrain, Zip code boundaries

// Data layers — free US demographic context
Map → Map Layers → Data Layer →
   Population, Median Household Income, Age…

// Lock the zoom so viewers can't get lost
Map → Map Options → uncheck Pan and Zoom
Strip the map back

The default background has roads, city names and terrain competing with your data. Turn most of it off — usually you want coastlines and borders and nothing else. The map is a backdrop; the data is the chart.

Dual-axis maps

  1. Build the base layer

    Double-click State, set Marks to Map, put SUM(Profit) on Color.

  2. Duplicate the Latitude pill

    Hold Ctrl / ⌥ and drag Latitude to create a second copy on Rows. You now have two stacked maps.

  3. Make it dual axis

    Right-click the second Latitude pill → Dual Axis. The maps overlay, and the Marks card splits in two.

  4. Style each layer separately

    Set the second Marks card to Circle, swap State for City on Detail, and put SUM(Sales) on Size. Filled states with city bubbles on top.

⚠️ Unknown locations

A small grey indicator in the bottom-right corner reading "42 unknown" means Tableau couldn't place some values. Those rows are silently missing from your map — and from any total shown on it.

Why it happens

  • Misspellings and inconsistent naming
  • Ambiguous city names — there are 30-odd Springfields
  • Abbreviations Tableau doesn't recognise
  • Missing country context on an international dataset

How to fix it

  • Click the indicator → Edit Locations
  • Match unrecognised values manually
  • Set the correct country/region context
  • For cities, put State on Detail to disambiguate
Never ignore the indicator

Unknown locations don't produce an error — they just vanish. If 200 rows fail to geocode, your map's totals are quietly short by those 200 rows, and nothing on screen says so. Check it every time, and reconcile the map's total against a plain bar chart of the same measure.

🧠 Key intuitions

  • Double-click a geographic field — fastest map there is
  • Maps answer "where", not "how much" — bar charts rank better
  • Filled maps for rates, symbol maps for counts
  • Area distorts — big empty states dominate a choropleth
  • Centre diverging palettes on zero manually, every time
  • Latitude and Longitude are generated — you never make them
  • Put State on Detail for city maps to resolve ambiguity
  • Always check the unknown-locations indicator
  • Strip the background — the map isn't the chart

🎯 Practice exercises

  1. Create a filled map of Sales by State by double-clicking State
  2. Change it to show Profit instead and apply a diverging palette
  3. Centre the palette on zero and describe how the map changed
  4. Build a symbol map of Cities sized by Sales and coloured by Profit
  5. Add State to Detail on the city map and check the unknown-locations count
  6. Compare Sales by State against profit margin by State — which map is more useful, and why?
  7. Build a dual-axis map: filled states with city bubbles on top
  8. Change the background style and turn off everything except borders
  9. Add a Region filter and use it to zoom the map to one territory
  10. Write a custom tooltip showing sales, profit, margin and order count per state
  11. Find every state with negative total profit — how many are there?
  12. Build the same "which states underperform" answer as a sorted bar chart and decide which you'd present
🏆 Challenge: where should we stop discounting?

Build a map coloured by profit margin (not profit) with bubble size showing sales volume — so you can see states that sell a lot but earn little. Add average discount to the tooltip. Then find the states where high volume meets negative margin, and check whether discount explains it. Pair the map with a sorted bar chart of the worst offenders, because the map shows where and the bar chart shows how bad. That pairing is a small dashboard, and it's the right shape for a geographic finding.
