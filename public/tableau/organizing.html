🗃️ Sorting, Groups & Sets

Raw fields rarely match the way you want to analyse them. Seventeen sub-categories are too many to read; a continuous Sales column needs buckets; "our top 20 customers" needs to be a thing you can filter by. Groups, sets, hierarchies and bins are how you reshape dimensions without touching the data.

📖 Terms on this page

Defined before they're used. Every term on the site is also in the glossary.

Group
Several members of a dimension merged into one. A fixed, hand-picked list.
Example: Combining East and West into "Coastal".
Set
A dimension split into In and Out, usually by a rule that re-evaluates itself.
Example: "Top 20 Customers by Sales" — put it on Color to compare them against everyone else.
Bin
A measure sliced into equal-width ranges, turning it into a dimension.
Example: Sales binned into $100 buckets, then counted — that's a histogram.
Hierarchy
Dimensions nested so you can drill down with a + on the pill.
Example: Category → Sub-Category → Product Name.
Alias
A display name for a dimension member. Changes the label only, not the data.
Example: Renaming "Office Supplies" to "Office" when the column is narrow.

⚡ Quick sort

The sort icon on the axis, or in the toolbar. One click, applies to the current view only.

🎯 Sort by field

Right-click the dimension pill → Sort → By Field. Sort Sub-Category by Profit while displaying Sales.

✋ Manual sort

Drag headers into the order you want. Right for Small/Medium/Large, wrong for anything measured.

Quick sort breaks inside dashboards

A toolbar quick sort is computed within the view and can behave unpredictably when a dashboard filter changes what's present. For anything you're publishing, use Sort → By Field, which is an explicit, stable definition. If you need the sort to respond to filters in a specific way, sort by a table calculation instead.

On a dashboard, drilling changes the view for everyone looking at it. If you want controlled exploration instead, use a parameter or a filter action — see Dashboards.

📦 Groups — combine members together

A group merges several values of a dimension into one. Select the members in the view (Ctrl-click), then click the paperclip icon — or right-click the field → Create → Group.

A chart someone can actually act on.

Select members in the view with Ctrl-click, then click the paperclip icon in the tooltip. The original field is untouched — you get a new grouped field beside it.

What groups are good for

  • Consolidating the long tail — an "Other" bucket for everything below a threshold
  • Fixing inconsistent data — merging "USA", "U.S.A." and "United States" into one member
  • Ad-hoc business groupings — combining regions into "Coastal" and "Inland"
  • Simplifying a colour legend — twelve colours become four
Groups are static — that's the trade-off

A group is a fixed list of members. If a new sub-category appears in the data next quarter, it lands in "Other" (or nowhere) until someone edits the group by hand. For anything rule-based — "everything under $10k" — use a calculated field or a set instead, so it updates itself.

📊 Bins — turn a measure into buckets

Bins convert a continuous measure into a discrete dimension by slicing it into equal-width ranges. This is how you build a histogram.

Right-click a measure → Create → Bins → set the bin size.

Superstore's sales distribution is heavily right-skewed — most orders are small, a few are enormous. That shape is why the mean ($230) is four times the median ($54).

Bin size changes the story

Too wide and you flatten real structure into one block; too narrow and you get noise. Try a few sizes before settling. Better still, create a parameter for the bin size and let the viewer adjust it — it takes two minutes and makes the histogram genuinely explorable.

🎯 Sets — a dynamic in/out membership

A set splits a dimension's members into In and Out. Unlike a group, a set can be defined by a rule and re-evaluates itself as the data changes.

Dynamic set — a rule

Right-click a dimension → Create → Set → Top tab.

  • Top 20 customers by Sales
  • Sub-categories with negative Profit
  • States above the average

Membership recalculates automatically as data or filters change.

Fixed set — a list

Select marks in the view → the set icon in the tooltip.

  • A hand-picked list of key accounts
  • Regions in a specific pilot programme

Stays exactly as you defined it until you edit it.

A set is just a TRUE/FALSE membership. That's why it works on Color, on Filters, and inside calculations without extra work.

Three ways to use a set

  1. As a filter

    Drop the set on the Filters shelf and keep only In. Equivalent to a Top N filter, but reusable across every sheet.

  2. On Color — the comparison view

    Drop the set on Color and you get In vs Out side by side. "Our top 20 customers versus everyone else" in one drag — the single most useful thing sets do.

  3. Inside a calculation

    A set returns TRUE/FALSE, so it works directly in logic: IF [Top 20 Customers] THEN [Sales] END.

Set Actions — genuinely powerful interactivity

Dashboard → Actions → Change Set Values. Now clicking marks in one chart redefines the set, and every view using it updates. That's how you build a dashboard where a user selects any group of products and instantly sees them compared against the rest — without writing a parameter or a single calculation.

➕ Combined sets

Select two sets in the Data pane, right-click → Create Combined Set. You get the standard set operations, and they answer questions that are awkward any other way.

Union (all members)In either setEveryone who is a top customer or a recent customer

The churn analysis in three steps

Build "Customers in 2016" and "Customers in 2017" as two sets, then combine with Except. The result is your lapsed customers — a list a sales team can act on this afternoon. No calculations required.

Group vs set in one line: a group answers "these things belong together"; a set answers "these things meet a condition". If the answer could change when the data does, you want a set.

🧠 Key intuitions

  • Always sort by value unless the categories have a natural order
  • Sort → By Field beats the quick-sort button for anything published
  • Build hierarchies immediately — Product and Location take seconds
  • Groups are static; sets are dynamic
  • A set on Color is the fastest "these vs the rest" comparison there is
  • Bins turn measures into dimensions — that's how histograms happen
  • Combined sets answer churn and overlap questions with no calculations
  • If a rule defines it, don't hand-pick it — hand-picked lists go stale

  10. Build two sets — customers in 2016, customers in 2017 — and combine with Except to find churn
  11. Use a set inside a calculated field: IF [Top 20 Customers] THEN [Sales] END
  12. Add a Set Action so clicking a bar in one chart redefines the set
🏆 Challenge: how concentrated is the customer base?

Build a view answering "what share of revenue comes from our top 10% of customers?" Create a dynamic Top N set driven by a parameter, put it on Color, and show the In/Out revenue split as a percentage. Then move the parameter from 5% to 50% and watch the curve. If the top 10% drive far more than 10% of revenue, that concentration is both an opportunity and a risk — and it's the kind of finding that changes how a business thinks about its accounts.
