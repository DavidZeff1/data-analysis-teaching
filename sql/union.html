UNION

What it does

UNION combines results from two or more SELECT statements into a single result set. It stacks rows vertically.

Types

UNION      - Combines and removes duplicates
UNION ALL  - Combines and keeps duplicates (faster)

Syntax

SELECT columns FROM table1
UNION
SELECT columns FROM table2;

Examples

-- All names from artists and customers
SELECT Name AS Name FROM Artist
UNION
SELECT FirstName FROM Customer;

-- Cities from both customers and employees
SELECT City, 'Customer' AS Type FROM Customer
UNION
SELECT City, 'Employee' AS Type FROM Employee;

-- UNION ALL to keep duplicates
SELECT Country FROM Customer
UNION ALL
SELECT BillingCountry FROM Invoice;

-- Multiple UNIONs
SELECT 'Artist' AS Type, Name FROM Artist WHERE Name LIKE 'A%'
UNION
SELECT 'Album' AS Type, Title FROM Album WHERE Title LIKE 'A%'
UNION
SELECT 'Track' AS Type, Name FROM Track WHERE Name LIKE 'A%';

-- ORDER BY applies to final result
SELECT Name, 'Artist' AS Source FROM Artist
UNION
SELECT Title, 'Album' FROM Album
ORDER BY Name
LIMIT 10;

Key intuitions

  • All SELECTs must have the same number of columns
  • Column types should be compatible
  • Column names come from the first SELECT
  • UNION ALL is faster because it skips duplicate checking
  • ORDER BY goes at the very end, applies to combined result
  • Use UNION to merge similar data from different tables

Practice

  1. Get a list of all unique countries from both Customer and Invoice tables
  2. Combine employee and customer names with a column indicating their type
  3. Get all emails from both Employee and Customer tables
  4. Find all names (artists, albums, tracks) that contain the word "Love"
  5. Get all cities where we have either customers or employees, sorted alphabetically
