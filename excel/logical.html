Logical Functions

What they do

Logical functions make decisions based on conditions. They return different results depending on whether conditions are TRUE or FALSE.

Key Functions

IF(condition, if_true, if_false)
IFS(cond1, val1, cond2, val2, ...)
AND(cond1, cond2, ...)  - All must be true
OR(cond1, cond2, ...)   - Any must be true
NOT(condition)          - Reverses true/false
IFERROR(formula, if_error)

Examples

Using Superstore: Profit=Q, Sales=N, Discount=P

// Simple IF - is profit positive?
=IF(Q2>0, "Profit", "Loss")

// Nested IF - categorize profit
=IF(Q2>100, "High", IF(Q2>0, "Low", "Loss"))

// IFS - cleaner multiple conditions
=IFS(Q2>100, "High", Q2>0, "Low", Q2<=0, "Loss")

// AND - both conditions must be true
=IF(AND(Q2>0, P2=0), "Profitable No Discount", "Other")

// OR - either condition can be true
=IF(OR(L2="East", L2="West"), "Coastal", "Central")

// IFERROR - handle divide by zero
=IFERROR(Q2/N2, 0)

// Combine with calculations
=IF(N2>500, N2*0.1, 0)  // 10% bonus on sales over 500

Key intuitions

  • IF is the foundation - learn it well before others
  • IFS is cleaner than nested IFs for multiple conditions
  • AND/OR go inside IF, not standalone
  • IFERROR catches #DIV/0!, #N/A, #VALUE! errors
  • Conditions use: =, <>, >, <, >=, <=

Practice

  1. Create a column that shows "Discounted" if Discount > 0, otherwise "Full Price"
  2. Categorize Sales into "Small" (<100), "Medium" (100-500), "Large" (>500)
  3. Flag orders that are both "Consumer" segment AND have negative profit
  4. Create a "Priority" column: "High" if Same Day shipping OR Sales > 1000, else "Normal"
  5. Calculate profit margin (Profit/Sales) but show "N/A" if Sales is 0
